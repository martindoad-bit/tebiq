# TEBIQ 50 位高阶产品经理独立评审

日期: 2026-04-28

对象: TEBIQ 当前产品形态摘要、已实现功能、商业模型、合规边界、视觉定位与后续路线图。

说明: 本文重新执行两个任务:

1. 用 50 位不同背景的资深产品经理独立审视 TEBIQ,每位 PM 都给出重构级建议。
2. 在 50 个视角之后追加跨 PM 综合: 共识问题、共同优势、差异化洞察、以及“只能听 1 位 PM 时该听谁”。

原则:

- 不把 50 位 PM 写成 50 个同温层角色。
- 不重复普通 SaaS PM 话术。
- 每位 PM 都从自己的行业经验、业务约束和成功/失败样本里看 TEBIQ。
- 允许激进,但所有建议必须能落到产品、增长、商业或组织能力上。

核心判断先行:

TEBIQ 当前不是一个“AI 翻译日文文件”的产品,而是一个处在更大机会门口的产品: **在日中文用户的生活事务操作系统**。拍照即懂只是入口;真正的商业壁垒会来自 Document / Case / Task / Profile / Reminder / Expert Escalation 之间的闭环。

## 1. 增长飞轮 PM

背景: 字节增长、外卖本地生活、美团到店业务出身,长期负责从内容获客到交易转化的漏斗。

### 一句话判断

TEBIQ 有强触发点,但还没有增长飞轮;现在更像“用户焦虑来一次,解释完走一次”。

### 他会盯的核心问题

他不关心首页文案是否漂亮,先问四个数: 自查完成率、结果页拍照率、首拍后 7 日复访率、免费到订阅转化率。当前链路把焦虑导向一次 AI 解释,没有把用户导向下一次打开。邀请送 7 天会员也偏弱,因为它奖励的是拉人,不是共同完成一件生活事务。

### 重构级建议

把增长主线从“内容 -> 自查 -> 拍照 -> 付费”改成“内容 -> 自查 -> 个人事务档案 -> 自动待办 -> 拍照补证 -> 提醒 -> 复访”。增长不应只靠小红书/抖音不断灌新用户,而要让每个用户沉淀一个持续状态: 我的在留、税务、保险、区役所事项现在安全吗。

### 90 天验证动作

上线“续签准备进度”轻量版,自查后自动生成 3-5 个待办,每个待办可以拍照补充材料和设置提醒。验证首拍后 7 日复访率是否提升 30% 以上。

### 最尖锐挑战

不要把“用户还会收到日本文件”当作留存策略,那是自然频次,不是产品飞轮。

## 2. C 端订阅 PM

背景: Netflix / Spotify / 知乎付费 / 网易云会员出身,专长是订阅心理、权益包装、取消挽留和年付转化。

### 一句话判断

¥980/月不贵,但“无限拍照 + 文字”是弱订阅理由,它听起来像买 AI 次数,不是买生活安全感。

### 他会盯的核心问题

用户为什么现在买? 为什么下个月不取消? 为什么年付? 当前答案不够强。用户不是每天都有日本文件,所以订阅不能围绕使用频次定价,必须围绕“重要事项有人守着”定价。

### 重构级建议

重写订阅分层:

- 免费: 临时看懂一份文件。
- ¥980/月: 个人生活文件守护,含提醒、归档、待办、风险追踪。
- ¥9,800/年: 年度在留与生活事务守护,强调覆盖一个完整在留年度。
- 家庭版候选: 2-4 人档案、共同提醒、一键转发。

当前 ¥19,800 高级会员不建议继续主推。“AI 优先服务 + 早期功能”缺乏情绪价值,更适合改成家庭版或年度重点事项包。

### 90 天验证动作

付费页做 A/B: A 讲无限次数,B 讲期限提醒、重要文件归档、续签风险持续追踪。订阅 PM 会押 B 胜出。

### 最尖锐挑战

用户会为“别让我错过重要事”续费,不会长期为“AI 可以多问几次”续费。

## 3. 工作流工具 PM

背景: Notion / Linear / freee 出身,擅长对象模型、工作台、状态流转和任务闭环。

### 一句话判断

TEBIQ 最大的产品缺口不是功能少,而是缺少 Document / Case / Task 这套主对象模型。

### 他会盯的核心问题

用户拍完一份文件后,它到底去了哪里? 是一条历史记录,还是一个可推进的事项? 多份文件如何归到同一个在留更新 Case? 续签自查和拍照结果是否互相补充? 如果没有主对象,所有功能都会变成一次性页面。

### 重构级建议

建立四个核心对象:

- Document: 文件、图片、解析字段、置信度。
- Case: 一件生活事务,如 在留期間更新、住民税、国民健康保険、保育園、確定申告。
- Task: 用户要做的一步,如补拍、缴费、去区役所确认、保存收据。
- Profile: 签证类型、家庭成员、历史文件、重要期限。

拍照即懂不是终点,而是创建或更新 Case。续签自查不是报告,而是生成“在留期間更新 Case”。

### 90 天验证动作

先做最小 Case 系统: 每个拍照结果允许保存为事项,AI 自动生成 1-3 个待办,用户能勾选和设置提醒。衡量事项保存率、待办完成率、同一事项二次打开率。

### 最尖锐挑战

没有 Case 和 Task,TEBIQ 的“档案”只是仓库;有了 Case 和 Task,档案才是护城河。

## 4. AI 评测与安全 PM

背景: Notion AI / Cursor / Perplexity / 企业 AI 平台出身,负责过模型评测、引用、置信度和安全策略。

### 一句话判断

TEBIQ 的 AI 不该追求“像专家一样多说”,而要追求“在高风险生活场景里稳定、可验证、会降级”。

### 他会盯的核心问题

期限识别准确率是多少? 金额识别准确率是多少? 信封误判率是多少? 用户档案注入后会不会诱导模型越界判断在留资格影响? 当前 12 条 prompt 约束必要但不够,因为合规不能只靠一句“不要输出法律建议”。

### 重构级建议

把 AI 输出改成固定 schema:

- document_type
- sender
- deadline
- amount
- field_confidence
- urgency_level
- user_relevance
- missing_information
- general_next_steps
- related_knowledge_ids
- compliance_flags
- retake_required

UI 只渲染 schema,不直接展示模型长文。不确定时默认降级: 请补拍、请确认字段、请打开内容、请补充背面。模型克制程度要成为核心质量指标。

### 90 天验证动作

建立 100-200 份日本生活文书测试集,每次 prompt 或模型变化跑回归。指标包括字段准确率、越界率、低置信度降级率和用户纠错率。

### 最尖锐挑战

TEBIQ 的 AI 护城河不是“更会回答”,而是“关键时刻知道不能回答”。

## 5. 移民服务平台 PM

背景: Boundless / Fragomen Tech / Deel Mobility 类跨境身份服务产品出身,熟悉移民流程的数字化边界。

### 一句话判断

TEBIQ 不应把自己定位成“签证顾问”,而应做“身份周期的材料与事项层”。

### 他会盯的核心问题

续签、永住、配偶者、经管、高度人材都不是问卷能解决的。真正有价值的是把用户事实、文件、期限和材料缺口整理到专业人士可接手的状态。否则 TEBIQ 会在用户最焦虑的地方被迫承担自己不该承担的判断责任。

### 重构级建议

把续签自查结果从“风险等级”改成“确认事项清单”:

- 已确认事实
- 待确认事实
- 待准备材料
- 关键期限
- 不适合 AI 判断的部分
- 可导出给行政書士的材料包

咨询入口也要变成“复杂情况升级”,而不是结果页里的销售按钮。

### 90 天验证动作

做“续签交接包 MVP”: 用户自查 + 拍照上传后,生成一份结构化 PDF/网页材料清单。找 5 位合作专家评估是否能减少首次沟通时间。

### 最尖锐挑战

用户不需要一个假装懂入管的 AI,需要一个把材料整理到足够清楚的系统。

## 6. 日本本地化 PM

背景: LINE / PayPay / Yahoo Japan / 日本本地生活服务出身,熟悉日本用户对信任、文书、支付和客服的预期。

### 一句话判断

TEBIQ 的中文语境很准,但必须更像一个在日本运营的可信服务,而不是海外团队做的 AI 小工具。

### 他会盯的核心问题

用户会问: 公司是谁? 数据在哪? 支付安全吗? 真的懂日本文书吗? 出错怎么办? 能不能删除? 有没有客服? 日本生活服务的信任来自细节,不是只来自 AI 能力。

### 重构级建议

产品化“日本可信度”:

- 上传前说明数据处理。
- 结果页标注不确定字段。
- 档案页提供删除和导出。
- 付费页明确会社信息、特商法、隐私政策、退款边界。
- 重要页面保留日文术语原文。
- 客服入口不要藏太深。

### 90 天验证动作

在上传和付费链路加入轻量信任说明,对比上传完成率、付费转化率和客服咨询率。短期摩擦如果换来高意向用户转化,就是正收益。

### 最尖锐挑战

在日本做生活事务产品,信任不是法务页,信任是每一步交互的默认姿态。

## 7. B2B HR SaaS PM

背景: SmartHR / Rippling / Deel / Workday 出身,长期做员工档案、雇佣合规和 HR 工作流。

### 一句话判断

TEBIQ 的 C 端入口成立,但更稳定的付费方可能是雇佣外国人的日本中小企业。

### 他会盯的核心问题

日本中小企业的 HR 不想处理中文沟通,也不想替员工解释区役所、税、保险、在留相关文件。员工的生活事务一旦出问题,会反向影响出勤、续签、离职和管理成本。C 端用户嫌 ¥980/月,B 端可能觉得每人每月几百日元很便宜。

### 重构级建议

不做完整 B2B SaaS,先做“雇主福利版”:

- 公司购买邀请码。
- 员工使用个人版 TEBIQ。
- 企业只看匿名概览: 使用人数、高频问题、是否存在大量续签准备需求。
- 不让雇主看到个人文件。

### 90 天验证动作

访谈 20 家雇佣中国员工的会社、派遣会社、餐饮连锁、介护机构。卖 ¥30,000-¥100,000/年的轻量试点,验证预算存在性。

### 最尖锐挑战

如果 TEBIQ 只卖给焦虑个人,会很累;如果也卖给承担管理成本的组织,商业会稳很多。

## 8. 定价与包装 PM

背景: Adobe / Canva / Dropbox 商业化 PM,擅长免费边界、套餐命名、价格锚点和升级路径。

### 一句话判断

TEBIQ 现在的价格点不少,但套餐逻辑不够像一个系统,更像几块临时收入拼在一起。

### 他会盯的核心问题

免费 3 次拍照、文字 5 次、¥980/月、¥9,800/年、¥19,800/年、¥980 材料包、¥9,800 咨询,这些 SKU 之间的升级关系是什么? 用户会不会困惑: 我到底该订阅、买材料包,还是找咨询?

### 重构级建议

把所有收入统一成三类:

- 守护型: 月/年订阅。
- 事件型: 续签准备包、搬家包、税务季包。
- 升级型: 专家咨询。

高级会员先不做“早期新功能”,改成“家庭守护”或“年度重点事项”。一次性材料包应能抵扣订阅或专家咨询,形成升级梯子。

### 90 天验证动作

做付费路径重构实验: 自查用户优先看到“续签准备包 + 年度守护”,拍照高频用户优先看到“个人守护月付”,复杂用户最后看到“专家升级”。

### 最尖锐挑战

价格不是数字问题,是用户在焦虑状态下理解“我买的到底是什么”的问题。

## 9. 内容平台 PM

背景: 小红书 / 知乎 / Reddit / Substack 出身,擅长内容供给、搜索、案例化和分发飞轮。

### 一句话判断

TEBIQ 的知识库是资产,但还没变成飞轮;真正有价值的是脱敏案例库。

### 他会盯的核心问题

50+ 篇文章是不是来自真实高频问题? 每篇内容有没有引导用户上传自己的文件? 用户会不会搜“收到这张纸怎么办”而不是搜抽象知识标题? 文章如果不和拍照、自查、案例相似度连接,SEO 价值会变弱。

### 重构级建议

建立编辑型匿名案例库,不是开放社区:

- 收到 住民税 決定通知書 怎么看?
- 公司让交 課税証明 怎么拿?
- 国民健康保険 高额通知要不要马上处理?
- 入管来信中哪些词表示期限?

每个案例都连接: 样例字段、常见误解、一般步骤、上传你的文件确认是否同类。

### 90 天验证动作

从历史查询和模拟样本整理 Top 50 高频文书,做 50 个案例页。衡量案例页到上传的转化率,而不是只看阅读量。

### 最尖锐挑战

不要做“知识库”,做“在日中文用户看过这类纸的集体记忆”。

## 10. 教育产品 PM

背景: Duolingo / Khan Academy / Coursera 出身,关心学习路径、完成度、掌控感和知识内化。

### 一句话判断

TEBIQ 不应该只让用户每次问 AI,还要让用户逐渐变得没那么怕日本生活文书。

### 他会盯的核心问题

用户用完一次后有没有更懂? 术语有没有被积累? 续签路径有没有完成度? 用户是否从“看不懂”进化到“知道先看期限、金额、发件机构和对象人”?

### 重构级建议

知识库升级成“生活能力路径”:

- 刚来日本 30 天。
- 第一次住民税。
- 续签前 90 天。
- 年末调整 / 確定申告。
- 家庭与孩子手续。

每条路径包含术语卡、样例文件、小测、待办和拍照入口。

### 90 天验证动作

做“续签前 90 天准备路径”,把自查结果映射到路径节点。衡量路径开始率、完成率、路径内拍照率和付费率。

### 最尖锐挑战

最强留存不是让用户永远依赖 AI,而是让用户感到自己在日本生活里越来越有掌控感。

## 11. Trust & Safety PM

背景: Airbnb / TikTok / 金融风控平台出身,负责欺诈、误导、敏感内容和平台责任边界。

### 一句话判断

TEBIQ 最大风险不是用户不来,而是用户把 AI 输出当成可以替代专业判断的承诺。

### 他会盯的核心问题

哪些文案会被用户理解为保证? 哪些场景会诱导用户延误处理? 哪些复杂 case 应该强制降级? 咨询入口是否会被误解为 TEBIQ 自己提供专业服务?

### 重构级建议

建立红色文件与红色场景:

- 入管出頭通知
- 不许可通知
- 督促 / 差押 相关通知
- 身份、税务、犯罪、婚姻重大变化
- 个案可否通过判断

命中红色场景时,AI 只做文件解释、期限提取、材料整理,不输出建议路径。

### 90 天验证动作

设计 red-flag classifier,人工标注 200 个样本。上线前先做内部评测,目标不是覆盖所有复杂场景,而是避免明显越界。

### 最尖锐挑战

越是用户焦虑的场景,越不能靠更自信的 AI 话术安抚。

## 12. 隐私安全 PM

背景: Apple / 1Password / 企业安全产品出身,关注敏感数据、最小化收集、用户控制权和信任界面。

### 一句话判断

TEBIQ 收到的是用户最敏感的生活文件,隐私控制必须成为核心功能,不是设置页里的附属项。

### 他会盯的核心问题

用户能否选择不保存? 能否局部遮挡? 能否一键删除? 文件是否用于训练? 图像是否长期保存? 家庭共享如何避免隐私泄露? 被邀请的人能看到什么?

### 重构级建议

上传前提供三种模式:

- 临时识别: 不入档案。
- 保存到档案: 可生成事项和提醒。
- 脱敏保存: 自动提示遮挡 My Number、账户号等敏感字段。

档案页提供删除、导出、访问记录。家庭共享默认只分享摘要和待办,不分享原图。

### 90 天验证动作

上线“不保存,仅本次识别”并观察首次上传率是否提升。对敏感产品,减少默认沉淀可能反而增加长期信任。

### 最尖锐挑战

如果用户不知道文件会怎样被处理,再强的 AI 都会被“不敢上传”卡死。

## 13. 专家网络 PM

背景: JustAnswer / GLG / LegalZoom 专家市场出身,擅长 AI 初筛、专家分发、质量控制和责任边界。

### 一句话判断

¥9,800 咨询不是简单导流,它应该成为“AI 整理后的专家升级层”。

### 他会盯的核心问题

什么 case 该推荐专家? 专家看到的材料是否结构化? 咨询后结果是否回流到用户 Case? 合作行政書士质量如何控制? 退款和期待管理怎么做?

### 重构级建议

建立 Expert Escalation 流:

1. AI 判断是否红色/复杂。
2. 生成交接包。
3. 用户选择是否付费咨询。
4. 专家基于交接包回应。
5. 用户 Case 更新下一步任务。

咨询不是外链跳走,而是 TEBIQ 事务系统的一部分。

### 90 天验证动作

找 2-3 位合作专家试运行 20 个模拟 case。衡量专家首次理解所需时间、用户满意度、咨询后待办完成率。

### 最尖锐挑战

专家服务如果不回流到产品,TEBIQ 只是在卖线索;回流到 Case,才是在建设系统。

## 14. 家庭协作 PM

背景: Life360 / Google Family / 家庭健康照护产品出身,关注家庭成员权限、提醒协作和代理处理。

### 一句话判断

TEBIQ 的高频付费动机可能不是“我自己看不懂”,而是“我要帮家人别出事”。

### 他会盯的核心问题

在日华人家庭里,谁实际处理文书? 可能是日语最好的人、收入最高的人、孩子、配偶、公司同事。产品如果只按单用户设计,会错过真实协作关系。

### 重构级建议

做家庭协作但从轻开始:

- 一键把结果摘要发给家人。
- 家庭成员共享期限提醒。
- 为父母/配偶建立档案。
- 权限分级: 只看提醒、看摘要、看原图。

家庭版可以成为比高级 AI 更强的付费权益。

### 90 天验证动作

先做“一键发给家人”与“共同提醒”。观察分享率、被分享者注册率、家庭版假门点击率。

### 最尖锐挑战

在生活事务里,真实用户常常不是“文件所有者”,而是“家里负责处理事的人”。

## 15. 弱势用户 / 可访问性 PM

背景: 公共服务数字化、银发产品、低识字/低数字能力用户研究出身。

### 一句话判断

TEBIQ 当前对年轻、懂产品的人友好,但最需要它的人可能只想知道三件事: 这是什么、急不急、下一步。

### 他会盯的核心问题

字体够大吗? 按钮够少吗? 用户是否知道拍哪里? 结果页是否太长? 低日语用户会不会被“风险、自查、档案、订阅”这些抽象词劝退?

### 重构级建议

做“简单模式”:

- 首页只保留拍文件、看待办、发给家人。
- 结果页默认三段: 这是什么、急不急、下一步。
- 语音读出。
- 高对比大按钮。
- 复杂说明全部折叠。

### 90 天验证动作

找 10 位 45 岁以上或低日语用户做可用性测试。目标不是问喜不喜欢,而是看能否独立完成上传、理解紧急程度、设置提醒。

### 最尖锐挑战

真正弱势的用户不会夸你功能完整,他们只会在第三步迷路后离开。

## 16. 金融 / 保险 PM

背景: Lemonade / Plaid / 日本保险服务出身,关注风险、保障、文件、理赔和长期信任。

### 一句话判断

TEBIQ 的情感位置更像“生活保障”而不是“工具”,这会影响订阅、文案和合作方向。

### 他会盯的核心问题

用户什么时候最愿意持续付费? 往往不是天天使用,而是相信“出事时它在”。保险产品低频但可订阅,靠的是风险覆盖心智。TEBIQ 也可以走“重要事项守护”逻辑。

### 重构级建议

把订阅页改成保障式表达:

- 不错过期限。
- 重要文件留档。
- 续签前提前准备。
- 家人可共同接收提醒。
- 复杂情况可升级给专家。

长期可以探索和外国人保险、租房保证、医疗翻译等服务合作,但 1.0 不要分散。

### 90 天验证动作

把年付页从功能清单改成年度守护清单,测试年付率。对比“无限拍照”与“全年重要事项提醒”哪个更能成交。

### 最尖锐挑战

TEBIQ 卖的不是使用频次,是用户相信自己不会在关键生活制度里孤立无援。

## 17. 税务 / 会计 SaaS PM

背景: freee / Money Forward / QuickBooks 出身,熟悉税、票据、申报、凭证和中小企业工作流。

### 一句话判断

税务类文书会成为 TEBIQ 的高价值场景,但边界必须比普通生活通知更严格。

### 他会盯的核心问题

住民税、源泉徴収票、確定申告、消费税、法人税相关通知混在一起,个人与法人责任不同。TEBIQ 如果只做“解释”,价值有限;如果做“该怎么报税”,会越界。

### 重构级建议

将税务场景分层:

- 个人生活税: 解释字段、金额、期限、支付方式。
- 个体/副业: 解释一般流程,提示保存凭证。
- 法人/经管: 只做文件整理和期限提醒,复杂事项升级税理士/专家。

不要把税务内容混进普通知识库,应建立“税务文件专区”并明确不是税务代理。

### 90 天验证动作

先支持 10 类高频税务文件的字段识别和提醒,不做申报建议。衡量税务文件保存率和提醒设置率。

### 最尖锐挑战

税务场景很值钱,也最容易让一个生活助手不小心变成无牌税务顾问。

## 18. GovTech PM

背景: 数字政府、公共服务、行政流程再设计出身,理解政府文书与市民服务的断点。

### 一句话判断

TEBIQ 应该把自己看成“民间行政可理解层”,不是政府流程替代品。

### 他会盯的核心问题

区役所、入管、税务署、年金事务所、保育园等机构发出的文件风格不同。用户痛点不是单纯日语,而是行政系统默认用户知道流程。TEBIQ 要补的是“文书 -> 事项 -> 下一步”的理解层。

### 重构级建议

按发件机构建立文书分类:

- 区役所
- 税务署
- 年金
- 健康保险
- 入管
- 学校 / 保育园
- 公司 / 雇主

每类都有固定字段、常见期限和一般处理路径。知识库按机构与事项双索引,不要只按文章主题。

### 90 天验证动作

选 5 个机构各 10 类文书,建立文书地图。拍照结果显示“这通常来自哪个机构,通常和什么事项有关”。

### 最尖锐挑战

用户不是在读日文,是在和一个看不见的行政流程打交道。

## 19. Super App PM

背景: 微信 / 支付宝 / LINE 生态产品出身,关注入口、轻应用、支付、消息和高频触点。

### 一句话判断

TEBIQ 作为独立 App/Web 可以成立,但高频触达更可能发生在消息和提醒里。

### 他会盯的核心问题

用户真的会每天打开 TEBIQ 吗? 不会。那提醒从哪里来? 邮件没配、SMS 没接、LINE 是否可用? 日本华人很可能习惯 LINE、微信、小红书,TEBIQ 要选一个最现实的触达链路。

### 重构级建议

1.0 不急着做 super app,但要把通知架构做好:

- 邮件 magic link 只是登录,不能当提醒体系。
- LINE 通知或邮件提醒至少要有一个可靠渠道。
- 结果页必须鼓励用户设置具体提醒。
- 分享摘要要适配微信/LINE。

### 90 天验证动作

优先接通邮件正式发送或 LINE 通知其中之一。衡量提醒设置率、提醒打开率、提醒后任务完成率。

### 最尖锐挑战

没有可靠触达渠道,所有“守护”都是产品里的自我感动。

## 20. Marketplace / 本地服务 PM

背景: Uber / Airbnb / 美团到家服务出身,擅长线上线下服务编排和供需质量。

### 一句话判断

TEBIQ 如果接专家咨询,就已经不是纯软件,而是进入服务网络管理。

### 他会盯的核心问题

专家供给够不够? 响应时效如何? 价格是否透明? 用户期待是否被管理? 服务失败谁承担? 咨询后的下一步是否落回产品?

### 重构级建议

专家咨询先做小而严:

- 限定问题类型。
- 限定响应时效。
- 明确交付物: 一次咨询纪要 + 下一步清单。
- 专家评分不公开娱乐化,但内部做质量管理。

### 90 天验证动作

用 30 个真实/模拟复杂 case 测专家供给流程,看从用户上传到专家输出的时间、满意度和后续任务完成率。

### 最尖锐挑战

你一旦卖咨询,用户买的是确定性服务体验,不是一个外链。

## 21. 数据平台 PM

背景: Amplitude / Mixpanel / Segment 出身,负责事件模型、指标体系和产品分析。

### 一句话判断

TEBIQ 现在最应该补的不是更多页面,而是一套能回答战略问题的数据事件模型。

### 他会盯的核心问题

用户从哪里来? 第一次使用什么功能? 哪些文件导致付费? 哪些结果导致流失? 免费额度耗尽是转化点还是挫败点? 没有事件模型,50 个 PM 的建议都无法验证。

### 重构级建议

建立北极星指标: “被成功推进的生活事项数”。配套事件:

- self_check_started/completed
- document_uploaded
- document_type_detected
- case_created
- task_created/completed
- reminder_set/fired/opened
- paywall_viewed/converted
- expert_escalation_viewed/paid

### 90 天验证动作

先埋 20 个关键事件,每周看 cohort。不要等数据完美,但必须能看出用户是否从一次性使用进入 Case 流。

### 最尖锐挑战

如果北极星还是“拍照次数”,团队会把产品优化成更贵的 OCR。

## 22. 搜索 / 答案引擎 PM

背景: Google Search / Perplexity / 知识问答产品出身,关注检索意图、引用、答案可信度和查询改写。

### 一句话判断

文字即懂不应只是粘贴解释,它可以成为“在日生活答案引擎”的入口,但必须有来源和边界。

### 他会盯的核心问题

用户粘贴一段日文,意图可能是理解、判断急迫性、准备材料、回信、还是确认术语。一个通用回答会很平。答案有没有引用知识库? 有没有指出不确定? 有没有下一步检索建议?

### 重构级建议

文字即懂改成意图分流:

- 术语解释
- 文件段落解释
- 期限/金额抽取
- 和我的档案相关
- 生成待办
- 找相似案例

结果中必须展示相关知识来源,但不要用“书士审核”背书。

### 90 天验证动作

对 200 条用户粘贴文本做意图标注,训练/规则化分流。衡量用户是否点击相似案例、是否创建待办、是否继续拍照。

### 最尖锐挑战

文字即懂如果只是翻译,它会被通用 AI 吃掉;如果能连接档案、案例和任务,它才是 TEBIQ。

## 23. 视觉与情感设计 PM

背景: Apple / Linear / Airbnb Design 出身,关注第一感、情绪节奏、微文案和品牌一致性。

### 一句话判断

“在日生活のお守り”是 TEBIQ 最有差异化的品牌承诺,但现在界面更像功能菜单,还不像护身符。

### 他会盯的核心问题

用户打开时是被稳住,还是被要求选择功能? 结果页是否先安抚、再解释、再行动? 不确定和错误状态是否足够诚实? 首页大 slogan 对首次用户有用,但对复访用户可能浪费首屏。

### 重构级建议

首页改成“今日状态”:

- 今天没有紧急事项 / 有 1 件需要确认。
- 在留期限还有 X 天。
- 最近文件。
- 即将到期任务。
- 主按钮: 拍一份新文件。

产品调性少一点工具箱,多一点生活手账。

### 90 天验证动作

做首页 A/B: 功能入口版 vs 今日状态版。衡量复访用户的待办点击率、提醒设置率和拍照发起率。

### 最尖锐挑战

“お守り”不是品牌口号,是用户每次打开时感到“有人替我看着”的状态。

## 24. Onboarding PM

背景: Dropbox / Slack / Figma 增长激活出身,专攻首次体验、aha moment 和权限请求时机。

### 一句话判断

TEBIQ 的首次体验应该让用户在 60 秒内看到“它懂我的这张纸”,而不是先理解产品结构。

### 他会盯的核心问题

新用户第一次来,是被要求注册、选择功能、读说明,还是直接上传? 信任说明要给,但不能压垮首次体验。拍照即懂和续签自查哪个更适合作为首个 aha? 不同流量来源答案不同。

### 重构级建议

按来源定制 onboarding:

- 小红书“收到这张纸”流量 -> 直接上传/案例页。
- 续签内容流量 -> 自查。
- 首页自然流量 -> 拍照 + 自查双入口。
- 老用户 -> 今日状态。

首次上传后再引导建档案,不要一上来要求用户理解档案价值。

### 90 天验证动作

给 3 类入口做不同首屏和 CTA,验证首次完成率和 7 日留存。重点看“首次上传前是否注册”的摩擦。

### 最尖锐挑战

不要让用户先相信你的系统,先让他相信你看懂了他手里那张纸。

## 25. Referral / Virality PM

背景: Dropbox / PayPal / 拼多多裂变增长出身,关注邀请动机、奖励成本和网络关系。

### 一句话判断

邀请朋友送 7 天会员太泛,TEBIQ 更强的裂变场景是“帮我看这份文件”。

### 他会盯的核心问题

为什么用户会邀请? 因为奖励,还是因为需要协作? 在敏感文书场景里,泛邀请可能弱;定向分享给家人/朋友/同事更自然。月度封顶 30 天合理,但没有解决邀请质量。

### 重构级建议

邀请体系重构为三类:

- 协作型: 请家人一起看待办。
- 求助型: 把摘要发给懂日语的朋友。
- 福利型: 朋友注册双方得会员。

优先做协作型,因为它连接真实关系和真实事项。

### 90 天验证动作

结果页增加“发给朋友确认”按钮,分享的是脱敏摘要而非原图。衡量分享打开率、被邀请注册率、二次上传率。

### 最尖锐挑战

TEBIQ 的传播不是炫耀型传播,而是求助型传播。

## 26. 国际化 / 跨境 PM

背景: TikTok 海外 / 滴滴海外 / Shopee 出身,专注跨文化产品适配和海外运营。

### 一句话判断

继续锁定中文母语者是对的,但“在日中文用户”内部也不是一个市场。

### 他会盯的核心问题

大陆、台湾、香港、新加坡华人用词不同;简体/繁体偏好不同;来日阶段不同;签证与收入差异巨大。产品文案用中文,但术语保留日文,这很好;下一步要做到场景分层。

### 重构级建议

用户分层至少按三轴:

- 来日阶段: 刚来、稳定工作、续签前、永住/归化、家庭阶段。
- 身份类型: 技人国、经管、配偶者、特定技能、留学等。
- 生活复杂度: 单身、家庭、创业、照护、孩子教育。

首页和知识推荐应根据这三轴变化。

### 90 天验证动作

注册/自查后生成轻量 persona 标签,用来调整首页模块和知识推荐。看推荐点击率和拍照转化率是否提升。

### 最尖锐挑战

中文不是一个足够深的用户分层,只是 1.0 的共同语言入口。

## 27. CRM / 生命周期 PM

背景: Amazon Prime / Booking / 电商会员生命周期出身,关注触达节奏、召回和生命周期价值。

### 一句话判断

TEBIQ 需要从“功能使用”运营升级到“用户生命周期”运营。

### 他会盯的核心问题

刚来日本用户 30 天内会遇到什么? 续签前 180/90/30 天该触达什么? 税务季该提醒什么? 年度更新什么时候推年付? 当前如果没有生命周期编排,用户只会在出事时想起你。

### 重构级建议

建立生命周期日历:

- 新来日 30 天: 国保、年金、住民票、税。
- 就职后: 源泉徴収票、年末調整。
- 续签前 180/90/30 天。
- 搬家后 14 天相关事项。
- 税务季。

所有触达必须连接到 Case/Task,不是群发文章。

### 90 天验证动作

先做“续签倒计时运营”: 用户输入在留期限后,自动生成 180/90/30 天提醒和准备路径。

### 最尖锐挑战

TEBIQ 如果没有生命周期运营,就只能被动等用户焦虑时回来。

## 28. B2B2C 渠道 PM

背景: 学校、保险、银行、通信运营商合作渠道出身,擅长通过机构触达终端用户。

小红书/抖音是获客入口,但 TEBIQ 还应该找“已经拥有在日中文用户”的机构渠道。

### 他会盯的核心问题

语言学校、专门学校、派遣会社、外国人不动产中介、SIM 卡服务、汇款服务、保险代理,都可能有用户触达能力。TEBIQ 是否能成为他们的增值服务?

做渠道版 landing:

- 给学校: 新生日本生活文件理解工具。

## 29. 法律科技 PM

背景: LegalZoom / Rocket Lawyer / 合规文档自动化出身,熟悉“信息提供”和“法律建议”的产品边界。

### 一句话判断

TEBIQ 的合规边界不能只靠免责声明,必须体现在问题设计、输出结构和升级路径里。

### 他会盯的核心问题

“风险等级”是否构成用户可依赖判断? “材料清单”是否被理解为完整申请方案? “复杂 case 引导咨询”是否过晚? 用户是否能区分一般信息、个人化整理、专业建议?

### 重构级建议

建立三级输出标签:

- 一般信息: 解释术语和常见流程。
- 个人化整理: 基于用户上传文件整理字段、期限、待确认事项。
- 专业判断: TEBIQ 不提供,只可升级给合作专家。

每个结果页都明确处在哪一级。

### 90 天验证动作

把现有结果页文案做合规重写,去掉容易被理解为判断承诺的表达。找 2 位行政書士和 1 位法律顾问做红线审阅。

### 最尖锐挑战

免责声明是最后一道线,不是第一道产品防线。

## 30. Document Intelligence PM

背景: Adobe Scan / Kofax / Google Lens / 企业 OCR 出身,专注文档解析、版式、图像质量和字段抽取。

### 一句话判断

拍照即懂要从“识别一张图”进化到“理解一组文档”。

### 他会盯的核心问题

日本文书常有多页、背面、附件、缴费票、二维码、印章、表格。用户拍一页可能不完整。系统如何判断缺页、背面、模糊、遮挡、不是内容页?

### 重构级建议

增加文档质量与完整性层:

- 信封检测。
- 模糊检测。
- 背面/多页提示。
- 缴费票与正文分组。
- 自动提示“这份文件可能还缺第 2 页”。

Document 对象支持多图片合并,不要把每张照片当成独立事件。

### 90 天验证动作

收集 50 组多页/不完整样本,训练或规则化质量提示。衡量补拍率和最终字段准确率。

### 最尖锐挑战

用户拍错不是边缘情况,而是文档 AI 产品的默认情况。

## 31. Customer Support PM

背景: Zendesk / Intercom / 高触达客服产品出身,关注自助、人工升级、客服成本和满意度。

### 一句话判断

TEBIQ 的用户焦虑高,所以客服体验会直接影响信任和退款。

### 他会盯的核心问题

AI 看不懂时用户怎么办? 付费后结果不满意怎么办? 咨询和 AI 的责任边界如何解释? 用户会不会把客服当成免费专家咨询?

### 重构级建议

建立客服分层:

- 产品问题: 登录、上传、支付。
- 识别问题: 图像不清、字段错误、补拍。
- 信息边界问题: 告诉用户哪些不能回答。
- 专家升级问题: 引导到付费咨询。

客服模板必须和合规边界一致。

### 90 天验证动作

建立 30 条客服宏和 10 个不可回答模板。上线反馈入口: “识别错了 / 看不懂 / 我需要进一步确认”。衡量问题类型分布。

### 最尖锐挑战

AI 产品的客服不是售后,是信任系统的一部分。

## 32. 高 ARPU 复杂用户 PM

背景: 财富管理、企业主服务、高净值用户产品出身,关注复杂需求、隐私和高客单服务。

### 一句话判断

经管、高度人材、永住、投资/创业用户不应该被塞进 ¥980 普通订阅逻辑。

### 他会盯的核心问题

这些用户上传的不是普通通知,而是公司、税务、投资、家庭、资产和身份规划材料。他们愿意付更多,但要求更高信任、更强隐私、更明确专家背书。

### 重构级建议

做“复杂身份与家庭事务包”假门:

- 经管/高度人材材料室。
- 永住准备档案。
- 家庭成员与资产相关文书归档。
- 专家交接包。

定价不应只是 ¥19,800 年费,而可以是事件包 + 专家升级。

### 90 天验证动作

对经管/高度人材用户访谈 15 人,测试 ¥29,800-¥98,000 事件型服务包的支付意愿,不要只问 ¥980 订阅。

### 最尖锐挑战

TEBIQ 如果只围绕低价订阅设计,会错过最复杂、最愿意付费、也最需要边界设计的用户。

## 33. 学生与早期来日 PM

背景: 留学服务、语言学校 SaaS、年轻用户教育产品出身。

### 一句话判断

刚来日本用户不是续签焦虑,而是“第一年文件爆炸”;这是 TEBIQ 的低价高频入口。

### 他会盯的核心问题

留学生和刚入职者会收到 NHK、国保、年金、住民票、学校、银行、手机、租房文件。首页如果过早强调续签,会错过这一批。

### 重构级建议

新增“刚来日本模式”:

- 我刚来 30 天。
- 我收到第一封区役所信。
- 国保/年金/NHK/银行/租房文件识别。
- 初始档案建立。

这批用户短期付费能力低,但生命周期长,适合年付折扣或学校渠道。

### 90 天验证动作

做“刚来日本 30 天文件清单”内容和入口,投放到小红书/语言学校渠道。衡量首拍率和 30 日留存。

### 最尖锐挑战

续签是高焦虑,但刚来日本是高频;两个入口不能互相替代。

## 34. 经营管理签 PM

背景: 创业者服务、公司设立、跨境电商工具产品出身。

### 一句话判断

经管用户是 TEBIQ 的高价值人群,但不能用普通生活助手的方式服务。

### 他会盯的核心问题

法人税、消费税、社会保险、雇佣、办公室、营业许可、银行、补助金,都和个人生活文书混杂。经管用户需要“公司事项”和“个人在留事项”分离。

### 重构级建议

为经管用户建立双档案:

- 个人档案: 在留、家庭、税、保险。
- 法人档案: 公司通知、税务、雇佣、许可、期限。

AI 不判断事业是否满足基准,只整理材料、期限和缺口。

### 90 天验证动作

做经管用户访谈和假门: “添加公司档案”。测试点击率和愿意上传的文件类型。

### 最尖锐挑战

经管用户不是“多几个问题的续签用户”,而是个人和公司高度耦合的复杂账户。

## 35. 低收入 / 蓝领用户 PM

背景: 特定技能、派遣、工厂/介护员工服务产品出身,关注低收入、高风险、低信任用户。

### 一句话判断

TEBIQ 如果只按 ¥980/月订阅设计,可能会错过特定技能和育成就劳这类高痛点用户。

### 他会盯的核心问题

这些用户收入低、日语弱、公司/支援机构信息不对称、换工作和续签风险高。他们需要极简、可信、可转发给支援者的工具,但付费能力未必稳定。

### 重构级建议

对蓝领用户做:

- 简单模式。
- 一键转发给支援者/朋友。
- 事件包而非月订阅。
- B2B 雇主或支援机构支付。

### 90 天验证动作

和介护/食品工厂/派遣会社访谈,验证 B2B 支付意愿。同时测试 ¥300-¥500 的单次文件包或免费基础 + 机构付费模式。

### 最尖锐挑战

最需要帮助的人,不一定是最容易直接付费的人。

## 36. Retention PM

背景: 消费订阅留存、健身/学习/效率工具出身,专长 churn 诊断和习惯形成。

### 一句话判断

TEBIQ 的留存不能靠日常习惯,只能靠“关键事项状态”和“提前提醒”。

### 他会盯的核心问题

用户取消时会说什么? “最近用不到”。这不是产品不好,而是场景低频。解决方法不是强行日更,而是让用户相信不用打开时 TEBIQ 也在替他守着。

### 重构级建议

订阅内必须有后台价值:

- 期限监控。
- 提醒。
- 档案归档。
- 年度准备路径。
- 家庭共享。

每月账单前发送“本月守护报告”: 识别文件、即将到期事项、已归档资料。

### 90 天验证动作

对订阅用户发送月度摘要邮件/站内报告。观察取消率、二次打开率和提醒设置率。

### 最尖锐挑战

如果用户只有打开时才感到价值,月订阅很容易被取消。

## 37. Unit Economics PM

背景: SaaS 投资人、AI 成本优化、PLG 商业化出身。

### 一句话判断

TEBIQ 必须尽早算清 AI 成本、客服成本、咨询毛利和订阅流失,否则看起来增长,实际亏在每次焦虑交互里。

### 他会盯的核心问题

每次拍照成本多少? 免费用户平均用几次? 付费转化覆盖不了 Bedrock 成本怎么办? 复杂 case 带来的客服成本是否吞掉 ¥980? 咨询分成后毛利多少?

### 重构级建议

建立 unit economics 仪表盘:

- AI 成本 / 拍照
- AI 成本 / 付费用户
- 免费额度消耗到付费转化
- 客服成本 / 复杂 case
- 咨询毛利
- 月流失率

AI 输出要尽量结构化和缓存案例,减少重复解释成本。

### 90 天验证动作

按文书类型统计成本和转化,找出“高成本低转化”场景降级或限额。对复杂场景优先引导整理包/咨询,不要无限免费深聊。

### 最尖锐挑战

AI 工具最容易把用户喜欢误读成商业成立。

## 38. 品牌策略 PM

背景: Consumer brand + product strategy 出身,做过新品牌定位和长期心智建设。

### 一句话判断

TEBIQ 不要占“AI 翻译”心智,要占“在日生活不慌”心智。

### 他会盯的核心问题

品牌名、视觉、文案、功能是否统一指向“お守り”? 用户会怎么向朋友介绍 TEBIQ? 如果他说“一个 AI 翻译”,就输了;如果他说“收到日本文件先拍给它看”,就赢了一半。

统一品牌表达:

- 不是翻译工具。
- 不是行政書士。
- 不是中介。
- 是在日生活文件和事项的守护工具。

所有核心页面围绕“不慌、看懂、知道下一步、有人提醒”写。

### 90 天验证动作

做 20 个用户访谈,让他们用一句话向朋友介绍 TEBIQ。记录自发表达,反向优化首页和付费页。

### 最尖锐挑战

如果用户说不清 TEBIQ 和通用 AI 的区别,增长越大,品牌越弱。

## 39. Policy Monitoring PM

背景: 政策情报、金融监管、企业合规监控产品出身。

### 一句话判断

政策自动抓取是未来壁垒之一,但现在不能先做成信息流,必须和用户事项绑定。

### 他会盯的核心问题

政策更新对谁有影响? 技人国、经管、特定技能、配偶者、永住用户影响不同。单纯抓取政策会变成噪音;真正价值是“这条更新和你的档案有关吗”。

### 重构级建议

政策系统三层:

- 官方来源抓取。
- 人工/AI 结构化成影响标签。
- 与用户 Profile/Case 匹配,只推相关提醒。

1.0 可以半自动,不要追求全自动。

### 90 天验证动作

选 3 类政策源,人工维护更新,测试“与我相关”提醒点击率。不要先做大而全政策库。

### 最尖锐挑战

政策更新本身不值钱,和用户当前身份/事项相关才值钱。

## 40. Real Estate / Moving PM

背景: SUUMO / Zillow / 搬家服务 / 租房平台出身。

### 一句话判断

搬家是 TEBIQ 可扩展到日常生活基础设施的关键场景,因为它天然产生大量文书和期限。

### 他会盯的核心问题

搬家涉及転出届、転入届、国保、年金、驾照、银行、邮便転送、水电煤网、学校/保育园。用户收到的文件密集且焦虑,非常适合 Case 化。

### 重构级建议

做“搬家 Case”:

- 搬家前后时间线。
- 需要处理的机构。
- 拍照归档通知。
- 14 天提醒。
- 相关知识路径。

这会让 TEBIQ 从签证焦虑扩到生活事务。

### 90 天验证动作

做搬家专题页 + 待办模板,看是否能带来非续签场景的拍照和订阅。

### 最尖锐挑战

如果 TEBIQ 只围绕续签,它会很窄;搬家是证明“生活事务守护”的好战场。

## 41. 医疗 / 介护 PM

背景: 医疗预约、介护、家庭照护产品出身。

### 一句话判断

医疗与介护文书是高情绪、高隐私、高复杂度场景,适合做守护,但必须谨慎避开医疗建议。

### 他会盯的核心问题

用户收到的是费用通知、保险、介护认定、医院说明、检查预约、儿童医疗补助。TEBIQ 能解释文书,但不能做诊疗建议。家庭协作在这里非常重要。

### 重构级建议

医疗/介护只做:

- 文件类型解释。
- 期限/金额/机构抽取。
- 需要携带/保存的一般材料。
- 家庭共享提醒。

明确不解释病情、不建议治疗。

### 90 天验证动作

支持 10 类医疗/介护行政文书,结果页加入“这不是医疗建议”。看家庭分享率。

### 最尖锐挑战

医疗场景的产品价值巨大,但一句越界建议就足以摧毁信任。

## 42. Education / Childcare PM

背景: 保育园、学校管理、家长沟通产品出身。

### 一句话判断

有孩子的在日华人家庭会遇到比续签更高频的文书洪水,TEBIQ 可以成为家庭刚需。

### 他会盯的核心问题

保育园、学校、补助金、児童手当、健康检查、疫苗、PTA、入学材料,都需要读懂期限和提交物。家长不一定会为自己付费,但会为孩子不漏事付费。

做“孩子事项”子档案:

- 学校/保育园文件归档。
- 提交期限提醒。
- 家庭共同提醒。
- 常用表单术语解释。

这比“AI 优先服务”更有年付理由。

### 90 天验证动作

上线“孩子/学校文件”标签,观察有孩子用户的保存率、提醒率、家庭分享率。

### 最尖锐挑战

家庭用户的长期价值不在签证,在孩子每天带回来的那一叠纸。

## 43. 消费者支付 PM

背景: Stripe / PayPay / App Store 订阅支付出身。

### 一句话判断

支付暂停会直接阻断商业学习,TEBIQ 必须尽快恢复至少一种真实收款路径。

### 他会盯的核心问题

没有 Stripe,所有付费判断都是意向,不是事实。用户愿意点击和愿意付钱差别很大。¥980、¥9,800、¥19,800 哪个成立,必须靠真实支付数据。

### 重构级建议

支付优先级:

1. 恢复 Stripe 或可用替代。
2. 订阅和一次性包都可售。
3. 支付页清楚解释服务边界。
4. 未付款也记录 paywall intent。

### 90 天验证动作

上线真实支付后跑 3 个价格/权益实验。没有支付前最多只能做假门,不能做商业定论。

### 最尖锐挑战

支付没接上时,团队很容易把“用户说想要”误判成“用户会买”。

## 44. Operations PM

背景: 本地生活、客服运营、专家运营、内容运营复合背景。

### 一句话判断

TEBIQ 未来不是纯代码产品,而是 AI、内容、专家、客服、政策更新共同运转的运营系统。

### 他会盯的核心问题

谁维护知识库? 谁审核高风险文案? 谁处理识别反馈? 谁更新政策? 谁管理专家? 如果没有运营机制,产品越复杂越容易出错。

建立运营后台最小能力:

- 高频问题看板。

- 知识条目更新状态。
- 红色 case 人工复核候选。
- 专家咨询状态。

### 90 天验证动作

即使没有完整后台,也用表格/脚本维护运营队列。每周复盘 Top 错误、Top 文件、Top 咨询原因。

### 最尖锐挑战

AI 不是替代运营,AI 会制造更高要求的运营。

## 45. Competitive Strategy PM

背景: 大厂战略产品、竞品防御、平台生态出身。

### 一句话判断

TEBIQ 最大竞争对手不是另一个 TEBIQ,而是通用 AI + 小红书经验帖 + 行政書士 + 朋友。

### 他会盯的核心问题

用户为什么不用 ChatGPT 拍照? 为什么不问小红书? 为什么不直接找书士? TEBIQ 的差异必须是组合能力: 日文文书识别、个人档案、中文语境、合规克制、提醒、案例和专家升级。

### 重构级建议

每个核心页面都体现“通用 AI 做不到的三件事”:

- 结合我的在日档案。
- 形成待办和提醒。
- 连接相似案例和专家升级。

不要和通用 AI 拼回答长度。

### 90 天验证动作

做用户替代方案访谈: 他们现在用什么解决? ChatGPT、小红书、朋友、书士分别在哪些场景胜过 TEBIQ? 用结果反推差异化。

### 最尖锐挑战

如果 TEBIQ 只是更懂日文的 AI,它会被模型吞掉;如果它是生活事务系统,模型只是底层能力。

## 46. Platform Ecosystem PM

背景: Shopify / Slack / Zapier 平台生态出身,关注开放接口、合作伙伴和生态位。

### 一句话判断

TEBIQ 早期不该开放平台,但要从第一天避免把数据结构做死。

### 他会盯的核心问题

未来可能连接行政書士、税理士、学校、雇主、保险、不动产、支付。Document/Case/Task 如果设计太随意,后面所有合作都会变成一次性定制。

### 重构级建议

为未来生态预留:

- 标准 Case 类型。
- 标准 Task 状态。
- 专家交接包格式。
- 文件导出格式。
- 用户授权模型。

现在不开放 API,但内部 schema 要像未来会开放一样严谨。

### 90 天验证动作

定义 v0 数据字典和导出格式。先让专家咨询、B2B 雇主、家庭共享三个内部场景共用同一套对象。

### 最尖锐挑战

早期偷懒的数据模型,会在你终于找到商业模式时变成最大负债。

## 47. Product Marketing PM

背景: PLG 产品营销、SaaS 定位、落地页转化出身。

### 一句话判断

TEBIQ 的市场话术要从“功能清楚”升级到“场景尖锐”。

### 他会盯的核心问题

“拍照看懂日文文件,续签前先做风险自查”清楚,但还可以更强。不同场景需要不同落地页: 收到区役所信、续签前 90 天、公司让交证明、孩子学校通知、经管税务通知。

### 重构级建议

建立场景落地页矩阵:

- 收到日本政府信,先拍照看懂。
- 续签前 90 天,先确认材料缺口。
- 公司让你交 課税証明。
- 孩子学校通知看不懂。
- 经营管理签老板的公司文书。

每个落地页都连接一个案例和一个行动入口。

### 90 天验证动作

做 10 个场景页,配合自媒体内容投放。看哪类场景带来最高首拍率和付费率。

### 最尖锐挑战

不要只卖 TEBIQ 是什么,要卖“你手里这张纸现在该怎么办”。

## 48. Product-Led Sales PM

背景: PLG 转销售、SMB SaaS、销售辅助产品出身。

### 一句话判断

TEBIQ 的 B 端机会不需要先建销售团队,可以从产品内信号触发轻销售。

### 他会盯的核心问题

哪些用户来自同一家公司? 哪些文书和雇主有关? 哪些用户多次分享给 HR? 哪些 case 显示企业有批量需求? 这些都是 B2B 线索。

### 重构级建议

在不侵犯隐私的前提下捕捉 B2B 信号:

- 用户可填写公司邀请码。
- 分享给 HR 的摘要模板。
- 企业试点页。
- 匿名聚合需求报告。

### 90 天验证动作

做“公司福利版”landing 和 waitlist。个人用户结果页可选择“想让公司提供 TEBIQ”。收集企业线索。

### 最尖锐挑战

B2B 不一定从销售开始,也可以从 C 端用户不断把问题带给公司开始。

## 49. 0->1 创始型 PM

背景: 连续创业、早期产品战略、PMF 搜索出身。

### 一句话判断

TEBIQ 最大危险不是方向错,而是同时看到太多正确方向后过早发散。

### 他会盯的核心问题

续签、税务、孩子、搬家、B2B、专家、家庭、政策、经管、医疗都对,但早期不能全做。PMF 需要一个尖到能穿透的 wedge。当前最像 wedge 的是“日文生活文件 -> 事项/提醒/档案”。

### 重构级建议

接下来 90 天只追一个主命题:

“用户拍一份重要日本文件后,是否愿意把这件事交给 TEBIQ 持续跟进?”

围绕这个命题做 Case、Task、Reminder、Paywall。其他都先做成标签和内容,不要做重功能。

### 90 天验证动作

定义 PMF 信号:

- 30% 以上拍照结果保存为事项。
- 25% 以上事项设置提醒。
- 15% 以上免费用户触发付费页。
- 5-8% 免费用户付费。
- 订阅用户 30 日留存超过 50%。

### 最尖锐挑战

不要把路线图当战略;战略是决定接下来 90 天不做什么。

## 50. 董事会级产品战略 PM

背景: 大型平台 GM、投资人转产品、跨 C 端/B 端/AI/服务型业务操盘经验。

### 一句话判断

TEBIQ 的终局不是 App,而是一个围绕在日外国人生活制度摩擦建立的数据、工作流和服务网络。

### 他会盯的核心问题

这个市场是否足够大? C 端订阅是否扛得住 churn? B 端是否能补稳定收入? AI 成本是否可控? 合规边界是否能成为壁垒而不是限制? 团队是否能运营内容、专家、政策和客服?

### 重构级建议

战略分三阶段:

1. Wedge: 中文用户的日本文书理解与事项守护。
2. System: 档案、提醒、案例、专家升级、家庭协作。
3. Network: 雇主、学校、专家、保险/不动产/税务合作。

每一阶段都要建立不可逆资产: 文件类型库、案例库、用户档案、事项状态、专家交接格式、B2B 渠道。

### 90 天验证动作

不追求功能数量,只验证两条曲线:

- C 端: 从拍照到事项保存到提醒到付费。
- B 端: 从企业访谈到试点合同到员工激活。

### 最尖锐挑战

TEBIQ 如果只做 AI 工具,是功能生意;如果掌握用户生活事务状态,才可能是基础设施生意。

# 跨 PM 综合

## 50 位 PM 共同指出的核心问题（出现 5 次以上）

### 1. 产品仍过度停留在“解释器”,没有完成“事务系统”跃迁

出现频次: 39 / 50

共同判断: 拍照即懂是强入口,但解释文本不是长期价值。TEBIQ 必须把文件转成 Case,把 Case 转成 Task,再通过 Reminder 和 Profile 形成持续关系。

### 2. 订阅价值锚点偏 AI 使用权,不够像生活守护

出现频次: 31 / 50

共同判断: 用户不太会长期为“无限拍照/文字”续费,但可能为期限提醒、档案归档、家人协作、年度准备和重要事项守护续费。

### 3. 主对象模型缺失会限制所有后续方向

出现频次: 28 / 50

共同判断: 无论是 AI schema、专家交接、B2B、家庭共享、案例库、政策匹配还是生命周期运营,都需要 Document / Case / Task / Profile 这套底层结构。

### 4. 合规与信任必须产品化,不能只靠免责声明和 prompt

出现频次: 26 / 50

共同判断: 用户上传的是在留、税务、收入、家庭、医疗等敏感文件。TEBIQ 必须让用户在上传前、结果中、档案页、咨询入口都感到边界清楚、数据可控。

### 5. C 端月订阅可能低频,需要事件包、家庭版、B2B 和专家升级共同支撑

出现频次: 24 / 50

共同判断: 单靠 ¥980/月有 churn 风险。更稳的商业结构是 C 端订阅 + 事件型材料/准备包 + 专家升级 + 雇主/学校/机构 B2B2C。

### 6. 内容库不应只是 SEO 文章,应升级成案例库和路径系统

出现频次: 23 / 50

共同判断: 用户不是搜索抽象知识,而是搜索“我收到这张纸怎么办”。高价值内容形态是样例文书、匿名案例、术语卡、生活能力路径和结果页内嵌知识。

### 7. 用户分层过粗,在日中文用户内部差异巨大

出现频次: 21 / 50

共同判断: 刚来日本、续签前、经管老板、特定技能、家庭用户、高度人材、低日语用户、学生、永住/归化用户不是同一种产品体验。

### 8. 触达体系薄弱会削弱“守护”承诺

出现频次: 18 / 50

共同判断: 如果没有可靠邮件、LINE、SMS 或其他提醒渠道,任务清单和守护定位都无法闭环。提醒不是附加功能,是订阅价值核心。

### 9. AI 质量需要评测集、置信度和降级机制

出现频次: 17 / 50

共同判断: TEBIQ 的 AI 风险不在“不会回答”,而在“看似回答得很好但越界或误导”。必须建立测试集、字段置信度、red-flag、补拍和不确定降级。

### 10. 支付与 unit economics 需要尽快真实验证

出现频次: 12 / 50

共同判断: 支付暂停会让商业判断停留在意向层。AI 成本、免费额度、订阅转化、咨询毛利和客服成本必须用真实交易验证。

## 50 位 PM 共同认可的核心优势

### 1. 需求真实且高焦虑

日本生活文书天然有语言、制度、期限和后果压力。用户不是为了好奇使用,而是因为手里真的有一张看不懂但可能重要的纸。

### 2. 中文母语者定位足够尖

1.0 不服务非中文用户是正确取舍。它让文案、渠道、术语、内容、案例和信任建设可以做深。

### 3. “拍照即懂”是极强 aha moment

用户无需理解复杂产品结构,只要拍一张日本文件就能立刻感受到价值。这是非常好的 0->1 wedge。

### 4. 续签自查是高价值引流入口

续签自查能捕捉身份焦虑、收集档案、连接材料、触发拍照和订阅,比普通内容页更接近商业闭环。

### 5. 合规意识领先于多数 AI 工具

不输出法律建议、不做行政指导、不预测审查时间、不滥用“书士审核”背书,这些克制是未来信任资产。

### 6. 品牌定位有情感空间

“在日生活のお守り”比“AI 翻译工具”更有长期心智。它允许 TEBIQ 从解释扩展到守护、提醒、档案、家庭和专家升级。

### 7. 多条商业化路径都能从同一个核心对象模型长出来

C 端订阅、家庭版、事件包、专家咨询、B2B 雇主、学校渠道、案例库和政策提醒,都可以围绕 Document / Case / Task 复用,不是完全分散的业务。

## 50 位 PM 中最差异化的洞察（只 1-2 位 PM 提,但角度刁钻）

### 1. “不保存,仅本次识别”可能比强制建档更能建立信任

来自: 隐私安全 PM、跨境本地化 PM。

刁钻点: 大多数产品想尽快沉淀数据,但敏感文件场景下,允许用户不保存反而可能提高首次上传率和后续信任。

### 2. 蓝领/弱势用户最需要 TEBIQ,但最不适合直接月订阅变现

来自: 低收入/蓝领用户 PM、弱势用户 PM。

刁钻点: 高痛点不等于高付费。这个人群可能需要 B2B 雇主、支援机构、学校或事件包来承接商业化。

### 3. 医疗、孩子学校、搬家可能比续签更能证明“生活守护”的长期价值

来自: 医疗/介护 PM、Education/Childcare PM、Real Estate/Moving PM。

刁钻点: 续签是强焦虑,但低频。孩子、医疗、搬家文书更能制造家庭长期使用场景。

### 4. 专家咨询如果不回流到 Case,TEBIQ 只是在卖线索

来自: 专家网络 PM、Marketplace PM。

刁钻点: 很多团队会把咨询当高 ARPU 外挂收入,但真正的系统价值是专家输出变成用户 Case 的下一步任务和长期档案。

### 5. 政策自动抓取本身不值钱,和用户档案匹配才值钱

来自: Policy Monitoring PM。

刁钻点: 做政策库很容易显得高级,但用户不需要更多政策新闻,只需要知道“这条和我有关吗,我有没有事要做”。

## 如果只能听 50 位 PM 中 1 位的建议,哪个最值得听?为什么?

最值得听: **工作流工具 PM**。

原因: 他提出的 Document / Case / Task / Profile 对象模型,是其他 49 个建议能够成立的地基。

- 增长飞轮需要 Task 和 Reminder 才能复访。
- 订阅价值需要 Case 和 Profile 才能从“AI 次数”变成“生活守护”。
- AI 评测需要 Document schema 才能可验证。
- 合规边界需要把输出限定在字段、事项和待确认点里。
- 专家咨询需要 Case 交接包。
- B2B 雇主需要匿名 Case 聚合,不能碰个人原文。
- 家庭共享需要围绕 Task 和权限,不能直接共享敏感文件。
- 内容案例库需要从 Document 类型和 Case 类型反哺。
- 政策提醒需要 Profile 和 Case 匹配。
- 生命周期运营需要在留期限、税务季、搬家、孩子等 Case 时间线。

如果只听增长 PM,可能会优化漏斗但没有长期资产。如果只听订阅 PM,可能会重写权益但底层体验支撑不足。如果只听 AI PM,会提高回答质量但仍可能停留在解释器。如果只听商业模型 PM,会过早分散到 B2B、专家、家庭和事件包。

工作流工具 PM 的建议最值得听,因为它不是一个功能建议,而是把 TEBIQ 从“拍照理解页面”重构成“在日生活事务系统”的底层架构。

## 50 位 PM 综合后的重构路线

### P0: 定义并实现最小对象模型

- Document: 文件与解析字段。
- Case: 事项。
- Task: 下一步动作。
- Profile: 用户/家庭/身份档案。
- Reminder: 触达与完成闭环。

### P1: 拍照结果页从解释页变成行动页

结果页第一屏只回答:

- 这是什么。
- 急不急。
- 和我有没有关系。
- 下一步一般动作。

随后才展示字段、知识、案例、保存、提醒和专家升级。

### P2: 续签自查从风险报告变成准备系统

把“风险等级”弱化为“确认事项”:

- 已确认。
- 待确认。
- 缺材料。
- 待拍文件。
- 关键期限。
- 不适合 AI 判断的点。

### P3: 订阅从 AI 次数重写为生活守护

付费权益围绕:

- 重要文件守护。
- 期限提醒。
- 历史档案。
- 续签/税务/搬家准备。
- 家庭共享。
- 复杂情况升级。

### P4: AI schema、评测集和 red-flag 降级

建立结构化输出、字段置信度、红色场景、补拍机制、合规 flag 和回归测试集。

### P5: 打通一个可靠触达渠道

邮件、LINE 或 SMS 至少要有一个真正可用。没有提醒触达,TEBIQ 不能兑现“守护”。

### P6: 内容升级为案例库 + 生活路径

优先做:

- Top 50 高频文书案例。
- 续签前 90 天路径。
- 刚来日本 30 天路径。
- 税务/孩子/搬家专题路径。

### P7: 商业化做三条轻验证

- C 端订阅: 个人守护。
- 事件包: 续签/搬家/税务季。
- B2B 试点: 雇主/学校/派遣会社邀请码。

## 一句话终局判断

TEBIQ 不应把自己做成“更懂日本文件的 AI”,而应做成“在日中文用户的生活事务操作系统”。AI 是入口,文件是对象,事项是主线,提醒是留存,档案是壁垒,专家和 B2B 是商业放大器。
