# TEBIQ 续签自查形态对比：问卷式 vs 清单式

研究对象：`/check` 续签自查从“13-18 题必答问卷”改为“清单自选 / 混合入口”的产品形态取舍。

核心判断来自创始人原话：

> 每次进来都让用户做一遍题目挺傻的，有点强迫，但肯定有不少人需要做题。做题是增益功能，不是必选。

本文用 30 个差异化画像做深度模拟。结论先写在前面：**1.0 建议做形态 C，但不是 A/B 平权的混合；应做“清单为默认底座 + 完整自查为增益入口”的 C-lean-B 形态。**

## 1. 三种形态详细定义

### 形态 A：当前问卷式（基准）

用户路径：

1. 用户进入 `/check`
2. 选择签证类型
3. 进入 13-18 题必答问卷
4. 输出风险等级、风险点、材料清单

用户感受：

- 优点：确定、完整、像一次正式体检；焦虑用户愿意被强引导。
- 缺点：任何入口都变成“考试”；重复进入时，用户会觉得系统没有记忆。
- 本质：适合“一次性完整评估”，不适合“日常维护型自查”。

### 形态 B：清单自选式

用户路径：

1. 用户进入 `/check`
2. 选择签证类型
3. 看到该签证的风险维度清单，约 10-15 项
4. 用户点开感兴趣项，每项展开 1-3 个具体问题
5. 单项给出判断，并自动沉淀到“我的提醒 / 档案时间线”

示例清单：

- 工作变更：去年有换工作吗
- 住民税缴纳：按时交了吗
- 社保 / 年金：有没有断缴
- 出入国记录：离日天数是否影响续签 / 永住
- 课税证明：能不能开出完整年度
- 违规记录：罚金、严重交通违规、入管警告
- 收入水平：是否低于常见风险线
- 雇用契约：正社员、契约、派遣、试用期、业务委托
- 护照 / 在留卡：有效期与基础材料
- 届出义务：搬家、换工作、身份关系变化是否 14 日内处理

用户感受：

- 优点：轻、可重复、可以只查变化项；更像工具，不像考试。
- 缺点：用户可能漏选自己“不知道重要”的维度。
- 本质：适合日常维护、探索、复查、档案沉淀。

### 形态 C：混合式

用户路径：

1. 用户进入 `/check`
2. 选择签证类型
3. 默认看到清单
4. 顶部提供“完整自查（约 5 分钟）”入口
5. 用户可以走完整问卷，也可以点单项清单

关键定义：

- 不建议做成“A 和 B 两个同级大按钮”。
- 建议做成“清单是默认页面，完整自查是顶部增益操作”。
- 完整自查跑完后，应把每个维度的状态回写到清单，而不是只生成一个孤立结果页。

用户感受：

- 优点：焦虑用户可以完整扫，轻量用户可以只点一项；档案用户可以只看变化。
- 缺点：首屏信息层级更难做；若设计不当，会让用户犹豫“我到底该点哪个”。
- 本质：覆盖面最广，但要求信息架构更克制。

## 2. 30 画像逐个对比反馈

### 画像 01：技人国，续签剩 2 个月，刚收到入管不明信件

画像背景：

他在东京 IT 派遣公司工作，日语能读但不敢判断入管信件。上次续签顺利，所以这次突然收到信件后高度紧张。目标不是学习知识，而是想知道“我是不是有事”。

形态 A（问卷式）体验：

- 第一感受：像正式体检，虽然有压力，但可信。
- 完成意愿（1-10）：9
- 完成后满意度（1-10）：8
- 重复使用意愿（3个月后会再做吗）：低，除非又收到信。
- 关键摩擦点：他想先判断信件，但系统先问一整套续签问题。

形态 B（清单式）体验：

- 第一感受：可以先点“入管信件 / 届出 / 材料补正”，很快进入问题。
- 点开清单项数（10-15项中点几项）：4 项
- 单项体验满意度（1-10）：8
- 重复使用意愿：中，信件处理后会回来查税和工作。
- 关键摩擦点：他不确定不点“违规记录”会不会错过大事。
- 担心：会，焦虑时反而不知道该点哪些。

形态 C（混合式）体验：

- 第一感受：看到“完整自查”会松一口气，清单又能先处理信件。
- 实际选择哪条路（A完整/B清单）：先 B 查信件，再 A 完整。
- 选择理由：先解决眼前不明物，再做完整确认。
- 满意度：9

这个画像的最优形态：C。因为他既需要即时入口，也需要最终确定感。

### 画像 02：技人国，刚换工作，担心 14 日届出漏办

画像背景：

她从大阪转职到东京，HR 没有提醒入管届出。朋友说“换工作要报入管”，她开始焦虑。她知道自己的问题很具体。

形态 A（问卷式）体验：

- 第一感受：前几题正好问换工作，觉得命中。
- 完成意愿（1-10）：8
- 完成后满意度（1-10）：8
- 重复使用意愿（3个月后会再做吗）：中低，除非下次换工作。
- 关键摩擦点：后面税、护照、违规题对她当下像噪音。

形态 B（清单式）体验：

- 第一感受：直接点“工作变更 / 届出义务”，路径短。
- 点开清单项数：3 项
- 单项体验满意度（1-10）：9
- 重复使用意愿：高，之后会查住民税和雇用契约。
- 关键摩擦点：如果“工作变更”没有标高风险，她可能误以为只是普通事项。
- 担心：中等，知道自己要查什么，但不知道税与契约也相关。

形态 C（混合式）体验：

- 第一感受：清单直接解决问题，顶部完整自查像兜底。
- 实际选择哪条路：先 B 清单。
- 选择理由：问题明确，不想答完全部。
- 满意度：9

这个画像的最优形态：C。因为清单先解决高意图问题，完整问卷保留为兜底。

### 画像 03：永住申请准备者，担心住民税和年金记录

画像背景：

他来日 9 年半，准备永住，过去一年换过住址，也有一次住民税晚缴。他不是单纯续签，而是在长期资格稳定性上找确定感。

形态 A（问卷式）体验：

- 第一感受：完整，但“续签”题感不完全贴合永住准备。
- 完成意愿（1-10）：8
- 完成后满意度（1-10）：7
- 重复使用意愿：中，会因为永住资料周期回来。
- 关键摩擦点：一次问卷无法表达“过去 3 年 / 5 年 / 10 年”的持续性。

形态 B（清单式）体验：

- 第一感受：看到“税、年金、出入国、课税证明”几个维度，觉得可控。
- 点开清单项数：8 项
- 单项体验满意度（1-10）：8
- 重复使用意愿：高，适合每月补资料。
- 关键摩擦点：如果没有“永住准备”专属权重，他会担心续签清单太轻。
- 担心：中高，永住高风险项太多，不想凭自己选择。

形态 C（混合式）体验：

- 第一感受：既能全扫，也能反复维护重点维度。
- 实际选择哪条路：先 A 完整，再长期用 B 清单。
- 选择理由：第一次需要基线，之后只更新变化。
- 满意度：9

这个画像的最优形态：C。因为永住准备本质是长期档案维护，不是一次问卷。

### 画像 04：被行政书士提醒过交通罚单，续签剩 3 个月

画像背景：

她过去一年有多次交通违章，其中一次罚金性质不确定。书士说“要看具体记录”。她最怕漏报、误报。

形态 A（问卷式）体验：

- 第一感受：被强制问到违规记录，反而觉得安全。
- 完成意愿（1-10）：9
- 完成后满意度（1-10）：9
- 重复使用意愿：低，违规记录不会频繁变化。
- 关键摩擦点：问题要足够细，否则“轻微交通违规”与“罚金”分不清。

形态 B（清单式）体验：

- 第一感受：能直接点“违规记录”，效率高。
- 点开清单项数：2 项
- 单项体验满意度（1-10）：8
- 重复使用意愿：低。
- 关键摩擦点：她可能只点交通，不点入管警告 / 行政处分。
- 担心：高，违规是低频高损失项，漏选后果重。

形态 C（混合式）体验：

- 第一感受：会犹豫要不要完整自查，但最后会选完整。
- 实际选择哪条路：A 完整。
- 选择理由：她不信自己能判断“哪些该查”。
- 满意度：8

这个画像的最优形态：A。因为低频高损失事项必须被系统主动问到。

### 画像 05：経営・管理签老板，遇到新规和资本金问题

画像背景：

他 2025 年前后拿到经管签，公司规模小，听说资本金、常勤职员、日语能力基准变严。风险维度多，且很多项要公司资料配合。

形态 A（问卷式）体验：

- 第一感受：能快速知道红黄绿，但 15 题像压缩版尽调。
- 完成意愿（1-10）：8
- 完成后满意度（1-10）：7
- 重复使用意愿：中，公司经营状况会变化。
- 关键摩擦点：一口气答完后，很难分配给税理士、HR、自己逐项处理。

形态 B（清单式）体验：

- 第一感受：风险维度展开后很适合项目管理。
- 点开清单项数：10 项
- 单项体验满意度（1-10）：9
- 重复使用意愿：高，会按月看经营、税、办公室、雇员。
- 关键摩擦点：首次进入时项目太多，容易不知道先看哪项。
- 担心：中，若有“新规必看”标记可下降。

形态 C（混合式）体验：

- 第一感受：完整自查建立基线，清单适合后续追踪。
- 实际选择哪条路：先 A 完整，再 B 清单维护。
- 选择理由：经管签不是单点风险，是一组经营合规指标。
- 满意度：10

这个画像的最优形态：C。因为经管签需要“首次扫描 + 长期维度管理”。

## 3. 综合分析

### 3.1 三种形态各自的胜出场景

形态 A 最适合：

- 用户处在高焦虑、高风险、高损失场景：上次不许可、收到入管信、书士警告、违规记录、即将递交。
- 用户不知道自己该查什么，需要系统强制覆盖盲区。
- 产品目标是“生成一次完整风险等级 + 材料清单”，而不是沉淀长期档案。

形态 A 的问题：

- 它把所有用户都当成“现在就要正式体检的人”。
- 重复进入时会浪费已有信息，破坏 TEBIQ 档案中心化方向。
- 对探索用户和单项任务用户太重。

形态 B 最适合：

- 探索用户：来日 1-3 年、朋友推荐、无具体焦虑，只想轻量了解。
- 单项任务用户：刚搬家、刚换工作、收到税信、护照快到期。
- 长期维护用户：永住准备、家庭多签证、经管签资料维护、HR 分发。

形态 B 的问题：

- 它让用户自己决定检查范围，而很多关键风险正是用户不知道的。
- 低频高损失项容易被忽略，例如违规记录、入管警告、课税证明开不出、派遣 / 业务委托形态。
- 如果没有风险预警和快速扫描，B 会变成“看起来轻，实际漏”的产品。

形态 C 是否能两头讨好：

- 能，但前提是 C 不是两个入口平铺。
- 如果首屏是“完整自查”和“清单自查”两个同级按钮，用户会开始思考“我该选哪个”，首屏复杂度会伤害轻量用户。
- 正确的 C 应该是：默认清单页面；顶部有一个明确但不压迫的“完整自查”入口；系统根据档案和风险标记推荐用户是否需要完整扫。
- 也就是说，C 的产品定义不是“A+B”，而是“B 底座 + A 增益”。

### 3.2 30 画像投票

投票结果：

- A 最优：2 人
- B 最优：12 人
- C 最优：16 人

分组观察：

- 焦虑性用户 10 人：A 2，C 8，B 0。
- 探索性用户 10 人：B 9，C 1，A 0。
- 重度信息用户 10 人：B 3，C 7，A 0。

覆盖最广：

- C 覆盖最广，但必须采用“清单默认 + 完整自查增益”的设计。
- B 是实际使用频率最高的底座，因为大部分重复访问都不是重做完整问卷。
- A 仍然不可删除，因为高焦虑 / 高风险用户会把它当成确定感来源。

### 3.3 关键洞察：B 的漏选风险

形态 B 的漏选风险真实存在，而且不是小问题。它集中出现在三类事项：

1. 低频高损失：罚金、刑事记录、严重交通违规、入管警告、不法滞在。
2. 用户无感但入管敏感：课税证明开不出、住民票脱漏、换工作 14 日届出、社保断缴。
3. 用户误以为正常但规则变化：派遣形态、业务委托、经管签新资本金 / 常勤职员要求、特定技能 1 号通算 5 年。

真实损失场景：

- 用户没点“违规记录”，因为他以为“只是交通罚单”；实际属于罚金记录，递交时未说明，后续被要求补正或影响许可年限。
- 用户没点“课税证明”，因为他已经补缴住民税；实际某年度没有完整申告，市役所开不出证明，递交前才发现。
- 用户没点“雇用契约”，因为他觉得自己一直在工作；实际是业务委托，不符合技人国雇用稳定性预期。
- 用户没点“出入国记录”，因为每次回国都不超过 3 个月；实际一年累计过长，影响永住连续居住判断。

防止漏选的设计：

- 风险预警标记：给低点击但高损失项打“建议必看”，例如违规记录、课税证明、届出义务、雇用形态。
- 快速扫描模式：进入清单后先用 30 秒过一遍全部标题，每项只回答“无 / 有 / 不确定”。用户点“不确定”才展开 1-3 个问题。
- 档案驱动推荐：如果档案里有换工作、搬家、文件识别到督促状、签证到期 90 天内，相关清单项自动置顶。
- 签证类型必查项：不同签证给 3-5 个“本类型必查”维度，不能完全依赖用户兴趣。
- 提交前完整扫提示：当用户只点了少于 4 项且到期小于 90 天，提示“递交前建议完整自查一次”。

风险预警标记需要做吗：

- 需要。
- 但不要做成红色恐吓。建议使用克制文案：“建议必看”“递交前确认”“档案缺失”。
- 标记逻辑按“损失严重度 x 用户自觉概率”计算：高损失且低自觉的项优先标。

快速扫描模式需要做吗：

- 需要，且是 B/C 能成立的关键。
- 30 秒快速扫描不是完整问卷；它只是帮助系统识别用户应该点哪些清单项。
- 推荐形式：一屏一项标题，三个按钮“无 / 有 / 不确定”，扫完后生成“建议查看 4 项”。

形态 C 的设计陷阱：

- 陷阱 1：两个大按钮平铺，用户不知道完整和清单哪个更权威。
- 陷阱 2：完整自查太显眼，所有人都点 A，清单失去意义。
- 陷阱 3：清单太轻，所有人都图省事点 B，递交前漏项。
- 陷阱 4：完整自查结果与清单状态不互通，用户做了 A 后仍看到 B 全部未查。

规避方式：

- 默认进入清单。
- 顶部显示一个小但明确的完整自查入口：“完整自查：约 5 分钟，适合续签前 / 高风险情况”。
- 清单项有状态：“未查 / 已查 / 需处理 / 3 个月内已查 / 已过期”。
- 完整自查完成后，自动把每个维度写回清单状态。

### 3.4 TEBIQ 档案积累的影响

档案积累是 B/C 强于 A 的核心理由。

重复使用时：

- A 形态：用户已经有档案，仍要重做 13-18 题；这等于告诉用户“TEBIQ 不记得你”。
- B 形态：每个清单项可以显示“上次检查：2026-04-12”“3 个月内有效”“因换工作建议重查”。
- C 形态：首次完整自查生成基线，后续只提醒变化项；到递交前再建议完整扫一次。

这与 TEBIQ 产品哲学一致：

- TEBIQ 不是工具集合，而是档案中心化工具系统。
- 自查不应该只产出一次结果页；它应该产出可追踪的风险维度状态。
- 清单天然适合沉淀：每个维度就是一个档案节点。

因此，A 可以保留为“完整体检”，但不能继续作为所有用户每次进入的唯一入口。

### 3.5 跨签证类型差异

技人国：

- 当前 18 题覆盖工作变更、税、社保、违规、收入、护照、课税证明、住民票、雇用契约。
- 对首次正式自查，A 仍然合理。
- 但对换工作、派遣、课税证明等重复维护，B/C 更好。

経営・管理：

- 维度更多且更像企业合规：资本金、常勤职员、日语能力、经营经验、事业计划、办公室、法人税、个人税、经营状态。
- A 会显得像压缩尽调，结果页不够承载后续处理。
- B/C 优势明显，应以清单和状态管理为主。

永住 / 永住准备：

- 本质是长期维护：纳税、年金、出入国、连续居住、收入、证明材料、违规记录。
- A 的一次性测评价值有限。
- B 最适合日常维护；C 适合“年度完整扫描 + 平时清单维护”。

配偶者 / 永住者配偶者：

- 风险围绕婚姻实质、同居、收入、税保、出入国、证明材料。
- 单次问卷能做基础判断，但关系证据需要长期积累。
- 推荐 C：清单维护证据，续签前完整扫。

特定技能：

- 有明确硬项：1 号通算 5 年、支援机构、技能测试、工作范围、雇用形态。
- 用户可能日语弱、规则理解弱，B 要配“建议必看”。
- 推荐 C：高风险项清单置顶，续签前完整扫。

定住者：

- 生活基础、税保、出境、违规、课税证明、住民票、雇用稳定性都重要。
- 日常维护和递交前完整检查都需要。
- 推荐 C。

是否不同签证类型需要不同形态：

- 需要不同默认权重，不需要完全不同产品形态。
- 技人国：完整自查入口可以更强。
- 经管签 / 永住准备：清单状态管理更强。
- 特定技能：必查项和低语言负担更强。
- 配偶 / 定住：证据维护和关系 / 生活基础维度更强。

## 4. 推荐方案

### 4.1 1.0 用哪种形态

推荐：**形态 C，但做成“清单默认 + 完整自查增益”**。

不要继续把 A 当默认唯一入口。原因：

- 它对焦虑用户有效，但会损失探索用户和重复用户。
- 它不符合档案中心化方向。
- 它把“做题”从增益功能变成了强制门槛。

也不建议纯 B。原因：

- B 会漏掉用户不知道的重要风险。
- 对递交前、高风险、上次不许可、违规记录等场景，B 的确定感不足。

推荐的 1.0 首屏：

- 顶部：签证类型 + 最近自查状态
- 主操作：风险维度清单
- 次操作：完整自查（约 5 分钟）
- 系统提示：根据签证类型和档案，标出“建议必看”项

首屏文案建议：

- 页面标题：续签自查
- 副标题：按风险项逐个确认；续签前可做一次完整自查
- 完整入口：完整自查（约 5 分钟）
- 清单状态：未查 / 已查 / 需处理 / 3 个月内已查

### 4.2 清单具体怎么设计

每个签证类型 10-15 个维度。每个维度由四层组成：

1. 标题：一句话，让用户知道这是什么。
2. 状态：未查 / 已查 / 需处理 / 不确定 / 已过期。
3. 轻问题：1-3 个问题，只判断该维度。
4. 结果：该项风险、处理建议、是否加入提醒。

通用维度建议：

- 工作变更 / 所属机构届出
- 住民税
- 所得税 / 确定申告
- 健康保险 / 年金
- 课税证明 / 纳税证明
- 出入国记录 / 连续居住
- 违规记录 / 罚金 / 入管警告
- 收入水平
- 雇用契约 / 派遣 / 业务委托 / 试用期
- 护照 / 在留卡有效期
- 住民票 / 搬家记录
- 材料准备

经管签追加：

- 资本金
- 常勤职员
- 办公室实体
- 事业计划书
- 法人税
- 经营流水 / 赤字 / 现金流
- 新规过渡期

配偶者追加：

- 婚姻关系存续
- 同居 / 别居理由
- 配偶收入与扶养
- 关系证明材料

特定技能追加：

- 1 号通算年限
- 支援机构
- 技能测试有效性
- 工作范围一致性

### 4.3 风险预警怎么做

清单项需要三类标记：

- 建议必看：低频高损失项，例如违规记录、届出义务、课税证明、雇用形态。
- 档案触发：系统从档案判断该项可能相关，例如最近识别到督促信、换工作、搬家、到期 90 天内。
- 已过期：上次检查超过有效期，例如税务 / 保险 3 个月，出入国 1 个月，材料准备 2 周。

预警排序建议：

1. 到期 90 天内 + 未查必看项
2. 档案触发项
3. 上次结果为黄 / 红的项
4. 签证类型基础必查项
5. 其他普通项

不要用强情绪文案。TEBIQ 的 voice 应该克制：

- 用“建议必看”，不用“千万别漏”
- 用“递交前确认”，不用“危险”
- 用“档案缺失”，不用“你还没做”

### 4.4 数据库 schema 怎么改

当前 `quiz_results` 适合存“整次问卷结果”，但不适合存“用户点过哪些清单项、每项什么时候查过、是否过期、是否加入提醒”。

建议保留 `quiz_results`，并新增维度级表。

最小可行 schema：

```sql
check_dimension_results
- id
- member_id nullable
- session_id nullable
- visa_type
- dimension_key
- dimension_version
- source_mode enum('checklist', 'full_quiz', 'quick_scan')
- answers jsonb
- result_color enum('green', 'yellow', 'red')
- triggered_items jsonb
- status enum('checked', 'needs_action', 'unknown', 'dismissed')
- checked_at
- expires_at
- created_at
- updated_at
```

事件表：

```sql
check_dimension_events
- id
- member_id nullable
- session_id nullable
- visa_type
- dimension_key
- event_type enum('viewed', 'opened', 'answered', 'completed', 'skipped', 'dismissed')
- metadata jsonb
- created_at
```

可选运行表：

```sql
check_runs
- id
- member_id nullable
- session_id nullable
- visa_type
- mode enum('full_quiz', 'checklist', 'quick_scan')
- result_color nullable
- summary jsonb
- started_at
- completed_at nullable
```

关系：

- 完整问卷完成后，写 `check_runs`，同时把每个维度拆写进 `check_dimension_results`。
- 清单单项完成后，只写对应 `check_dimension_results`。
- 快速扫描只写 `check_dimension_events` 和推荐项，除非用户完成具体问题。
- `my/reminders` 可读取 `needs_action` 和 `expires_at` 生成提醒。

索引建议：

- `(member_id, visa_type, dimension_key)`
- `(session_id, visa_type, dimension_key)`
- `(member_id, expires_at)`
- `(member_id, status)`

### 4.5 改造工作量估算

小：

- 只把 `/check/[visa]` 前面加一个静态清单页，点击清单仍跳现有问卷局部题。
- 不做持久化，只用 sessionStorage。
- 能验证交互方向，但不符合档案中心化。

中：

- 新增维度清单 UI。
- 将现有题库映射为 dimension。
- 新增 `check_dimension_results` 持久化。
- 完整问卷结果回写维度状态。
- 我的提醒读取需处理项。

大：

- 加快速扫描模式。
- 档案自动触发预警。
- 多成员 / 家庭 / HR 权限。
- 永住准备、经管签长期合规看板。

推荐 1.0 工作量：中。

原因：

- 只做小会变成皮肤改造，没有解决“重复使用浪费已有信息”。
- 直接做大会拖慢上线。
- 中等方案能验证核心假设：清单是否提升进入率、重复率、单项完成率，同时保留完整自查的确定感。

### 4.6 1.0 后演进路径

第一阶段：C-lean-B 基础版

- 清单默认。
- 完整自查入口保留。
- 维度结果持久化。
- 完整问卷回写清单。

第二阶段：快速扫描

- 30 秒扫全部标题。
- 生成建议查看项。
- 统计低点击高风险项。

第三阶段：档案触发

- 文件识别到督促信、入管信、税单后，自动点亮相关清单项。
- 用户档案里有到期日、换工作、搬家、上次自查红黄项时，自动置顶。

第四阶段：签证类型深水区

- 经管签合规看板。
- 永住准备长期维护。
- 家庭多签证管理。
- HR 分发但隐私隔离。

关键指标：

- `/check` 进入后清单项打开率
- 完整自查启动率
- 完整自查完成率
- 单项完成率
- 7 天 / 30 天重复访问率
- 已查维度数量
- 高风险必查项漏查率
- 清单项转提醒率

## 5. 给创始人 1 句话

续签自查不该是一张每次重填的考卷，而应该是一组会记住用户状态的风险清单；完整问卷要保留，但只作为递交前和高焦虑场景的增益扫描。
