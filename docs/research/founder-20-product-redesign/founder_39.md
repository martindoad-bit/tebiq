# founder_39 河合美穗｜LegalTech 合规产品顾问，消费者保护/个人信息

## 1. 付费模式判断

**选择：选项 1。** 选 1。不是因为当前模式最好，而是因为选 2/3 都需要更成熟的隐私与同意设计。先不要扩大档案保存和搜索入口风险。

**¥980 的消费心理：** ¥980/月在日本必须清楚写自动更新、取消、保存数据、删除权。否则价格越低越像不透明服务。

**我做过的订阅里，触发付费的最强信号：** 最强信号不是付费，而是 consent_given_for_archive。TEBIQ 先要确认用户真的理解保存什么。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** 法律/行政问题自由搜索容易诱导个案咨询。使用率不重要，风险更重要。

**如果不是搜索，应该是什么：** 用结构化分诊问题，不要开放式搜索作为唯一入口。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文之后的任何语言都必须同步隐私/条款/免责声明。日语 UI 给非日语用户不满足 informed consent。

**“日语 UI 给非日语用户用”是否可行：** 不可作为主 UI。用户同意必须用可理解语言。

## 4. 拍照定位降级判断

**判断：** 拍照降级可以，但上传前的同意要提升。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 单独也涉及敏感信息，不能只看商业价值。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动能撑订阅，但更强地处理个人信息。

**我担心场景驱动的最大风险：** 最大风险是 purpose creep：为一个场景收的信息被另一个场景使用。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做隐私权限矩阵：每类文书/场景保存哪些字段、保存多久、用于哪些提醒，用户可逐项删除。

## 创始人最该警惕的 1 件事

“档案”这个词在商业上好听，但在法律上意味着持续责任。
