# founder_43 青木莉子｜前 Yappli 客户成功/移动会员 PM

## 1. 付费模式判断

**选择：选项 3。** 选 3。移动订阅最重要是第二周留存。搜索限制让用户第一天就烦，进度/提醒让用户第二周回来。

**¥980 的消费心理：** ¥980/月是 mobile utility 可以接受的上限，需要强通知价值。

**我做过的订阅里，触发付费的最强信号：** 最强信号是 push/email permission。没有通知权限，提醒权益不存在。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** App 内搜索通常不是主入口，tab 和固定菜单更高频。

**如果不是搜索，应该是什么：** 首页固定“进行中 / 新建场景 / 拍照添加”三入口。

## 3. 多语言节奏判断

**判断：基本同意，但要分清 UI、源信息和协作者界面。** 中文后做日语 UI 有利于日本 app store 和合作伙伴，但用户操作仍需中文。英语/越南语分阶段。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 可以作为外层品牌，但核心解释要母语。

## 4. 拍照定位降级判断

**判断：** 拍照降级没问题，但应作为 tab 常驻。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 不是订阅价值。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动更能做 push 生命周期。

**我担心场景驱动的最大风险：** 风险是通知过度和权限被拒。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会围绕 option 3 做 notification onboarding：用户创建第一个场景时选择邮件提醒，解释 D-30/D-7/D-1。

## 创始人最该警惕的 1 件事

没有通知授权，提醒会员就是空权益。
