# founder_09 罗骁｜留学生租房/兼职平台创始人，前校园增长 PM

## 1. 付费模式判断

**选择：选项 2。** 选 2。留学生/新人用户会把免费工具当无限问答，用量高但付费低。搜索 3 次/天能控制成本，超过再付 ¥980，转化可能 2%。

**¥980 的消费心理：** ¥980 对留学生等于两顿学食或一个月部分手机小套餐，心理压力明显。除非临近签证/租房纠纷，否则不会持续月付。

**我做过的订阅里，触发付费的最强信号：** 最强信号是用户连续两天使用，并上传契约书或学校通知。TEBIQ 应抓 day2_retention + document_upload。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** 学生产品里搜索使用率 20% 左右，更多是点“我收到学校通知/我要搬家/我要打工”。日语手续词搜索能力低。

**如果不是搜索，应该是什么：** 首页用任务卡，不用搜索框替代 CTA。搜索可以服务高阶用户。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文之后我会先做英语留学生版，因为学校生态里英语更通用。越南语可通过专门学校合作，不要自己硬上。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 给留学生不现实。刚来日本的人看日语 UI 会直接关掉。

## 4. 拍照定位降级判断

**判断：** 拍照不能降太深，学生收到学校/区役所纸张时第一反应就是拍。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 单独对学生有传播价值，但他们不会为 OCR 订阅。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动更好，但留学生生命周期短，订阅要按“90 天场景包”设计。

**我担心场景驱动的最大风险：** 风险是做太多低支付人群场景，消耗 AI 成本和客服。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做“新人来日 30 天”和“留学打工资格外活动”两个场景，免费但强引导注册，先验证次日回访。

## 创始人最该警惕的 1 件事

不要用留学生数据误判全盘价格，他们会用，但未必付。
