# founder_12 山本翔｜前 LINE 官方账号/通知产品 PM

## 1. 付费模式判断

**选择：选项 3。** 选 3。日本市场愿意为“通知不会漏”付费，尤其是期限类。LINE 经验里，订阅触发不是阅读次数，而是用户设置第一个 reminder。

**¥980 的消费心理：** ¥980/月比 LINE Sticker Premium 贵很多，用户会把它当业务工具看。只要提醒能避免一次迟延，它就合理；如果只是搜索，就太贵。

**我做过的订阅里，触发付费的最强信号：** 最强信号是 reminder_created。TEBIQ 应让每个场景自然产生一个期限，创建提醒后提示订阅可邮件/LINE 通知。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** LINE 官方账号菜单点击远高于搜索输入，菜单按钮 CTR 常是 25-40%，自由输入只有个位数到十几。

**如果不是搜索，应该是什么：** 首页用“场景菜单 + 最近提醒”。搜索放顶部，但不要替代 CTA。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文之后，我会做“日语通知模板”而不是日语 UI。越南语如果要做，通知渠道应优先 LINE，因为在日越南用户也常用 LINE。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 给非日语用户不行，但日语通知标题可以帮助他们在邮箱/LINE 里识别官方词。

## 4. 拍照定位降级判断

**判断：** 拍照可以降级。通知产品的核心是事件和时间，不是输入方式。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 单独是 feature，不是 product。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动更适合订阅，因为每个场景都有节点和通知。知识库没有自然通知点。

**我担心场景驱动的最大风险：** 风险是提醒太多变成噪音。日本用户对通知频率很敏感，默认只发 D-30/D-7/D-1。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会把 20 个场景统一抽出 deadline schema，每个场景最多 3 个提醒点，并先接邮件，LINE 通知后置。

## 创始人最该警惕的 1 件事

别把“我的提醒”做成时间线墓地，必须是可执行的通知系统。
