# founder_23 金莎拉｜Meta Japan Creator Monetization PM，社群付费与创作者工具

## 1. 付费模式判断

**选择：选项 2。** 选 2。如果 TEBIQ 依赖社交流量，清楚的免费额度更容易传播：“每天 3 个问题免费”。创作者类产品免费到付费通常 1-3%，quota trigger 解释成本低。

**¥980 的消费心理：** ¥980/月接近一个小型创作者订阅或流媒体附加包。用户 7 天内看不到反复价值就会取消。

**我做过的订阅里，触发付费的最强信号：** 最强信号是分享后回访。用户把场景发给配偶/朋友并回来，付费概率高。TEBIQ 应记录 shared_scenario_opened。

## 2. 搜索作为主入口的判断

**判断：同意，但必须被场景化约束。** 社交流量带着视频里的语言进来。如果 landing query 被预填，搜索可以工作；空白搜索弱很多。

**如果不是搜索，应该是什么：** 如果 campaign 深链能预填搜索，我支持搜索；否则首页还是场景卡。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文第一。下一语言取决于获客伙伴：有越南社区就做越南语试点，有外资渠道就做英语。不要把日语 UI 当 1.1 C 端。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 会伤害社交流量转化。用户从母语内容点进来，就期待母语安全感。

## 4. 拍照定位降级判断

**判断：** 拍照不应从营销里消失。“拍一下”是强视觉 hook，但产品内可以降级。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 是 social hook，不是付费核心。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动更好，尤其每个场景都能生成可分享摘要。

**我担心场景驱动的最大风险：** 风险是用户截图场景页当法律建议传播。要显眼写“仅整理/确认，不做结论”。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做可分享的场景结果卡：3 件需确认、2 份要保存、1 个期限，不写法律结论。

## 创始人最该警惕的 1 件事

拍照如果藏太深，会丢掉短视频最强 demo 素材。
