# founder_28 加西亚玛丽亚｜Uber Eats Japan PM，低频高压支持体验

## 1. 付费模式判断

**选择：选项 2。** 选 2。支持型产品如果完全免费，容易被滥用。每日额度清楚，但紧急页面不能被挡。付费转化约 2%。

**¥980 的消费心理：** ¥980 对 gig worker 偏高。只有问题威胁收入或身份时才会付费。

**我做过的订阅里，触发付费的最强信号：** 最强信号是账户/收入影响。对应 TEBIQ，就是雇主文书、税金滞纳通知、在留期限。要检测 severity 并分流。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** 配送员 app 里用户很少搜索，他们点问题分类。搜索通常发生在已经很烦的时候。

**如果不是搜索，应该是什么：** 用 guided categories 和 urgency tags，搜索做辅助。

## 3. 多语言节奏判断

**判断：拒绝这个节奏。** 拒绝日语 UI 过渡。非日语劳动者要母语优先：越南语、印尼语、菲律宾语，但前提是有客服/支持。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 对低日语劳动者会失败。不要靠 icon 解决，直接用简单母语任务标签。

## 4. 拍照定位降级判断

**判断：** 拍照要一键可达，因为劳动者常有文件；但不应是唯一入口。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 不够，他们需要下一步安全行动。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动更强，因为支持问题天然是 issue type。

**我担心场景驱动的最大风险：** 最大风险是给出伪法律/劳动建议。严重情况要导向官方窗口。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做 urgency router：7 天内期限、有人要钱、雇主变化、家庭状态变化，四类先分流。

## 创始人最该警惕的 1 件事

如果没有支持运营就扩到劳动者语言，会制造伤害和客服债。
