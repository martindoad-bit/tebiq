# founder_47 吉田圭｜前 Yahoo News / SmartNews 编辑产品经理

## 1. 付费模式判断

**选择：选项 2。** 选 2。信息产品免费太多会造成无目的浏览和成本，搜索 3 次/天能让用户珍惜问题。但深层提醒也应是会员权益。

**¥980 的消费心理：** ¥980/月对新闻太高，对“政策影响我的提醒”可以。付费文案必须远离“读更多”。

**我做过的订阅里，触发付费的最强信号：** 最强信号是 user says “relevant to me”。TEBIQ 应让每个政策/场景有“这和我有关”按钮。

## 2. 搜索作为主入口的判断

**判断：同意，但必须被场景化约束。** 新闻 app 搜索低，热门入口高；但政策查询有明确问题时搜索可用。

**如果不是搜索，应该是什么：** 搜索作为主入口可以，但要配热点场景和编辑精选，不能空框。

## 3. 多语言节奏判断

**判断：基本同意，但要分清 UI、源信息和协作者界面。** 中文后做日语 UI 我支持，因为政策原文和日本公共信息更新是日语。英语/越南语必须有编辑质量。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 给非日语用户不充分，但日语原文链接和术语必须保留。

## 4. 拍照定位降级判断

**判断：** 拍照降级正确，政策/场景问题常先于文书。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 不是媒体价值。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动比知识库强，但知识库提供来源可信度。

**我担心场景驱动的最大风险：** 风险是政策更新滞后。过期信息会毁信任。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会建立“场景更新时间/来源”机制，每个场景顶部显示最后确认日期和官方链接。

## 创始人最该警惕的 1 件事

场景页不是一次写完的内容，它是需要维护的新闻产品。
