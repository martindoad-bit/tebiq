# founder_25 阮丹尼尔｜Amazon Japan Logistics PM，外国人仓储员工流程

## 1. 付费模式判断

**选择：选项 3。** 选 3。低收入外国员工不会为更多搜索付费，但当期限或雇主文书影响工作时会付。个人 C 端转化可能 1-2%，通过雇主更高。

**¥980 的消费心理：** ¥980 对很多员工偏高，接近几顿工作餐。它必须像“避免丢掉工作时间/在留资格风险”，不是信息费。

**我做过的订阅里，触发付费的最强信号：** 最强信号是上传雇主相关文件：雇用契約書、シフト、支援計画。TEBIQ 应识别 work_doc_type 并路由到场景。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** 仓储员工在帮助 app 里很少搜索，多数点 issue category，搜索使用率低于 10%。

**如果不是搜索，应该是什么：** 用“工作上发生什么了”分类入口，搜索做辅助。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文第一适合当前用户；如果扩张，我会在有社区/雇主渠道时优先越南语，而不是英语。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 对日语弱的员工不可行。应该是简单母语标签 + 日文官方术语。

## 4. 拍照定位降级判断

**判断：** 拍照必须浅层保留，因为员工有大量纸质合同和通知；但照片应进入场景。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 不够，员工需要知道“我该向雇主确认什么”。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动更好，工作问题有步骤和期限。

**我担心场景驱动的最大风险：** 风险是劳动/法律边界。不要指导争议，严重情况导向 労基署 或专家。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做“雇用条件变更”和“特定技能转职/受入機関变化”两个场景，保守边界并支持雇主文书上传。

## 创始人最该警惕的 1 件事

扩张语言需要社区分发，不是 app store 本地化。
