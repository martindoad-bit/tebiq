# founder_38 许佳｜签证材料收集 SaaS 创始人，前行政書士事务所运营

## 1. 付费模式判断

**选择：选项 3。** 选 3。材料收集系统的付费点是“我已经交了哪些、还缺哪些”，不是搜索。¥980/月对即将续签用户可接受，平时不行。

**¥980 的消费心理：** 用户把 ¥980 看成“比找书士便宜的整理费”。如果没有清单进度，就像卖文章。

**我做过的订阅里，触发付费的最强信号：** 最强信号是上传两份以上材料。TEBIQ 应在第二份材料后提示“建立本次场景档案”。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** 事务所客户不会搜索法律词，只会发微信问。搜索框不是主入口。

**如果不是搜索，应该是什么：** 用 intake 表单：你发生了什么、什么时候、现在的在留資格。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文先。日语 UI 给事务所后台，英语/越南语需要对应事务所合作。

**“日语 UI 给非日语用户用”是否可行：** C 端日语 UI 不可行；后台日语很有用。

## 4. 拍照定位降级判断

**判断：** 拍照不能太深，因为材料收集从上传开始。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 的价值是自动归类材料，而不是解释。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动更适合订阅，因为它形成材料缺口。

**我担心场景驱动的最大风险：** 风险是用户以为缺口清单等于申请可通过。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做“材料箱”：每个场景下面有需要上传的 slots，上传后只显示已识别/待确认/不适用。

## 创始人最该警惕的 1 件事

别用 AI 写太多建议。把材料归类做准，比建议多一句更值钱。
