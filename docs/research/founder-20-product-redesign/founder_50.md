# founder_50 黑田健｜外籍人士保险/金融服务创始人，前 InsurTech PM

## 1. 付费模式判断

**选择：选项 3。** 选 3。保险/金融里，免费诊断获客，付费在长期保障和记录。TEBIQ 的 ¥980 应卖“手续保障感”，不是卖搜索。预计个人 3%，渠道合作更高。

**¥980 的消费心理：** ¥980/月比保险附加服务便宜，但用户会要求“出事时有用”。最好有年付 ¥8,800 或和咨询抵扣。

**我做过的订阅里，触发付费的最强信号：** 最强信号是用户有 dependents：配偶、孩子、父母、雇员。涉及他人的手续更愿意付费。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** 保险产品里搜索低，诊断问答高。用户不知道该问什么。

**如果不是搜索，应该是什么：** 首页做“发生了什么”诊断，不做空白搜索。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文后，我会先做英语金融/保险友好版，再做越南语渠道版。日语 UI 给合作机构。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 对非日语用户不够，但对保险/不动产/HR 渠道必要。

## 4. 拍照定位降级判断

**判断：** 拍照降级合理，但文件上传是理赔/手续的自然行为，应保留。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 单独无法建立保障心智。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动更能撑订阅，因为它能把“未来风险”具体化。

**我担心场景驱动的最大风险：** 风险是把保障感做成虚假安全感。必须写清不能保证结果。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做“家庭/雇员影响标签”：每个场景标出影响自己、家人、雇主还是机构，然后优先推多人高价值场景。

## 创始人最该警惕的 1 件事

TEBIQ 若想长期赚钱，要识别“涉及他人”的场景，那些才有高付费和高焦虑。
