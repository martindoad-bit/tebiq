# founder_08 方敏｜名古屋中小企业后台 SaaS 创始人，前会计软件 PM

## 1. 付费模式判断

**选择：选项 3。** 选 3，但对经管用户我会加年付 ¥12,000 的法人档案。¥980/月对个人还行，对公司太便宜，反而不可信。日本小公司为会计/労務工具月付 ¥2,000-5,000 很正常。

**¥980 的消费心理：** ¥980 对经管签用户不是价格问题，是“你配不配处理我公司文件”的信任问题。比便利店午饭贵一点，但比税理士一通电话便宜太多。

**我做过的订阅里，触发付费的最强信号：** 最强信号是用户上传法人文件：登記簿、賃貸借契約書、決算書、法人税通知。TEBIQ 应把 business_doc_uploaded 当高价值线索。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** 后台 SaaS 里搜索很少当主入口，使用率 10-18%。老板用户更喜欢“公司搬迁、役员报酬、税金、雇佣”这种任务菜单。

**如果不是搜索，应该是什么：** 首页应有“个人手续 / 公司手续 / 家庭手续”三分流。经管用户看到“拍照”会觉得太轻，看到“公司变更事项”才会信。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文打穿后，日语 UI 应先给合伙人税理士/行政书士，不是给 C 端。英语版可以针对创业者，越南语不应碰经管。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 给中文老板可作为正式感，但解释仍要中文。

## 4. 拍照定位降级判断

**判断：** 拍照降级正确。经管场景中拍照只是收集证据，核心是公司事件时间线。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 单独没有商业价值，经管用户愿意为“公司手续清单 + 专家接入”付费。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动更容易撑高价订阅，因为公司事项跨机构。知识库只能带来咨询线索。

**我担心场景驱动的最大风险：** 经管场景最容易越界成“经营管理签证判断”。必须明确只是档案整理和期限提醒。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做“経営・管理 公司搬迁”路径，要求用户上传租约、登記、照片、税务通知，输出只做清单和期限，不做可否判断。

## 创始人最该警惕的 1 件事

如果继续用 ¥980 个人订阅承接经管用户，会把高价值用户做低。
