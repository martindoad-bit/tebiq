# founder_40 李正元｜税务/签证咨询 marketplace 创始人，前税理士法人产品化负责人

## 1. 付费模式判断

**选择：选项 3。** 选 3。免费浅层可以筛选意图，深层追踪和专家 handoff 收费。咨询 marketplace 里，保存资料的人咨询转化是普通浏览的 4 倍。

**¥980 的消费心理：** ¥980/月对咨询前整理很便宜，但必须能抵扣或衔接 ¥9,800 咨询，否则用户觉得重复付费。

**我做过的订阅里，触发付费的最强信号：** 最强信号是 expert_intent：用户点“何时找专家”、上传复杂文件、保存超过 3 项材料。

## 2. 搜索作为主入口的判断

**判断：同意，但必须被场景化约束。** 用户会搜索“要不要找专家”，搜索能捕捉需求。但结果要做分诊，不要直接答法律结论。

**如果不是搜索，应该是什么：** 搜索可以主入口，但高风险词触发专家分诊。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文后我会做日语专家端，再做英语 C 端。越南语需要越南语专家网络。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 给专家端可行，给非日语用户不够。

## 4. 拍照定位降级判断

**判断：** 拍照降级合理，但专家咨询前材料上传必须强。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 价值是咨询前预整理，可降低专家接待成本。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动最适合把低价订阅和高价咨询串起来。

**我担心场景驱动的最大风险：** 风险是订阅和咨询边界混乱。用户会问“我都订阅了为什么还要咨询”。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会设计“咨询前档案包”：订阅用户可一键生成给专家的材料摘要，¥9,800 咨询可抵扣一个月会员。

## 创始人最该警惕的 1 件事

订阅和专家咨询不要互相蚕食，要让订阅降低咨询准备成本。
