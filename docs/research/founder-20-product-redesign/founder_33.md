# founder_33 莎莉｜在日印尼穆斯林生活服务创始人，社区运营 9 年

## 1. 付费模式判断

**选择：选项 3。** 选 3。印尼社区用户愿意用免费工具，但付费常来自家庭/工作紧急场景。限制搜索会让他们回到 WhatsApp 群。

**¥980 的消费心理：** ¥980 对很多印尼用户偏高，必须给按场景短期付费或家庭共享。月付转化可能 1%，家庭/工作事件可高些。

**我做过的订阅里，触发付费的最强信号：** 最强信号是用户上传家庭或工作文件，并询问是否影响在留。TEBIQ 应捕捉 family_or_work_doc + deadline。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** 社区里用户不习惯搜索行政词，他们发照片问管理员。

**如果不是搜索，应该是什么：** 首页应该是“我收到文件 / 我换工作 / 我搬家 / 家人来日”四个简单入口。

## 3. 多语言节奏判断

**判断：拒绝这个节奏。** 如果服务印尼人，必须印尼语。日语 UI 完全不适合。英语也不能覆盖蓝领群体。

**“日语 UI 给非日语用户用”是否可行：** 不可行。日语 UI 会让用户觉得这不是给他的。

## 4. 拍照定位降级判断

**判断：** 拍照不要降级太多。照片是社区求助的原生行为。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 单独不能收费，但可以让用户从 WhatsApp 群迁移到产品。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动更好，因为社区管理员也是按场景答。

**我担心场景驱动的最大风险：** 风险是跨宗教/家庭/工作细节复杂，模板容易不适配。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做“收到日本政府信件”印尼语极简入口：拍照后只输出类型、期限、是否需要找窗口/专家。

## 创始人最该警惕的 1 件事

非中文扩张不要从复杂家庭签证开始，从文件识别 + 保守路径开始。
