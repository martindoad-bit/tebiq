# founder_36 张律｜行政書士法人数字化合伙人，签证案件 15 年

## 1. 付费模式判断

**选择：选项 1。** 选 1。合规视角下，新增搜索主入口和深功能订阅会增加“用户以为你在判断案情”的风险。先维持现模式，把场景页做成材料收集，不急着改付费。

**¥980 的消费心理：** ¥980/月比一次行政書士咨询便宜太多，用户会误以为能替代专家。价格低不等于风险低。

**我做过的订阅里，触发付费的最强信号：** 最强信号是复杂事实：离婚、拒签、滞纳、刑事、雇主纠纷。TEBIQ 应捕捉 high_risk_fact，并只导向专家/官方窗口。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** 法律咨询里搜索框很危险，用户会问“我能不能”。普通用户不会区分事实整理和法律判断。

**如果不是搜索，应该是什么：** 首页应是“场景分诊”，高风险直接降级输出，不开放自由搜索直答。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文后不要先日语 C 端。日语可给行政書士后台。越南语/英语必须有相应专家审核。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 给非日语用户不解决法律理解，还会增加误读。

## 4. 拍照定位降级判断

**判断：** 拍照降级可以，但拍照结果必须按文书风险分类。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 单独有用但不应承诺处理复杂案。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动更能收费，但也更接近个案指导。必须把输出限制为材料、期限、官方窗口。

**我担心场景驱动的最大风险：** 最大风险是离婚/配偶/经管/特定技能转职场景越界。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会先做“红线分类器”：20 个场景每个标注 safe/gray/red，red 场景只给事实归档和专家入口。

## 创始人最该警惕的 1 件事

场景页不是越详细越好，越详细越可能像行政指导。
