# founder_22 奈尔皮莉娅｜Amazon Japan Seller Experience PM，中小卖家流程产品

## 1. 付费模式判断

**选择：选项 3。** 选 3。Amazon 卖家不会为帮助文章付费，他们会为避免账户被卡的工具付费。TEBIQ 应该收“进行中场景追踪”的钱，不收搜索的钱；保存期限后的转化可期待 3-5%。

**¥980 的消费心理：** ¥980 如果解释为“避免漏掉行政期限”可以接受；如果只是内容订阅就不成立。

**我做过的订阅里，触发付费的最强信号：** 最强信号是业务/身份中断风险：账户审核、在留期限、缺材料。TEBIQ 应捕捉 deadline_added 和 missing_material_count。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** 帮助中心搜索有人用，但当用户缺少词汇时 deflection 很差。复杂操作任务里，guided issue selector 明显胜过搜索。

**如果不是搜索，应该是什么：** 用“发生了什么变化”选择器：办公室、工作、家庭、住所、收入、收到文书。搜索可在上方，但不要替代选择器。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文第一没问题。接着我会做英语专业人士/经营者版。日语 UI 用于 official source 对照和 partner workflow。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 给看不懂日语的人，不是本地化策略。

## 4. 拍照定位降级判断

**判断：** 拍照降级正确，因为用户经常在收到文书前就有问题。触发点是“发生了一件事”。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 本身价值低，issue resolution 才有价值。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景流更适合订阅，因为能展示进度和缺失项。

**我担心场景驱动的最大风险：** 风险是承诺过度。产品应说“ready to confirm”，不要说“safe”。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做一个 3 问 intake：发生什么、什么时候、涉及谁，然后路由到 20 个场景。

## 创始人最该警惕的 1 件事

不要让知识库成为产品表层。它应该是路径背后的证据。
