# founder_34 阿里夫｜印尼工程师转职平台创始人，前招聘产品经理

## 1. 付费模式判断

**选择：选项 2。** 选 2。工程师群体能接受 quota，像 API/AI 工具。免费 3 次搜索足够试，¥980 解锁全部。预计高技能外籍转化 3-5%。

**¥980 的消费心理：** ¥980 对工程师不高，低于一次咖啡聊天成本。但他们会和 ChatGPT Plus 比，所以要有日本手续数据库和档案。

**我做过的订阅里，触发付费的最强信号：** 最强信号是 job_change_selected + visa_type_entered。转职和签证关系强。

## 2. 搜索作为主入口的判断

**判断：同意，但必须被场景化约束。** 工程师会搜索，但会用英语/母语混合问。搜索框可以作为入口。

**如果不是搜索，应该是什么：** 支持搜索，但要允许英文/中文/印尼语混输，并返回日文术语。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文后我会做英语，再考虑印尼语。日语 UI 仅用于官方术语。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 对高技能外籍可勉强使用，但不会提升转化。

## 4. 拍照定位降级判断

**判断：** 拍照降级合理，工程师更多是先问“换工作对签证有什么手续”。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 单独价值低。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动和搜索结合最好，转职场景有明确 steps。

**我担心场景驱动的最大风险：** 风险是给出“可否转职”判断，必须避免。
