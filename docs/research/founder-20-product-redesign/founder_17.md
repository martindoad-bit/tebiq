# founder_17 小林健｜前 Recruit 住まい/HR 产品经理

## 1. 付费模式判断

**选择：选项 3。** 选 3。Recruit 的经验是生活事件比搜索更能转化。用户不是搜索“賃貸借契約 解約”，而是点“我要搬家”。保存进度和提醒付费可到 4%。

**¥980 的消费心理：** ¥980/月像生活事件 support fee。若只用一次，用户会希望 ¥500 一次性；若跨 30-90 天，月付成立。

**我做过的订阅里，触发付费的最强信号：** 最强信号是用户填写 event_date。搬家日、离职日、离婚日、在留期限都代表真实需求。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** 房产/招聘搜索高，因为物件/职位是可比较对象；行政手续不是。手续产品搜索使用率估计 15-25%。

**如果不是搜索，应该是什么：** 首页应是“生活事件卡片”，每张卡内再有搜索。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文后我会先做英语，目标是高技能外国人和外资员工。日语 UI 是给日本协作者，不是用户主界面。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 不适合作为非日语用户过渡。日本市场很重视“わかりやすい”，不是“日语就行”。

## 4. 拍照定位降级判断

**判断：** 拍照降级对获客有损，但对长期价值正确。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 是工具，不是 marketplace。单独产品很难超过免费翻译。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动更能撑订阅，因为每个生活事件自然跨多个步骤。

**我担心场景驱动的最大风险：** 风险是场景维护成本高，自治体/公司制度差异会让模板过期。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会挑 10 个“有日期”的场景，不做没有 deadline 的知识场景。每个场景第一步都是输入日期。

## 创始人最该警惕的 1 件事

没有日期的场景很难变成订阅。
