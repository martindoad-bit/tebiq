# founder_21 陈迈克｜Google Japan Maps/Search PM，东京本地搜索产品

## 1. 付费模式判断

**选择：选项 2。** 选 2。如果搜索成为核心界面，按次数限制最清楚。日本本地搜索里，高意图用户本来就是少数；每天 3 次免费能覆盖轻用户，同时控制 AI 成本，付费转化现实预估 2-4%。

**¥980 的消费心理：** ¥980/月不是小额消费，而是工具订阅。用户会和 Google One、Netflix、一次便利店午饭比较。它需要给 unlimited 的安心感，而不是“可能有用”。

**我做过的订阅里，触发付费的最强信号：** 最强信号是同一意图被反复改写。TEBIQ 应把“带母亲来日、父母签证、介护”聚成同一个 intent，并在第二次追问时提示保存为场景。

## 2. 搜索作为主入口的判断

**判断：同意，但必须被场景化约束。** 本地搜索能成立，是因为用户知道地点类别。政府手续是低词汇领域；没有 suggest 的裸搜索输入率会低，有 autocomplete 和场景 chips 才可能到 30-40%。

**如果不是搜索，应该是什么：** 我支持搜索捕捉意图，但它必须是 answer routing，不是自由 AI 聊天。生成答案前先展示匹配场景。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文第一正确。C 端扩张我会先做英语，因为英语专业人士付费能力高。日语应作为 source label 和官方术语层。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 给非日语用户不是好桥梁。双语 source snippet 有用，完整日语界面没用。

## 4. 拍照定位降级判断

**判断：** 拍照应成为场景里的输入方式。OCR 会被手机系统和 LLM commoditize。

**“图片识别 + OCR”作为单独产品的商业价值：** 单独 OCR 能获客，不能留存。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景流更能支撑订阅，因为它会生成保存状态；知识库不会。

**我担心场景驱动的最大风险：** 最大风险是错误路由。匹配错场景比不回答更糟。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会建立 50 个真实 query 的 intent taxonomy，映射到 20 个场景页，并给“不确定”场景保守兜底。

## 创始人最该警惕的 1 件事

搜索质量就是产品质量。没有 intent taxonomy 的搜索框只是风险。
