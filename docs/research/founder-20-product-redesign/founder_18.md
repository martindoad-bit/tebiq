# founder_18 渡边真央｜前 NTT docomo 会员/通知产品 PM

## 1. 付费模式判断

**选择：选项 3。** 选 3。日本用户会为长期保存和通知付费，尤其是年龄稍大的人。月付转化可能 3%，年付若 ¥6,800-8,800 更稳。

**¥980 的消费心理：** ¥980/月在日本通信服务里是一个 optional pack 的价格。必须让用户觉得取消简单，且不是被偷偷续费。

**我做过的订阅里，触发付费的最强信号：** 最强信号是 notification_opt_in。愿意收邮件/LINE 提醒的人才可能付费。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** docomo 老用户很少主动搜索，菜单和账单入口更重要。搜索框主入口会丢高龄用户。

**如果不是搜索，应该是什么：** 首页应是“下次要做什么”，搜索作为辅助。按钮文案要具体，不要让用户输入。

## 3. 多语言节奏判断

**判断：基本同意，但要分清 UI、源信息和协作者界面。** 中文后做日语 UI 我赞成一半，因为日本生活工具最终要有日语标准界面；但每个语言都要有母语解释。英语和越南语节奏不要同时上。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 给非日语用户只适合已在日多年用户，新人不行。

## 4. 拍照定位降级判断

**判断：** 拍照降级可以，但底部 tab 保留拍照很好。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 单独没有通信会员级别商业价值。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动比知识库适合订阅，因为提醒是会员权益。

**我担心场景驱动的最大风险：** 风险是复杂 UI。高龄/弱势用户需要 2-3 个明确动作，不要 20 个筛选器。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会重做未登录首页为 3 个大事件入口 + 最近文书示例，不放空白搜索框给弱用户。

## 创始人最该警惕的 1 件事

不要只为年轻产品经理设计。手续焦虑最高的人常常不是最会搜索的人。
