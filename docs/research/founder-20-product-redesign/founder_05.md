# founder_05 蔡宇航｜福冈跨境汇款/税务工具创始人，前金融产品经理

## 1. 付费模式判断

**选择：选项 2。** 我选 2，少数派。金融工具里完全免费会吸引大量低意图查询，AI 成本不可控。搜索 3 次/天足够，超过说明是强需求。类似工具 3 次后付费转化 2.5-4%。

**¥980 的消费心理：** ¥980/月在日本金融工具里是“认真使用”的价格，不是娱乐订阅。用户会和 MoneyForward ME 的付费心理比较：是否能持续管理钱和期限。

**我做过的订阅里，触发付费的最强信号：** 最强信号是用户连续问同一主题 3 次，或上传税/收入/缴费文件。TEBIQ 应捕捉 topic_repeat_count 和 sensitive_doc_uploaded。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** 金融 FAQ 的搜索框使用率可到 35%，但高质量搜索来自已经知道术语的人。普通用户仍要靠分类入口。

**如果不是搜索，应该是什么：** 可以有搜索，但默认先让用户选“税、年金、签证、住所、家庭、工作”。搜索次数限制应只限制 AI 生成，不限制浏览官方来源摘要。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文后我会先做英语，因为金融/税务高收入外国人更可能付费。越南语支付能力和支持成本不匹配，除非有合作伙伴。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 给非日语用户不解决理解问题，只能作为文书术语对照。

## 4. 拍照定位降级判断

**判断：** 拍照不能完全降级。税单、納税証明書、源泉徴収票是付费触发器，首页至少保留一个次级“拍文书”入口。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 单独在金融里有价值，但价值上限是收据扫描，不是 ¥980/月。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动适合订阅，但要防止 AI 生成过多导致成本吞掉毛利。知识库浏览应免费，深层个性化追踪收费。

**我担心场景驱动的最大风险：** 最大风险是免费无限搜索带来成本和合规双高。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做成本仪表盘：每个场景的 AI token 成本、搜索次数、保存率、订阅率。没有 unit economics 不要放开搜索。

## 创始人最该警惕的 1 件事

“浅功能全免费”只在成本可控时成立。Bedrock 成本不该靠信仰解决。
