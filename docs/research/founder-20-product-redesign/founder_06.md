# founder_06 唐晓琳｜东京母婴/教育社区创始人，前社区产品 PM

## 1. 付费模式判断

**选择：选项 3。** 选 3。家庭用户不怕 ¥980，怕“我买了以后还是要自己想”。我做保育园材料服务时，免费文章转付费 1-2%，但保存孩子资料、期限提醒后到 6%。

**¥980 的消费心理：** ¥980/月对家庭用户接近孩子一节线上课的零头，但必须让妈妈觉得它能减少一次错过締切的风险。

**我做过的订阅里，触发付费的最强信号：** 最强信号是用户输入家庭成员信息：孩子年龄、配偶状态、父母是否在日。TEBIQ 应把 family_member_added 作为高意图事件。

## 2. 搜索作为主入口的判断

**判断：同意，但必须被场景化约束。** 妈妈社区搜索使用率较高，约 40%，但搜索词很生活化：“孩子上小学要什么”。所以搜索框可以做入口。

**如果不是搜索，应该是什么：** 我支持搜索，但必须返回“场景路径”，不是文章列表。搜索“母亲养老”应直接进入“父母长期在日/介护/探亲”判断页。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文第一。第二步不要日语 C 端，先做繁体中文和“日文打印版清单”，因为家庭场景常要给日本配偶或学校看。

**“日语 UI 给非日语用户用”是否可行：** 纯日语 UI 对中文妈妈帮助有限；日文清单给窗口人员看有帮助。

## 4. 拍照定位降级判断

**判断：** 拍照降级会损失一点心智，但家庭用户的核心不是拍照，是别漏办。拍照在每个孩子/父母场景里自然出现。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 单独没有订阅价值，妈妈们会用 LINE OCR 或朋友翻译。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动比知识库更能订阅，因为家庭事项跨月跨年。知识库只会被收藏吃灰。

**我担心场景驱动的最大风险：** 家庭场景最怕给出过度确定的资格判断。父母养老、介护、儿童手当都要降级成材料与窗口确认。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做“家庭成员档案 + 3 个家庭场景”：孩子入学、父母长期来日、离婚/丧偶后手续。家庭成员变化自动生成提醒。

## 创始人最该警惕的 1 件事

不要把家庭场景当流量长尾，它可能是 TEBIQ 的高 LTV 用户群。
