# founder_03 周牧｜横滨中小企业 HR SaaS 创始人，前派遣行业产品负责人

## 1. 付费模式判断

**选择：选项 3。** 选 3，但我会把“进行中场景追踪”放到企业/家庭协作里。员工个人月付 ¥980 转化可能只有 2-3%，但公司为 10 名员工付 ¥3,000-5,000/月很容易谈。

**¥980 的消费心理：** 对年收 300 万的特定技能用户，¥980 是一顿外食，心理门槛高；对雇主 HR，它低于一次电话确认成本。TEBIQ 不要只看 C 端 ARPU。

**我做过的订阅里，触发付费的最强信号：** 最强信号是一个场景涉及两方：本人 + 公司/配偶/共同经营者。TEBIQ 应捕捉“邀请他人查看/确认材料”的事件，这比搜索次数更接近付费。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** HR 工具里搜索框使用率不到 15%，用户更常点“入社、退职、住所変更、在留更新”这种任务模板。日本企业用户尤其不爱从空白框开始。

**如果不是搜索，应该是什么：** 首页应是“手续类型选择 + 当前进行中”，搜索放右上。对公司搬迁这种场景，先问公司/个人/家属哪一方发生变化。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文 1.0 没问题，但 1.1 我不会做日语 UI C 端，而会做日语“协作者页面”：HR 看到员工授权过的材料清单。越南语应找派遣/注册支援机构合作。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 给非日语员工不可行，但给日本 HR 可行，而且更有付费能力。

## 4. 拍照定位降级判断

**判断：** 拍照降级合理。文书识别是输入，不是产品。公司侧真正关心是否已届出、是否缺税证。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 单独对 HR 没商业价值，HR 已经能读日文；价值在跨语言解释和提醒员工补齐。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动更能撑订阅，特别是多人协作场景。知识库是培训资料，不是系统。

**我担心场景驱动的最大风险：** 最大风险是 TEBIQ 同时服务员工和雇主时信任冲突。用户必须清楚哪些档案共享，哪些不共享。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做“公司/他人协作只读链接”的最小版本：场景页生成一份日文材料清单，可授权给 HR 看 7 天。

## 创始人最该警惕的 1 件事

多语言不是翻译问题，是谁付钱的问题。越南语个人版之前，先测试日本 HR 是否愿意为外国员工流程付费。
