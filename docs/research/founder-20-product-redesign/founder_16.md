# founder_16 伊藤里奈｜前 Yahoo! JAPAN 搜索/本地信息 PM

## 1. 付费模式判断

**选择：选项 2。** 选 2。搜索作为主入口时，次数限制是最自然的 monetization。日本用户熟悉免费 quota，但必须允许浏览结果摘要免费。预计搜索超限付费 2-3%。

**¥980 的消费心理：** ¥980/月若对应 unlimited search 比较好理解；若对应“深功能”则需要教育。它比 Yahoo Premium 便宜一点，但用户会要求稳定性。

**我做过的订阅里，触发付费的最强信号：** 最强信号是 query refinement：用户连续改写同一个问题。TEBIQ 应记录 same_intent_queries >= 3，然后提示保存为场景。

## 2. 搜索作为主入口的判断

**判断：同意，但必须被场景化约束。** 日本门户搜索使用率当然高，但在垂直手续领域，空搜索框要靠 autocomplete。无推荐词时移动端首屏输入率会低于 20%。

**如果不是搜索，应该是什么：** 我支持搜索替代 CTA，但必须是“带 autocomplete 的场景搜索”，并把 20 个场景作为 suggest。

## 3. 多语言节奏判断

**判断：基本同意，但要分清 UI、源信息和协作者界面。** 中文后做日语 UI 我部分赞成，因为日语源信息维护更容易。之后英语，再越南语。越南语没有审核资源不要上。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 可作为 source-first 产品，但结果解释必须母语。

## 4. 拍照定位降级判断

**判断：** 拍照降级合理。搜索能覆盖没有文书的前置问题。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 单独是搜索输入的一种，不是商业主体。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动和搜索不冲突：搜索负责 intent capture，场景负责订阅。

**我担心场景驱动的最大风险：** 风险是开放搜索导致低质量答案。必须优先返回 curated scenario。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做 query-to-scenario mapping：收集 100 个真实问法，映射到 20 个场景，未命中才调用 AI。

## 创始人最该警惕的 1 件事

搜索不是一个框，而是一套召回、改写、无结果、纠错系统。不要低估工程量。
