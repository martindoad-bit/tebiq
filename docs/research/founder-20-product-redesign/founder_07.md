# founder_07 黄泽｜京都行政内容运营创业者，前知识付费负责人

## 1. 付费模式判断

**选择：选项 3。** 选 3。资料包卖 ¥980 一次性转化高，但复购弱；订阅要靠“跟踪”。我卖签证资料包时，文章页付费率 2%，但到期提醒和材料状态能提高到 5%。

**¥980 的消费心理：** ¥980/月如果只是看内容，用户会说小红书免费；如果是“我的在留更新还有 63 天、缺两份材料”，它才像工具钱。

**我做过的订阅里，触发付费的最强信号：** 最强信号是用户在材料清单里打勾超过 3 项。TEBIQ 应捕捉 checklist_progress >= 30%，这时推订阅最自然。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** 内容产品里搜索框常被高阶用户用，普通用户更喜欢“热门问题”。搜索使用率约 22%。

**如果不是搜索，应该是什么：** 首页应先给“20 个高频场景”，搜索在上方但不是唯一主入口。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文先。日语 UI 只能作为 B 端审核/合伙人后台，不该当 1.1 C 端。英语比越南语更容易自己维护，越南语需要本地审核员。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 不能服务不懂日语的人，尤其签证内容，一行错读就会造成投诉。

## 4. 拍照定位降级判断

**判断：** 拍照可以降级。它是用户拿到文书后的补充，不是每个场景的开始。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 可作为获客 demo，但商业价值会被 ChatGPT 和手机翻译压低。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动更能撑订阅；知识库最好做 SEO 和证据库。

**我担心场景驱动的最大风险：** 场景页如果像文章，会失败；如果像申请代办，会违法。中间形态是“材料准备状态机”。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会把现有 batch-07 改成 20 个“状态机页面”，每页都有开始条件、所需文书、期限、保存按钮和不支持边界。

## 创始人最该警惕的 1 件事

不要再堆文章数量。165 篇已经够了，问题是它们没有变成产品状态。
