# founder_20 森田沙耶｜前数字银行 onboarding PM

## 1. 付费模式判断

**选择：选项 3。** 选 3。银行 onboarding 里，用户愿为安全保存和后续提醒付费，但必须先建立 trust。转化不是靠限制，而是靠“这个信息下次还会用”。

**¥980 的消费心理：** ¥980/月比很多银行附加服务高，用户必须看到“历史档案超过 30 天”的明确收益。

**我做过的订阅里，触发付费的最强信号：** 最强信号是 profile_completed。TEBIQ 应让用户完成签证类型、在留期限、住所、家庭 4 个字段后展示可省下的重复输入。

## 2. 搜索作为主入口的判断

**判断：不同意把裸搜索框替代首页 CTA。** 银行 app 搜索低于 10%，用户走固定流程。手续产品也应流程化。

**如果不是搜索，应该是什么：** 不要主打搜索。首页先显示“创建你的生活档案”，再给场景入口。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文后做英语更有付费潜力；日语 UI 可用于官方对照，不应作为非日语用户主界面。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 不能替代本地化信任说明。隐私和数据删除要母语。

## 4. 拍照定位降级判断

**判断：** 拍照降级合理，但上传前需要解释原图保存策略。

**“图片识别 + OCR”作为单独产品的商业价值：** OCR 单独无法成为银行级信任产品。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景驱动能撑订阅，因为它驱动资料完整度。知识库不会让用户填档案。

**我担心场景驱动的最大风险：** 场景越多，资料收集越多，最怕 purpose limitation 不清。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会做 profile completion meter：不是营销进度条，而是告诉用户哪些字段会用于哪些提醒，并允许删除。

## 创始人最该警惕的 1 件事

档案中心化必须配套“为什么保存、保存多久、怎么删”。
