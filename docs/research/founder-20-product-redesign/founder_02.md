# founder_02 林若曦｜池袋华人内容电商创业者，前内容社区 PM

## 1. 付费模式判断

**选择：选项 3。** 选 3。中文用户从内容平台进来时，先要确认“这件事跟我有关吗”，不能先收费。我的内容付费漏斗里，免费读 3 篇后直接月付只有 1.6%，但加入收藏/进度/提醒后能到 5-7%。

**¥980 的消费心理：** ¥980/月在华人私域里会被理解成“一个小服务”，不是内容订阅。它比小红书资料包贵，比一次行政咨询便宜太多，所以必须让用户感觉在买“持续跟进”。

**我做过的订阅里，触发付费的最强信号：** 最强信号是用户收藏或转发给配偶/合伙人。TEBIQ 应加“把这个场景发给家人/共同经营者确认”，分享后回来的人付费概率会高。

## 2. 搜索作为主入口的判断

**判断：同意，但必须被场景化约束。** 内容站大搜索框使用率在移动端约 28%，但有热点场景词时会到 45%。关键不是搜索框，而是默认 placeholder 要像真实话：“我离婚了，签证怎么办”。

**如果不是搜索，应该是什么：** 我支持搜索做主入口，但必须是“自然语言场景搜索”，下方马上给预制场景，不要让用户进入空结果页。

## 3. 多语言节奏判断

**判断：需要修改节奏。** 中文仍是第一语言。日语 UI 应作为给公司 HR、行政书士、配偶查看的副界面，不是给越南人过渡。英语和越南语前应先验证繁体中文。

**“日语 UI 给非日语用户用”是否可行：** 日语 UI 给非日语用户用只会降低安全感。用户在焦虑场景会回到母语，不会练日语。

## 4. 拍照定位降级判断

**判断：** 拍照降级可以，但拍照入口要在搜索结果和场景页上方固定出现。用户从内容进来时，最容易被“拍这个通知看看”触发。

**“图片识别 + OCR”作为单独产品的商业价值：** 单独 OCR 能带来分享传播，但不能带来稳定订阅。

## 5. 场景驱动 vs 知识库驱动判断

**哪种更容易撑订阅：** 场景比知识库更能订阅，因为它给“完成度”。内容用户愿意为“我还差 2 步”回来。

**我担心场景驱动的最大风险：** 风险是场景太多变成文章目录。每个场景必须有状态：未开始、待确认、已完成。

## 如果我今天接手 TEBIQ，第一个月最重要的 1 件事

第一个月我会把 20 个 batch-07 场景做成“小红书可分享标题 + 产品内 6 步路径页”，每页都接“保存到我的提醒”。先不做复杂搜索排序。

## 创始人最该警惕的 1 件事

别把知识库藏得太深。它不是主入口，但它是建立“你懂日本”的证据。
