# TEBIQ 1.0 上线就绪诊断：专家20打分

评估日期：2026-04-29  
基础参考：`docs/research/round-3/expert-20.md` 的合规、隐私、消费者保护和本地运营风险框架  
本轮专家配置：按任务要求重组为 4 类，每类 5 人：行政書士 / UX设计师 / 产品老兵 / 在日运营  
线上路径：`https://tebiq.jp` → `/photo` → `/check` → 选择/默认 `技術・人文知識・国際業務` → 单维度 → `/timeline` → `/pricing`

## 行政書士 5人

### A01 技人国续签行政書士，30年
- 专业判断1：`/check` 默认进 `技術・人文知識・国際業務` 可以理解，但用户没有先确认自己的在留资格，容易把结果套错。
- 专业判断2：维度标签 `建议必看` 和首页 `当前风险点` 仍像风险判断，不像材料准备。
- 专业判断3：单维度点击 404 是合规和信任双重问题；用户会以为 TEBIQ 没有认真验证续签路径。
- 如果是我的产品我会怎么改：把 `续签自查` 改成 `续签材料准备检查`，并把每个维度写成“递交前确认 X”，修好单维度后再开放。

### A02 永住再申请行政書士，25年
- 专业判断1：`永住准备` 出现在同级签证切换里，会吸引复杂用户，但页面没有说明不判断永住可能性。
- 专业判断2：时间线有价值，因为永住材料需要长期税、年金、收入记录；当前 `/timeline` 空态没有表达这个价值。
- 专业判断3：价格页没有数据保存期限和删除说明，永住用户不会上传十年材料。
- 如果是我的产品我会怎么改：给永住准备单独加边界说明：只整理材料和期限，不判断可否通过。

### A03 经管签行政書士，15年
- 专业判断1：`経営・管理` 在自查列表中出现，但没有“个人/法人资料分开”的提示，风险很高。
- 专业判断2：经管续签涉及办公室、经营实态、税务、雇佣，不适合和技人国共用同一套轻量维度表达。
- 专业判断3：拍照页支持 `契約書`，但合同/法人文件如果只给通用步骤，容易越界。
- 如果是我的产品我会怎么改：经管入口标注“仅整理文书事实，不判断经营稳定性”，并把公司文件默认不入个人时间线。

### A04 配偶者签证行政書士，8年
- 专业判断1：`配偶者` 用户会上传分居、离婚、家暴、收入依赖材料；当前页面没有复杂 case 降级机制。
- 专业判断2：首页 `做一次续签自查` 对家庭变故用户会制造风险预期。
- 专业判断3：单维度 404 对配偶者用户尤其危险，因为他们本来就焦虑且个案差异大。
- 如果是我的产品我会怎么改：配偶者维度只做事实整理，任何婚姻真实性、同居持续性判断都转为“咨询行政書士/弁護士”。

### A05 入管实务行政書士，20年
- 专业判断1：TEBIQ 可以帮用户看懂文书，但不能让用户觉得“AI 预测入管判断”。
- 专业判断2：结果页应该按机构写准免责声明：入管类以出入国在留管理庁通知为准，税务类以税务署/自治体为准。
- 专业判断3：`/pricing` 用订阅承接焦虑用户前，应明确不提供代办、代写、审查判断。
- 如果是我的产品我会怎么改：把边界写进按钮和状态，不靠页脚免责声明；例如 `保存材料提醒` 而不是 `判断风险`。

## UX设计师 5人

### U01 移动端工具UX设计师
- 专业判断1：首页视觉克制成立，但同屏两层同名入口破坏信息架构。
- 专业判断2：`TEBIQ` hero 留白过大，首屏没有露出结果样例，用户看不到完成态。
- 专业判断3：底部 Tab 是清楚的，但 `我的提醒` 在首页和底部重复出现时含义不同。
- 如果是我的产品我会怎么改：首页只留一个主 CTA，下面直接放一条脱敏样例记录：`住民税通知 / 6月30日 / 待缴`。

### U02 字体与可读性设计师
- 专业判断1：部分页面标题和签证名使用偏重字重，和 Visual Polish 12 的 300/400/500 规范冲突。
- 专业判断2：日文官方名和中文解释混在同一强标题层级，`技術・人文知識・国際業務` 看起来像中文粗标题，不像官方术语。
- 专业判断3：价格页数字可读性强，但过大金额压过权益说明，用户只记住价格。
- 如果是我的产品我会怎么改：日文术语用专门的 `.jp-text` 层级，字重 400/500，中文说明另起一行，不把日文官方名做成视觉主标题。

### U03 信息层级设计师
- 专业判断1：`/check` 顶部卡片、签证切换、维度清单同时出现，用户不知道当前是在“完整自查”还是“单项自查入口”。
- 专业判断2：维度行右侧 `未查` 与标签 `未查` 重复，信息密度浪费。
- 专业判断3：横向签证切换部分截断 `永住准备`，能提示可滑动，但也会让新用户误以为信息没显示完。
- 如果是我的产品我会怎么改：先做“选择在留資格”一步，再进入维度；维度行只保留一个状态字段。

### U04 移动端转化设计师
- 专业判断1：`/photo` 的拍照框够明显，但没有低风险试看的样例。
- 专业判断2：上传 PDF/截图入口位置好，但用户不知道图片上传后会不会保存原图。
- 专业判断3：`今日 残り 1 次` 是日文混中文，和按钮中文不自然。
- 如果是我的产品我会怎么改：把配额文案改为 `今天还可免费识别 1 次`，旁边放 `查看样例结果`。

### U05 空态与状态设计师
- 专业判断1：`/timeline` 0 件、0 个月、0 类机构的空态过冷，像后台统计。
- 专业判断2：空态没有展示提醒系统的未来状态，不会促使用户保存第一份文书。
- 专业判断3：404 页面视觉好，但从单维度来的用户需要“回到上一页”，不是只回首页。
- 如果是我的产品我会怎么改：未登录时间线放三条灰态样例：`国保通知 待缴`、`在留更新 递交前确认`、`学校缴费 7天后`。

## 产品老兵 5人

### P01 工具产品老兵
- 专业判断1：产品方向应是“文书事件系统”，不是“工具入口集合”。
- 专业判断2：当前首页仍有工具集合味道，时间线没有承担主叙事。
- 专业判断3：单维度 404 说明上线验收没有覆盖关键路径。
- 如果是我的产品我会怎么改：把验收路径定义为“拍一份文书 → 生成事件 → 生成提醒 → 回到时间线看见状态”，没有闭环不发布。

### P02 订阅商业化老兵
- 专业判断1：¥980/月可以卖，但不能卖“无限拍照”，必须卖“长期不漏办”。
- 专业判断2：免费每天 1 次会覆盖大量低频用户，付费点要落在历史、提醒、批量和保存。
- 专业判断3：价格页缺自动续费/取消/退款说明，会降低订阅信任。
- 如果是我的产品我会怎么改：会员页改成三段：免费识别、付费保存/提醒、年费覆盖全年行政周期。

### P03 AI产品老兵
- 专业判断1：AI 能力应隐藏在结构化结果里，当前 `/photo` 还没让用户看到结构化完成态。
- 专业判断2：拍照结果必须让金额、期限回到原文位置，否则用户不会把它当可靠工具。
- 专业判断3：高风险文书需要分类器和降级模板，不能只靠提示词。
- 如果是我的产品我会怎么改：先做 50 类真实文书安全模板，再开放泛文书识别。

### P04 增长产品老兵
- 专业判断1：传播点很强，但首页重复入口会降低首拍转化。
- 专业判断2：投放到 `/check` 前必须修 404，否则新增用户会被漏斗中段吃掉。
- 专业判断3：结果分享摘要是增长回路，现在没有出现。
- 如果是我的产品我会怎么改：只投放 `/photo`，用脱敏真实信件 demo 承接；自查修好前不买量。

### P05 垂直服务产品老兵
- 专业判断1：TEBIQ 必须守窄：看懂、整理、提醒，不代办。
- 专业判断2：经管、永住、高度人材会自然流入，但 1.0 不应主动承诺复杂判断。
- 专业判断3：价格不能靠焦虑驱动，必须靠“少漏办、可回查”驱动。
- 如果是我的产品我会怎么改：所有复杂签证入口增加边界卡，明确“本页不判断通过可能性”。

## 在日运营 5人

### O01 在日华人社群运营
- 专业判断1：社群里用户会先问“这封信什么意思”，拍照入口是对的。
- 专业判断2：如果有人点自查维度看到 404，会在群里直接截图吐槽。
- 专业判断3：繁体用户、久居用户、蓝领用户都需要被看见，当前简体新来日气质偏强。
- 如果是我的产品我会怎么改：先用 100 封真实脱敏社群信件做样例和测试，不从功能菜单定义需求。

### O02 留学生渠道运营
- 专业判断1：留学生会用拍照，但 `/check` 没有留学续签，容易失望。
- 专业判断2：¥980/月对学生偏高，免费 1 次会满足多数低频需求。
- 专业判断3：学校缴费、出勤率、年金/国保才是学生入口，不是技人国默认自查。
- 如果是我的产品我会怎么改：做留学生文书 landing：`学费督促 / 出席証明 / 国民年金 / 国保`，只引导拍照和提醒。

### O03 特定技能/蓝领渠道运营
- 专业判断1：蓝领用户最需要看懂工资、寮费、雇用条件，但首页没有这些词。
- 专业判断2：特定技能在横向列表里有，但默认技人国会让低日语用户不敢点。
- 专业判断3：¥980/月对这群人难守，短期包或渠道补贴更现实。
- 如果是我的产品我会怎么改：特定技能入口用 `工资单/雇用条件/寮費` 组织，不先讲续签风险。

### O04 华人家庭/妈妈群运营
- 专业判断1：保育园、学校、児童手当、就学援助是高频刚需，但当前页面没有形成家庭心智。
- 专业判断2：繁体和家庭成员支持缺失会影响妈妈群传播。
- 专业判断3：时间线若能跟踪孩子 deadline，会比泛签证更容易留存。
- 如果是我的产品我会怎么改：首页样例加入 `保育園通知`，时间线支持给记录加 `孩子` 标签，先不做复杂家庭账号。

### O05 经管/老板圈运营
- 专业判断1：经管用户付费能力强，但会上传公司税务、社保、租约，隐私和法人隔离是第一问题。
- 专业判断2：低价会员页会让老板怀疑专业性，尤其没有数据治理说明。
- 专业判断3：经管不应和普通个人文书混用同一承诺。
- 如果是我的产品我会怎么改：经管只做“文书识别+资料整理”，不要营销自查；公司文件默认不保存原图。

## 4类专家共识 Top 5

1. **单维度 404 必须上线前修掉**  
   行政書士看成合规信任问题，UX 看成路径断裂，产品看成验收失败，运营看成口碑风险。

2. **自查必须从“风险判断”退回“材料准备”**  
   `当前风险点`、`建议必看`、`续签自查` 的组合让用户以为产品在判断结果。

3. **拍照结果样例必须前置**  
   所有人都认可“拍一下，看懂”，但当前线上没有证明识别结果长什么样。

4. **隐私和数据控制是付费前提**  
   删除、导出、不保存原图、只保存摘要、数据保存期限必须在产品界面可见。

5. **时间线要变成待办，不是记录列表**  
   `我的提醒` 应承接状态、期限和完成动作，未登录空态也要展示未来价值。

## 4类专家分歧点

1. **首页要多冷还是多解释**  
   行政書士和产品老兵支持冷静克制；UX 和运营认为新用户首屏需要一个真实样例，否则过冷会劝退。

2. **¥980 是否可以守住**  
   产品老兵偏谨慎，认为月费对低频文书难；运营认为技人国、家庭、经管在特定窗口能付；行政書士不关心价格，关心边界和投诉。

3. **是否继续展示多签证类型**  
   运营希望覆盖特定技能、留学、家庭、经管；行政書士担心覆盖越多越像个案判断；产品老兵建议营销窄、识别能力宽。

4. **繁体是否上线前必修**  
   UX/运营认为繁体是身份信号，影响台湾/香港传播；产品老兵认为可排在单维度 404 和隐私之后。

5. **结果页要不要给下一步行动**  
   运营希望明确下一步，降低用户焦虑；行政書士要求复杂文书只给事实和材料，不给行动判断；折中方案是按文书风险分级输出。

## 专家侧上线判断

当前版本可以作为内部/小范围灰度试用，但不适合公开付费放量。原因不是产品价值不足，而是三条专业底线没打牢：

1. 续签自查路径存在硬断点。
2. 风险语言和签证类型覆盖容易制造合规误解。
3. 付费前隐私与消费者保护信息不足。

专家建议的上线前最小护栏：

1. 修复或关闭单维度入口。
2. 自查文案改为材料准备检查。
3. 拍照结果显示字段来源和原文核对。
4. 价格页补自动续费、取消、退款、数据保存/删除。
5. 高风险文书降级输出，不做个案行动建议。
