## 1. 三视角共同识别的 Top 10 必修问题

### P0-1. `/check` 单维度展开 404

页面/交互：`/check` → 默认或选择 `技術・人文知識・国際業務` → 点击 `在留期限` 等任一维度。  
发生了什么：进入 `这一页迷路了` 404。  
为什么必须先修：这是本轮最硬的上线断点。客户会退出，PM 会判定漏斗断，专家会判定续签路径未经验证。  
具体做什么：要么修好 `/check/{visa}/{dimension}`，要么在上线前取消维度行点击，只保留完整自查。

### P0-2. 首页入口重复，拍照主心智被稀释

页面/交互：首页首屏。  
发生了什么：两个大 CTA 与常用工具里的 `拍照即懂 / 续签自查` 重复。  
为什么必须先修：新用户第一眼不知道是流程入口、工具菜单，还是重复按钮。  
具体做什么：首页保留 `拍一份文书试试` 为主 CTA；`续签自查` 下沉到常用工具或二级入口；常用工具区改为样例/状态，不再重复大 CTA。

### P0-3. 拍照页没有样例结果，付费价值不可见

页面/交互：`/photo`。  
发生了什么：用户只看到拍照框、上传 PDF/截图和免费次数，看不到识别后结果。  
为什么必须先修：用户付费前最想知道“金额、期限、下一步、提醒怎么出现”。当前只能盲试。  
具体做什么：增加一个脱敏样例结果入口，展示 `文书类型 / 发件机构 / 金额 / 期限 / 下一步 / 保存到我的提醒`。

### P0-4. 自查语言仍像风险判断

页面/交互：首页 `做一次续签自查`、常用工具 `当前风险点`、`/check` 标签 `建议必看`。  
发生了什么：用户会把产品理解成判断续签风险，而不是整理材料。  
为什么必须先修：行政書士视角认为这是产品边界风险，客户视角会焦虑，PM 视角会拉高投诉成本。  
具体做什么：改成 `续签材料准备检查`、`准备事项`、`递交前确认`、`需要补齐`、`基本齐备`。

### P0-5. 价格页缺消费者保护信息

页面/交互：`/pricing`。  
发生了什么：只展示免费/月度/年度、Stripe 付款，没有自动续费、取消、退款、支付方式、数据保存说明。  
为什么必须先修：¥980/月不是单纯价格问题，是信任问题。  
具体做什么：在开通按钮前写清楚续费周期、取消路径、退款/不退款规则、支付方式、数据删除入口。

### P1-6. 隐私和数据控制没有产品化

页面/交互：`/photo`、`/timeline`、`/pricing`。  
发生了什么：用户要上传在留、税务、收入、银行、家庭、学校文件，但看不到“不保存原图/删除/导出/保存期限”。  
为什么必须修：专家20把个人信息集中保存列为最高风险，客户中高收入/经管/家庭用户也因此不付费。  
具体做什么：显示 `本次不保存原图`、`删除单份记录`、`删除全部档案`、`导出时间线`、`只保存摘要`。

### P1-7. `我的提醒` 空态像空数据库，不像待办系统

页面/交互：`/timeline`。  
发生了什么：显示 0 件、0 个月、0 类机构、暂无匹配记录。  
为什么必须修：时间线是 TEBIQ 对抗 ChatGPT 的核心，但当前空态没有展示长期价值。  
具体做什么：未登录也展示灰态样例：`住民税通知 待缴`、`在留更新 递交前确认`、`学校缴费 7天后`，并显示 `待确认 / 待处理 / 已完成` 状态。

### P1-8. 拍照结果页语言层级需要统一

页面/交互：`/photo/result/[id]`。  
发生了什么：创始人实测提到标题日文汉字、正文夹简体中文切换不自然；本轮因无样例文书没有在生产上传复测，但结果页模板仍允许日文 docType 原样做标题、中文 summary/actions 做正文。  
为什么必须修：中日混合是 TEBIQ 的定位资产，但混排要像在日华人真实表达，不像两套语言拼贴。  
具体做什么：标题用 `日文官方名 + 中文类型说明` 双行结构，例如 `納税通知書` 下方 `住民税缴费通知`；正文用中文，关键日文术语保留原文。

### P1-9. 字体/字重与 Visual Polish 12 不完全一致

页面/交互：`/check`、`/pricing`、部分标题。  
发生了什么：签证名、维度标题、部分页面标题视觉上偏重，和产品哲学里“只用 300/400/500”的工具感不完全一致。  
为什么必须修：视觉方向已经对了，但字体细节会让页面从“邮便局/银行清单”回到普通互联网 App。  
具体做什么：把用户可见标题的 `font-semibold` 降到 500；日文术语使用专门层级，不和中文说明抢同一标题权重。

### P1-10. 签证类型覆盖与默认选择容易误导

页面/交互：`/check`。  
发生了什么：用户直接进入技人国维度清单，其他签证在横向滚动里；留学、育成就劳、高度人材不在列表。  
为什么必须修：非技人国用户会以为被分错类；复杂用户会把自查误用到自己身上。  
具体做什么：先显示 `按在留カード上的「在留資格」选择`，选择后进入维度；不支持类型给出“先用拍照识别文书”的退路。

## 2. 三视角分歧点

1. **首页要不要继续克制**  
   客户和 UX 要更多样例；产品哲学要求克制工具感。折中：不加营销解释，加一条真实脱敏结果样例。

2. **¥980/月能不能守住**  
   客户中位付费意愿 5/10，PM 分布是 11 守住 / 21 边缘 / 18 守不住，专家不反对价格但要求消费者保护。结论：修复前守不住，修复后在文书密集期和年费上有机会。

3. **是否扩展到留学、特定技能、家庭、经管**  
   运营希望覆盖真实场景，行政書士担心越界，PM 建议营销窄、识别能力宽。结论：1.0 首屏仍主打拍照文书，不主动承诺复杂签证判断。

4. **繁体是否上线前必修**  
   客户中的繁体画像明显降信任；PM/运营认为影响台湾香港传播；产品优先级上仍排在 404、首页、隐私之后。结论：不是 P0，但应进入 P1。

5. **结果页要给多少行动建议**  
   客户需要下一步，运营担心用户看不懂，行政書士担心过度指导。结论：低风险文书给保守下一步，高风险文书只给事实整理和专家类型提示。

## 3. 整体上线就绪度

**58 / 100**

拆分判断：
- 用户痛点真实度：82/100。`拍一下，看懂` 仍是最强资产。
- 视觉成熟度：74/100。深墨蓝 + 灰度工具感成立，但字重和空态还要收。
- 核心路径完整性：45/100。单维度 404 是严重扣分。
- 付费承接：50/100。价格清楚，但价值和信任信息不足。
- 合规/隐私可控性：48/100。边界意识有，但还没产品化。

结论：**不建议公开付费放量；可以小范围灰度，但上线前至少修 P0-1 到 P0-5。**

## 4. 创始人最该立刻关注的3件事

1. **先修路径，不要先改漂亮**  
   立刻处理 `/check` 单维度 404，并决定单维度是不是 1.0 必须开放。如果来不及，宁可关闭点击，也不要让用户点进迷路页。

2. **把首页和拍照页改成一个明确承诺**  
   第一屏只让用户知道一件事：拍眼前这封日本信，TEBIQ 会给出金额、期限、下一步，并保存到提醒。重复入口、空洞空态、无样例结果都要让位。

3. **把 ¥980 的信任补齐**  
   价格页和结果页补上：自动续费/取消/退款、数据保存/删除、不保存原图、只保存摘要、以原文件和机构说明为准。用户不是不愿付，他们是不敢把人生文件交给一个边界不清的工具。
