# Work Packet — CODEXUI 保存 / 分享 / 回访 refinement

**对象**: CODEXUI
**任务标题**: 真实用户路径优化 — 词汇 / 微信 / 桌面端 / 信息层级
**Sprint**: TEBIQ 0.6 / Wave 3
**Issued by**: GM (per PL 2026-05-07 §4 directive)

---

## 背景

PL §4 真实观察：

> Mac 浏览器里系统分享不能稳定分享到微信，这是平台限制，不是单纯 UI bug。
> 微信内打开有自己的分享/复制/转发逻辑。
> "回访"这个词可能不符合用户语义。
> 复制链接 / 分享给自己如果藏得太深，用户不一定找得到。

PR #75 + #81 已经把保存/复制/分享/我的咨询记录 落到了主分支。但 PL 实测发现：

- "回访" 词不自然
- 微信和桌面端体验差异大
- 信息层级仍可能误导用户（藏太深 vs 太显眼）

## 当前问题

1. **"回访"** 这个词容易让用户误解（"再访问？谁访问谁？"）
2. **桌面端 navigator.share** 在 Mac 浏览器分享微信不稳定 → 用户期待落空
3. **微信内置浏览器** 有自己的分享逻辑（右上角三点菜单）→ 我们额外塞分享按钮可能冲突
4. **信息层级** — "保存这次咨询 / 复制链接 / 分享给自己 / 我的咨询记录" 4 项之间的优先级仍可能不清

## 目标

```
用户看完一条有价值回答后，能自然保存、复制、发给自己、下次回来。
```

具体：

1. 用户能直接理解如何保存这次咨询（不需要解读"回访"等词）
2. 用户能直接复制链接（一步操作）
3. 微信内打开时，不被引导到复杂外部流程（尊重微信原生分享）
4. 桌面端不强求一键分享到微信（接受平台限制）
5. "我的咨询记录" 路径清楚（PR #87 已改对，本 Pack 复核）
6. "回访" 这类容易误解的词由 CODEXUI 重新判断
7. 保存 / 分享 / 记录 / 详情 / 补充不形成后台控制面板感（per UX refinement 教训）

## 工作方式（先 UX 判断，再实现）

PR description 顶端 5 问：

```
1. "回访"这个词为什么用户不舒服？换什么词更自然？
2. 桌面端不能稳定分享微信 → 用户期待如何降级（不是 hack 而是诚实设计）？
3. 微信内置浏览器原生分享已存在 → 我们的按钮如何与之共存（或退让）？
4. 4 个动作（保存 / 复制 / 分享 / 我的记录）哪个是真主动作？哪些应弱化？
5. 用户什么时候会想"下次回来" — 答案显示中还是答案完成后？
```

## 约束

不做：

- ❌ 不再做 navigator.share 跨平台 hack（接受平台限制）
- ❌ 不在微信内强制弹"添加到桌面"
- ❌ 不在桌面端强求一键分享到微信
- ❌ 不重做 brand / Logo / 主色
- ❌ 不暴露内部词（"Learning Console" / "fact" / "consultation_id" 等）
- ❌ 不变成功能控制台（per UX refinement 教训）
- ❌ 不阻塞用户视线（保存/分享区域不能压过回答）
- ❌ 不绕过 GM 联系 ENGINE / FACT / DOMAIN / QA / PL

可做：

- ✅ 重写所有相关 microcopy（"回访" 等）
- ✅ 删除 / 合并 / 折叠现有按钮
- ✅ 重新决定移动端 / 桌面端不同信息层级
- ✅ 重新决定微信内置浏览器特殊处理（如检测 UA 后改文案 / 隐藏某按钮）
- ✅ 加 inline 提示文案（如 "微信里可点右上角三点分享"）
- ✅ 改 "我的咨询记录" → CODEXUI 判断更好措辞
