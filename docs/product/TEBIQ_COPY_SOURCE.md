# TEBIQ Copy Source

## 0. 用途

本文件存放用户端文案资产，供 Claude Code / VOICE / UI 窗口读取。

使用规则：

- 文案进入 UI 前必须经过总控确认
- VOICE 可以建议修改，但不能直接覆盖已冻结文案
- Claude Code 落地页面时优先读取本文件中的 A 层（已冻结）文案
- 本文件文案不代替 `TEBIQ_CONTEXT_PACK.md`，两者必须同时读取

## 2. 禁用词表

以下词句禁止出现在任何用户可见位置，包括按钮、标题、正文、气泡、事项卡：

- 别担心
- 我懂你
- 我可以帮你
- 重要风险提醒
- 你没问，但要注意
- AI 智能分析
- 一键开启专业守护
- 严重违规
- 立即预约专家
- fallback_reason（内部字段，不得出现在页面）
- safety_gate_replaced（内部字段，不得出现在页面）
- unknown / null / undefined（任何用户可见位置）

## 4. VOICE 任务卡模板

VOICE 窗口执行文案任务时，必须填写以下模板后方可输出：

```md
# TEBIQ Voice Task Card

## 页面位置
例如：回答页 / 首页主标题 / 事项卡 / 风险结果页

## 用户原话
用户实际怎么问

## 用户状态
在留资格、到期日、最近变更、家庭关联、公司情况等

## 已确认事实
只放已确认事实，不放模型猜测

## 页面目标
这段文案要让用户做什么

## 正向样例
给 1–3 个好例子，并说明为什么好

## 反向样例
给 1–3 个坏例子，并说明为什么坏

## 禁用词
列出本任务禁用表达

## 输出要求
字数、格式、按钮数量、是否需要内部说明
```

## 5. VOICE 独特约束

- VOICE 不允许决定页面结构
- VOICE 不允许自己发明法律事实
- VOICE 不允许一次写完整 6 屏
- VOICE 每次只处理一个页面或一个模块
- VOICE 必须基于 Context Pack + Task Card 工作
- 没有上下文卡时，VOICE 应停止并要求补充，而不是猜
- VOICE 输出的文案必须标注层级（B 层待验证），不能自行升级为 A 层
