## RP-01 — 時間敏感期限

> 用户存在明确的时间死线，错过将导致不可逆的在留后果。

**優先度**：HIGH  
**与 R01 的关系**：直接对应。R01 的目标是防止时间敏感题被路由为 out_of_scope。

### 覆盖题目（13 题）

| starter_tag | 题目 | 期限リスク |
|---|---|---|
| J03 | 签证快到期，材料没准备好 | 更新申請期限（P0）|
| F01 | 补材料期限赶不上 | 補材料期限（P0）|
| F02 | 补材料期限已过 | 補材料超過（P0 最緊急）|
| F05 | 收到不许可通知书 | 再申請・出国期限 |
| F06 | 不许可后还能待多久 | 出国期限内の行動 |
| F08 | 不许可通知书期限的意思 | 出国期限の誤解（P0）|
| G05 | 在留卡快到期，还没收到结果 | 特例期間の理解 |
| D06 | 配偶签离婚后多久要处理在留 | 6ヶ月期限（P0）|
| A09 | 配偶签离婚后转定住 | 変更申請期限 |
| J05 | 离婚了但没告诉入管 | 届出遅延リスク |
| E07 | 永住不许可后多久再申请 | 再申請タイミング |
| F07 | 不许可后还能再申请吗 | 再申請期限 |
| I04 | 更新申请中可以出境吗 | みなし再入国期限 |

### DOMAIN 语义观察

**核心模式**：用户提问时往往低估时间紧迫性，或不知道"还剩多少时间"。TEBIQ 的首要任务是：
1. **识别当前所处的时间节点**（申请中 vs 未申请 vs 已超期）
2. **传达对应阶段的紧急程度**（不是泛泛说"要抓紧"）
3. **给出具体的下一步行动**（联系入管 / 立即申请 / 当日 handoff）

**must_surface 候选（RP-01 通用）**：
- 当前的时间节点是什么（申请中 / 未申请 / 已过期）
- 对应的最高风险后果（在留取消 / 不许可 / 强制出国）
- 最紧急的下一步行动

**must_not_assert 候选（RP-01 通用）**：
- 「まだ大丈夫」（未确认具体时间节点时）
- 「特例期間があります」（不许可后的特例期间处理与申请中不同）
- 具体日期承诺（未看到通知书的情况下）

**handoff_trigger（RP-01）**：
- F01, F02, F05, F06, F08, J03：当日または翌日の行政書士相談を推奨
- 补材料期限超过 → 入管窓口への即日相談

## RP-02 — 身份変更転換

> 用户的在留根据发生了根本变化（离婚/换签/身份变化），需要在时限内完成资格变更。

**優先度**：HIGH  
**与 R02 的关系**：直接对应。配偶者离婚后的定住者/身份变更是 R02 的核心目标。

### 覆盖题目（16 题）

| starter_tag | 题目 | 変更リスク |
|---|---|---|
| A01 | 経管→技人国 | 転換申請・役員退任 |
| A02 | 技人国→経管 | 経管要件充足 |
| A03 | 経管→人文 | 経管義務・転換 |
| A04 | 人文→経管 | 活動制限・転換 |
| A05 | 家族→工作 | 変更前就労禁止（P0）|
| A08 | 人文→定住者 | 定住者要件限定 |
| A09 | 配偶→定住者 離婚 | 6ヶ月期限（P0）|
| D03 | 家族→工作 | 変更前就労禁止 |
| D05 | 配偶離婚後 留日可否 | 在留根拠喪失（P0）|
| D06 | 配偶離婚後 期限 | 6ヶ月起算点（P0）|
| D08 | 配偶→定住者 申請 | 定住者審査 |
| D10 | 家族 子供 卒業後 | 変更必要性 |
| J04 | 解雇された 在留 | 活動制限・変更 |
| J05 | 離婚 届出未了 | 届出遅延・6ヶ月 |
| J07 | 打工→開店 経管 | 変更前開業禁止 |
| J08 | 在留と仕事不一致 | 活動範囲外（P0）|

### DOMAIN 語義観察

**核心模式**：変更のタイミングと順序の誤解が最大リスク。ユーザーは「転換できる」ことは知っているが「変更許可が出るまで新しい活動は不可」を知らないことが多い。

**must_surface 候選（RP-02 通用）**：
- 変更申請の許可が出るまで新しい資格の活動は不可
- 変更申請には審査があり（自動的には通らない）
- 期限がある場合は起算点と期限を明示

**must_not_assert 候選（RP-02 通用）**：
- 「変更できます」と確定的承諾（審査結果は保証できない）
- 「まず就職/開業してから申請を」（順序の誤り）
- 「離婚後は定住者に変更できます」（条件付きで確定的ではない）

**新規 must_have 候補**：
- MH-R02-01: 配偶者離婚後の変更申請で「起算点は離婚届受理日」を明示すること
- MH-R02-02: 家族滞在からの転換では「変更申請許可前の就労は不法就労」を明示すること
- MH-R02-03: 経管→技人国転換では「役員退任登記が必要」を明示すること

## RP-03 — 経管合規離境

> 経営管理签持有者が廃業・回国を検討する場面で発生する複合的な在留リスク。

**優先度**：HIGH  
**与 R03/R05 の関係**：R03（経管域推断）と R05（帰国リスク識別）の核心。

### 覆盖题目（8 題）

| starter_tag | 題目 | リスク |
|---|---|---|
| B05 | 経管 赤字更新 | 更新拒絶リスク |
| B07 | 経管 放弃回国 | 清算+複合手続き（P0）|
| I01 | 経管 放棄 手続き | 清算順序（P0）|
| I08 | 会社清算なし 帰国 | 帰国リスク（P0 最高）|
| J06 | 会社ほぼ経営なし | 更新拒絶（P0）|
| B03 | 自宅兼事務所 | 更新拒絶リスク |
| B02 | シェアオフィス | 更新審査リスク |
| B06 | 資本金不足 | 審査要件 |

### DOMAIN 語義観察

**核心模式**：経管签持有者の最大リスクは「会社法上の義務」と「在留資格の義務」の二重性を理解していないこと。会社を「閉める」ことの複雑さと、清算未完了での離日の法的リスクを多くのユーザーが認識していない。

**must_surface 候選（RP-03 通用）**：
- 経管を「やめる」には清算（または代表交替）の法的手続きが必要
- 清算完了まで最低2〜3ヶ月かかる
- 税務・年金・代表変更の複合手続きが必要

**must_not_assert 候選（RP-03 通用）**：
- 「すぐ帰国できます」（清算なしに）
- 「会社は後で閉めればいい」
- 「赤字でも更新できます」（条件確認なしに）

**新規 must_have 候補**：
- MH-R03-01: 経管放棄帰国では「清算完了前の出国は在留取消リスク」を明示
- MH-R03-02: 経管放棄では「税務処理・年金脱退一時金・代表変更」の三領域に言及
- MH-R03-03: 経管継続更新が難しい場合は「行政書士相談当日」を推奨

**handoff_trigger（RP-03）**：
- I08, B07, I01：当日の行政書士相談を推奨（複合手続きの専門知識が必要）

## RP-04 — 活動範囲不一致

> 在留资格の活動範囲を超えた就労・業務が発生している/発生しそうな場面。不法就労リスクが最も高いパターン。

**優先度**：HIGH  
**与 R04 の関係**：直接対応。解雇・在留不一致は R04 の核心。

### 覆盖题目（9 題）

| starter_tag | 題目 | リスク |
|---|---|---|
| C03 | 人文 業務内容変化 | 活動範囲外・不法就労（P0）|
| C07 | 技人国 飲食現場 | 単純労働・不法就労（P0）|
| D02 | 家族滞在 28時間超 | 不法就労（P0 最高）|
| J08 | 在留と仕事不一致 | 不法就労（P0）|
| J04 | 解雇された 在留 | 活動中断リスク |
| A05 | 家族→工作 | 変更前就労 |
| D03 | 家族→工作 | 変更前就労 |
| C02 | 人文 離職 無職 | 活動中断リスク |
| C08 | 辞职後 就职期間 | 活動中断リスク |

### DOMAIN 語義観察

**核心模式**：ユーザーは「現在やっていること」が在留資格の範囲内かどうか判断できない。特に「管理職だから大丈夫」「少しだから大丈夫」という誤解が多い。

**must_surface 候選（RP-04 通用）**：
- 現在の業務内容が在留資格の活動範囲に該当するか確認が必要
- 範囲外の業務は不法就労（刑事罰の可能性）
- 業務内容の詳細確認なしに「大丈夫」とは言えない

**must_not_assert 候選（RP-04 通用）**：
- 「多少違っても大丈夫」（詳細確認なしに）
- 「飲食店でも大丈夫な場合がある」（条件確認なしに）
- 「28時間を少し超えただけなら」

**新規 must_have 候補**：
- MH-R04-01: 在留不一致/活動範囲外疑いでは「業務内容の詳細確認」を最初に行うこと
- MH-R04-02: 不法就労確認の場合は「即時中止 + 行政書士相談」を伝えること
- MH-R04-03: 解雇の場合は「14日以内の離職届出義務」を明示すること

## RP-05 — 家族連帯リスク

> 主たる在留者の資格変更が家族（家族滞在）の在留に連鎖的に影響するパターン。

**優先度**：HIGH  
**与 R02 の関係**：R02 の family cascade 側面。

### 覆盖题目（6 題）

| starter_tag | 題目 | リスク |
|---|---|---|
| D09 | 換签 家族 連帯影響 | 家族在留連動（P0）|
| D04 | 配偶签 分居 | 婚姻実態・更新 |
| D07 | 別居 離婚なし 更新 | 婚姻実態審査 |
| D10 | 家族 子供 卒業後 | 変更必要性 |
| A05 | 家族→工作 | 家族資格への影響 |
| D01 | 家族 アルバイト | 資格外活動 |

### DOMAIN 語義観察

**核心模式**：ユーザーは「自分の在留の質問」をしているが、家族全員の在留が連動していることを認識していない。

**must_surface 候選（RP-05 通用）**：
- 家族滞在は主たる在留者の資格に依存していること
- 主資格変更時は家族の手続きも同時に検討が必要

**must_not_assert 候選（RP-05 通用）**：
- 「家族の資格は独立しています」（家族滞在の場合は誤り）

## RP-06 — 公的義務漏れ

> 年金・税金・健康保険の未納・遅納・未申告が、在留更新または永住申請に影響するパターン。

**優先度**：HIGH  
**与 R01-R05 の関係**：直接対応するルールなし（Route修正では対応されない）。

### 覆盖題目（12 題）

| starter_tag | 題目 | リスク |
|---|---|---|
| E01 | 永住 年金未納 | 申請要件未充足 |
| E02 | 永住 住民税遅納 | 審査影響 |
| E05 | 永住 配偶者の税年金 | 連帯審査 |
| E10 | 永住 健康保険遅納 | 公的義務 |
| H02 | 国民年金未納 更新 | 更新・永住影響 |
| H03 | 住民税未納 更新 | 更新審査 |
| H08 | 確定申告未申告 | 脱税・更新影響 |
| H09 | 税務署補税 永住 | 永住審査不確定 |
| H10 | 住民税遅納vs未納 | 審査区別 |
| J09 | 税金年金問題 永住 | 複合問題 |
| E04 | 永住 収入不安定 | 安定性要件 |
| H07 | 回国前 納税管理人 | 出国後の義務 |

### DOMAIN 語義観察

**核心模式**：ユーザーは「払えばいい」と考えているが、「いつ払ったか」「何の理由で未払いだったか」が審査上の判断要素になることを知らない。

**must_surface 候選（RP-06 通用）**：
- 「遅納」と「未納」は審査上異なる扱い
- 配偶者の公的義務も審査対象
- 問題の種類（未納/遅納/未申告）によって解決方法と影響度が異なる

**must_not_assert 候選（RP-06 通用）**：
- 「払えば大丈夫」（時期・理由の確認なしに）
- 「未納でも問題ない」
- 「補納すればすぐ申請できます」

## RP-07 — 不許可後対応

> 不許可通知書を受け取った後、ユーザーが内容を誤解して対応を誤るパターン。

**優先度**：HIGH  
**与 R01 の関係**：期限誤解は R01 のカバー範囲。

### 覆盖題目（7 題）

| starter_tag | 題目 | リスク |
|---|---|---|
| F05 | 不許可通知書 対処 | 期限・オプション |
| F06 | 不許可後 滞在期間 | 出国期限の誤解 |
| F07 | 不許可後 再申請可否 | 再申請の誤解 |
| F08 | 不許可通知書 期限の意味 | 出国期限の誤解（P0）|
| E07 | 永住不許可後 再申請 | タイミング誤解 |
| G05 | 在留カード期限近い 待ち | 特例期間誤解 |
| I05 | 更新申請中 帰国影響 | 不在中の不許可 |

### DOMAIN 語義観察

**核心模式**：「不許可 = 即帰国」または「不許可でも引き続き住める」という二つの誤解が両方存在する。特に F08 は期限の意味を誤解すると最悪シナリオ（不法在留）につながる。

**最重要 must_not_assert**：
- 「特例期間があります」（不許可後は特例期間が不適用の場合が多い）
- 「すぐ帰国しなければなりません」（再申請・審査請求のオプションを閉じる）
- 「再申請すれば通ります」

## RP-08 — 出境再入国リスク

> 出国・帰国のタイミングと方法の誤解により、在留資格を失うパターン。

**優先度**：MEDIUM → 実害は HIGH  
**与 R05 の関係**：直接対応。

### 覆盖題目（7 題）

| starter_tag | 題目 | リスク |
|---|---|---|
| I04 | 更新申請中 出境 | みなし再入国・不在リスク |
| I05 | 更新申請中 帰国 影響 | 不在中の不許可 |
| I06 | 1年以上出境 在留 | みなし再入国期限超過 |
| I08 | 清算なし 帰国 | 在留取消（P0）|
| I03 | みなし再入国 仕組み | 1年制限の誤解 |
| I10 | 非再入国出境 意味 | 資格失効 |
| B07 | 経管 放弃回国 | 清算・帰国（P0）|

### DOMAIN 語義観察

**核心模式**：「帰国すること」と「在留資格を維持したまま帰国すること」の違いをユーザーが理解していない。

**must_surface 候選（RP-08 通用）**：
- みなし再入国の1年制限と在留期限の関係
- 更新申請中の出国リスク（不在中の不許可）
- 経管签の場合は清算状況の確認が先決

## RP-09 — 届出漏れ

> 転職・離婚・引越しなどの生活変化後に、法定の届出義務を果たしていないパターン。

**優先度**：MEDIUM  
**与 R01-R05 の関係**：R04（転職届出）に部分的に対応。

### 覆盖題目（8 題）

| starter_tag | 題目 | リスク |
|---|---|---|
| C01 | 人文 転職後 通知 | 14日届出義務 |
| C08 | 辞職後 就職期間 | 離職届出義務 |
| J05 | 離婚 届出未了 | 14日・6ヶ月 |
| G01 | 在留カード 住所不一致 | 14日届出義務 |
| G02 | 引越 両方通知 | 区役所届出 |
| G03 | 在留カード住所 期限 | 14日法定期限 |
| B01 | 経管 事務所移転 | 法務局+入管 |
| B08 | 経管 代表変更 | 法務局+入管 |

### DOMAIN 語義観察

**核心模式**：「届出が義務」であることを多くのユーザーが知らない。「転職は自由、届出は任意」という誤解が特に多い。

**must_surface 候選（RP-09 通用）**：
- 転職・離婚・引越し後の届出義務と期限（多くは14日以内）
- 届出未了は罰則の対象になりうる

## RP-10 — 複合高リスク（ユーザー自知リスク低）

> ユーザー自身がリスクの存在に気づいていない、または気づいていても問題の深刻さを認識していない複合シナリオ。TEBIQ の最重要バリュープロポジション。

**優先度**：MEDIUM（TEBIQ 活用価値は HIGH）

### 覆盖題目（6 題）

| starter_tag | 題目 | リスク |
|---|---|---|
| J01 | 何の手続きか分からない | 隠れリスクの発見 |
| J10 | 手続き漏れ 確認したい | 複合漏れ確認 |
| J06 | 会社ほぼ経営なし | 経管不実態（P0）|
| J02 | 入管からの手紙 | 緊急通知の可能性 |
| D09 | 換签 家族影響 | 連帯リスク未認識 |
| J09 | 税金年金問題 永住 | 複合問題の深刻さ |

### DOMAIN 語義観察

**核心模式**：TEBIQ の最大の価値は「ユーザーが聞いていないが知る必要があること」を伝えること。RP-10 は TEBIQ の存在意義を最も体現するパターン。

**TEBIQ 設計への含意**：
- RP-10 のシナリオでは、ユーザーが提示した情報から隠れたリスクを推論する能力が必要
- 「聞かれたことだけ答える」では不十分
- 複数のリスク（届出漏れ × 期限 × 公的義務）が同時発生している可能性を想定
