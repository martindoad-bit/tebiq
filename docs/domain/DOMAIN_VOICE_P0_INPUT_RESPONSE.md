## D-P0-01 Scope Bridge List

**input_id**: D-P0-01

**DOMAIN answer**:

以下 6 条是经过 DOMAIN 分析确认的 Scope Bridge，即"看起来不在 TEBIQ 直接支持范围内，实际上可以桥接到在留风险路径"的场景。

| Bridge | source_pattern | 桥接方向 | 桥接依据 |
|---|---|---|---|
| SB-01 | 配偶签 + 离婚 | → 在留資格変更（定住者 / 身份変更）| 离婚导致配偶者としての在留根拠喪失，6ヶ月期限发生 |
| SB-02 | 公司清算 / 放弃 / 回国（経管） | → 経営管理在留リスク（清算義務）| 経管签持有者の「帰国」は複合的在留義務を伴う |
| SB-03 | 税金 / 年金 + 永住 | → 永住申請要件リスク | 公的義務遵守は永住の直接要件 |
| SB-04 | 工作内容变化 / 解雇 | → 技人国活動範囲リスク（不法就労）| 活動範囲外業務は刑事罰対象 |
| SB-05 | 家人受影響 / 换签 | → 家族滞在連帯リスク | 家族滞在は主資格に依存 |
| SB-06 | 入管 / 区役所からの日文通知 | → 補材料 / 不許可 / 期限リスク | 通知の種類で緊急度が全く異なる |

**allowed_voice_direction**:
- 「この状況には在留手続きに関わるリスクが含まれている可能性があります」
- 「まず[具体的な情報]を確認する必要があります」
- 桥接後のドメインで必要な情報収集を促す表現

**forbidden_voice_direction**:
- 「この問題は TEBIQ の対象外です」（SB 条件に合致する場合）
- 桥接可能なのに `out_of_scope` として返す
- 桥接先の結論を確認なしに提示する（例：「定住者になれます」）

**human_review_level**: SB-01, SB-02, SB-06 → L2。SB-03, SB-04, SB-05 → conditional

**example_user_copy_direction**:
- SB-01: 「離婚後の在留資格の手続きについて、確認が必要な事項があります」
- SB-02: 「経営管理資格の方が帰国を検討される場合、会社の状況の確認が先決です」
- SB-06: 「入管からの通知は種類によって対応が異なります。内容を確認しましょう」

**uncertainty**: 桥接の適用はユーザーの具体的な状況に依存する。VOICE は桥接の方向を示すが、结论は出さない。

**needs_project_lead_decision**: no（Scope Bridge の原則は既に製品方針として確認済み）

## D-P0-03 P0 強提示使用条件

**input_id**: D-P0-03

**DOMAIN answer**:

「強提示」= ユーザーに不可逆的な後果を明示する表現。以下の条件を満たす場合のみ使用を許可する。

### 強提示が許可されるシナリオと、必ず言及すべき具体的後果

| # | シナリオ | 許可される強提示の方向 | 必ず言及すべき具体的後果 |
|---|---|---|---|
| SW-01 | 不許可通知書 期限が迫っている | 「通知書の日付を超えると在留資格が取り消される可能性があります」| 在留資格取消 → 再入国不可 |
| SW-02 | 補材料期限 超過または直前 | 「期限を超えると申請が不許可になるリスクがあります」| 補材料未提出 → 不許可 |
| SW-03 | 在留期限まで1ヶ月以内（未申請） | 「在留期限を超えると在留資格が失効します」| 在留失効 → 不法在留 → 強制退去 |
| SW-04 | 経管 清算なしの離日 | 「清算手続きをせずに出国すると、在留資格が取り消される可能性があります」| 在留取消 → 将来の日本入国に影響 |
| SW-05 | 不法就労が確認または強く疑われる | 「在留資格の活動範囲外の就労は不法就労となり、刑事罰の対象になる可能性があります」| 刑事罰（懲役・罰金）+ 在留取消 |
| SW-06 | 配偶離婚後 変更期限超過 | 「在留資格の根拠となる婚姻関係が終了した後、一定期間内に変更手続きをしない場合、在留取消のリスクがあります」| 在留取消 → 強制退去 |
| SW-07 | みなし再入国 1年超えての出国 | 「1年を超えて出国した場合、在留資格が失効する可能性があります」| 在留失効 → 帰国後の再入国不可 |

### 強提示が禁止されるシナリオ

| 禁止シナリオ | 理由 |
|---|---|
| 後果が確定していない状況での強提示 | 个别情况异なる。断言は誤情報になりうる |
| L1 シナリオへの L2 級強提示 | 过剰 escalation。ユーザーを不必要に驚かせる |
| none シナリオへの任意の強提示 | 過剰 handoff。TEBIQ の信頼性を損なう |

**allowed_voice_direction**:
- 上記テーブルの許可された後果を「可能性があります」「リスクがあります」の形で伝える
- 不確実性を保持した強提示（「〜の可能性があります」）

**forbidden_voice_direction**:
- 「必ず〜になります」「確実に〜です」（確定的な後果断言）
- 「すぐ帰国してください」（法的オプションを閉じる表現）
- 「諦めるしかありません」

**human_review_level**: 強提示を使う全シナリオは L2

**example_user_copy_direction**:
- 「補材料の期限が過ぎると申請が不許可になるリスクがあります。今すぐ入管か専門家に連絡してください」
- 「清算手続きをせずに出国すると、在留資格の取り消しリスクがあります。手続きの詳細を専門家に確認することをお勧めします」

**uncertainty**: 具体的な後果の確定は個別事情に依存する。「可能性があります」「リスクがあります」の形式を維持すること。

**needs_project_lead_decision**: yes（「強提示」のトーンと文体の最終承認は産品負責人・VOICE が協議の上決定）

## D-P0-04 不許可通知書の表達边界

**input_id**: D-P0-04

**DOMAIN answer**:

不許可通知書は TEBIQ の中で最も高リスクな出力シナリオの一つ。以下の境界は厳格に適用すること。

### 許可される表現

| 表現の方向 | 許可理由 |
|---|---|
| 「通知書に記載された日付を確認してください」| 事実確認を促す。断言なし |
| 「その日付が出国期限の可能性があります」| 「可能性」を保持した情報提供 |
| 「再申請・審査請求の選択肢がある場合があります」| 法的オプションの存在を示す。確約なし |
| 「専門家（行政書士等）への早急な相談をお勧めします」| 適切な handoff |
| 「処理窓口」という表現 → **条件付きで可** | 詳細は下記参照 |

### 「処理窓口」という表現について

**DOMAIN 判断**：**条件付きで使用可**。

- ✅ 許可：「対応できる処理窓口に早急に相談することをお勧めします」（専門家・行政書士を指す文脈で）
- ✅ 許可：「入管の処理窓口に直接問い合わせることも選択肢です」
- ❌ 不可：「処理窓口で解決できます」（結果を保証する文脈）
- ❌ 不可：「処理窓口」が「TEBIQ 内の機能」を指すと誤解される文脈

**理由**：「処理窓口」は行政・専門家を指す日本語として自然。ただし「処理できます」という結果の保証とセットになると誤情報になりうる。

### 絶対に禁止する表現

| 禁止表現 | 禁止理由 |
|---|---|
| 「再申請すれば通ります」| 結果の保証。不可能な断言 |
| 「すぐ帰国してください」| 法的オプション（審査請求等）を閉じる |
| 「特例期間があります」（不許可後に）| 誤情報。特例期間は更新申請「中」のみ |
| 「諦めるしかありません」| 法的オプションを否定する |
| 「まだ余裕があります」（日付確認なしに）| 確認なしの安心感付与 |
| 「大丈夫です」| 根拠のない安心感 |
| 不許可の理由について断言する | 通知書を見ていない状態での断言 |

### 出国期限と在留期限の混同に関する注意

- 不許可通知書に記載の日付は「出国期限」であり「在留期限」ではない
- この区別を誤解させる表現は P0 失敗とする
- VOICE が期限について言及する場合は「通知書に記載された日付」と表現し、その意味の解釈はユーザーと専門家に委ねること

**allowed_voice_direction**:（上記「許可される表現」参照）

**forbidden_voice_direction**:（上記「禁止表現」参照）

**human_review_level**: L2（必須）

**example_user_copy_direction**:
「不許可通知書を受け取られたとのこと、まず通知書に記載された日付を確認してください。その日付によっては早急な対応が必要な場合があります。再申請や審査請求などの選択肢が存在する場合もありますので、行政書士などの専門家への相談を今すぐ検討されることをお勧めします」

**uncertainty**: 通知書の実物を見ていないため、期限・内容・選択肢の詳細について断言不可。

**needs_project_lead_decision**: yes（「処理窓口」の使用可否の最終確認と、不許可通知書シナリオの文体指針の確認）

### 修正後もルーティングが失敗した場合の VOICE の動作原則

1. `status = out_of_scope` が返ってきても、VOICE は「この問題は対象外です」と告げない
2. 代わりに「もう少し状況を教えてください」と clarification を促す
3. ユーザーが SB-01〜SB-06 のいずれかのパターンを示している場合、VOICE は桥接方向を示す
4. VOICE が対応しきれないと判断した場合 → L2 human review direction を適用

**allowed_voice_direction**:
- 「もう少し状況を教えていただけますか」（routing failure 時の bridge）
- Scope Bridge に沿った再引導

**forbidden_voice_direction**:
- 「この問題は TEBIQ では対応できません」（out_of_scope をそのままユーザーに返す）
- ルーティング失敗を理由に回答を放棄する

**human_review_level**: routing failure が連続する場合 → L2（GM に報告）

**example_user_copy_direction**:
- out_of_scope 返却時: 「状況をもう少し詳しく教えていただけますか。[clarification_direction に沿った質問]」

**uncertainty**: R01-R05 修正後の actual routing は ENGINE 実装の結果次第。VOICE は修正後の routing に依存するが、fallback として Scope Bridge の方向は常に参照できる。

**needs_project_lead_decision**: no（routing failure 時の VOICE の bridge 動作は製品方針として確認済み）
