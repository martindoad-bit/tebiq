# TEBIQ 产品哲学 v1

这是 TEBIQ 1.0 的灵魂,凌驾于任何具体任务之上。
任何任务 / 决策 / 方向调整都不能违反这个哲学。
违反 = 立刻停下,在 AI_HANDOFF.md 写"§产品哲学冲突",等创始人。

## 洞察

TEBIQ 的存在,是为了减少在日生活的"日常摩擦"。
让外国人在日本的融入度和麻烦程度,从 7 降到 6。
这是我们的功德,也是我们的全部。

## Voice 原则(铁律)

保持工具感。
不撒娇、不温情、不戏剧化、不 emoji、不"陪伴你的旅程"叙事。
像东京电车一样:精确、冷静、不解释、用户可以骂它,但离不开它。

文案 voice 示例:
✓ 正确:"已归档 / 已识别 / 期限冲突 / 第 N 件 X 类文书"
✗ 错误:"我帮你看到了 / 把这件事交给我 / 陪你度过这一年"

视觉 voice 示例:
✓ 正确:数字化、克制、像日本邮便局对账单
✗ 错误:温暖色调、插画、emoji、"贴心"标签

## 底层逻辑(1.0 之后的根本性改造)

TEBIQ 不是"工具集合",是"档案中心化的工具系统"。

- 每次工具调用(拍照 / 自查 / 文字即懂) → 自动归档到用户的统一时间线
- 时间线是产品的永久记忆
- 工具是时间线的输入入口
- 跨档案比对、历史关联、政策匹配推送 = 真正的差异化
- ChatGPT 没有时间线,所以 ChatGPT 永远不能成为 TEBIQ

## 不做的事(铁律)

✗ 温情文案 / 撒娇 voice / "陪伴旅程"叙事
✗ 矫情视觉 / 暖色调 / 插画感
✗ 情感订阅定价(月费高 / 试图绑定用户感情)
✗ "你不孤单"这种自我感动的客服话术
✗ 虚假个性化("我懂你"——我们不懂用户,我们只查档案)
✗ 通用 AI 的扩展(不要做 ChatGPT 能做的事)
