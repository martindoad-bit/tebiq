### 核心一句话

```

```

### 内容结构（10 节）

1. TEBIQ 一句话（核心金句 + 不许变体）
2. 不说什么（4 大类禁用词清单：AI / 营销承诺 / 情感戏剧化 / 通用 AI 词）

6. 文案 voice 铁律（工具感 / 不替决定 / 不戏剧化 / 不温情）
7. customer_message 规范（≤150 字 + 微信语气 + 1+1+1 结构）
8. 首页文案规范（3 版方向 + 安全边界文案）
9. 高风险议题（不法滞在 / DV / 雇主雇错 / 永住带父母）

### 版本 A：情况入口（推荐默认）

```

```

- 工具入口在 hero 之下，不抢主位

**A/B 测试候选**：
- 版本 A vs 版本 C — funnel 比对
- 版本 A vs 版本 B — 输入意愿比对

## WP4：Q032 办公室搬迁特别重写

### 8 点全覆盖（按 brief）

| brief 要求 | customer_facing_answer 写入字段 |
|---|---|
| 1. 新办公室是否可用于法人办公和业务使用 | `do_today` + `applies_when` |
| 2. 合同是否公司名义 + 说明使用目的 | `do_today` |
| 3. 本店所在地变更 → 法务局登记 | `where_to_go` + `documents` |
| 4. 税务署需要做异动届 | `where_to_go` |
| 5. 社保 / 银行 / 许可 / 合同等地址也要更新 | `where_to_go` |
| 6. 入管续签时要能说明经营场所持续存在 | `consequences` + `customer_message` |
| 7. 保留新旧租约 / 照片 / 平面图 / 地址变更证明 | `documents` |
| 8. 只改邮寄地址 → 续签时材料不一致 | `consequences` + `expert_trigger` |

### 关键时间节点（Q032）

- 法務局 本店移転登記 **2 週間以内**
- 入管 所属機関等届出 **14 日以内**
- 税務 / 社保 / 雇用保険 異動届 **1 ヶ月以内**

### customer_message（Q032 转发版）

> 经营管理签的办公室搬迁不是「改个收件地址」就完事 — 涉及法务局 商業登記 + 入管届出 + 税务異動届 + 社保 / 银行 / 许可同步更新。今天先确认 3 件事：新办公室是否可用于法人办公 / 合同是否公司名义 / 是否本店所在地变更。其中法务局 2 週間内 + 入管 14 日内是硬期限。这种情况建议先和行政書士聊一下，避免登记地址 / 实际地址 / 申告地址不一致影响下次续签。

### 高风险题统一规则（已遵守）

- `conclusion`：必须以「这个问题不能简单回答」开头
- `do_today`：用户**现在**能先确认什么（具体 1-3 件）
- `where_to_go`：写「先不要直接申请。优先咨询：[专家]」
- `expert_trigger` 必须列 3 个具体触发条件，**不要只写「建议咨询专家」**
- `customer_message`：含「这个情况建议先和 [专家] 聊一下」+ 1 句先做什么

### 特殊处理

- **Q099**：严格区分 出国命令 / 退去強制 / 在留特別許可 三独立路径
- **Q091**：严格区分 雇主刑责（不法就労助長罪）vs 雇员处分（資格外活動 → 退去強制）
- **Q065**：制度路径（DV 防止法 + 入管特例措施）+ DV センター #8008 + 警察 #9110，不情感化（无「勇敢 / 保护自己」等情感化词）
- **Q024**：misconception 类，conclusion 客观陈述「不立即取消，但有 14 日 + 3 月两条死线」

## 语气合规检测

### 全文 grep 检测结果

✗ **全部不出现**：
- 保证（肯定式）/ 一定（肯定式）/ 绝对成功 / 100% 通过 / 拒签概率
- 赶紧委托 / 我们可以帮你办 / 立即委托
- 焦虑 / 困境 / 让 X 变简单 / 我们陪你 / 你不孤单
- 问 AI / 智能判断 / 秒懂 / 风险诊断 / 一键解决

✓ **频繁使用**：
- 通常 / 一般 / 多数情况下
- 需要先确认 / 需要先整理 / 建议先 / 一般会看
- 「这个情况建议先和 [专家] 聊一下」
- 客观时间和金额（取代戏剧化）

### 否定式合法用法（不算违规）

「无法保证」「不保证一定能 X」「非绝对禁止」「没有绝对安全数字」 — 都是「不 / 非 / 无法」开头的否定，符合 brief 精神（不在欺骗用户保证什么）。
