# TEBIQ App 文案准则（APP_COPY_FINAL_V2）

**版本**：v2 final
**生成**：2026-04-30

> TEBIQ 是 App，不是内容页。
> App 首页不需要画面感句子，不需要长 H1，不需要解释型文案。
> 文案目标不是「好听」，而是「少、准、能操作」。

本规范取代 `PRODUCT_COPY_BIBLE.md` 的部分文案方向（保留 voice 铁律 + 中日混合规则 + 高风险题处理）。冲突时本文件胜出。

---

## 核心判断规则

写每句话前问自己：

1. 微信会这样写吗？
2. 支付宝会这样写吗？
3. ChatGPT 首页会这样写吗？
4. Linear 会这样写吗？

**不会，就删掉。**

| ✗ 不写 |

| 情况入口 |
| 写一句你的情况 |

### AI / 营销

| ✗ 不写 |

### 替代表达

| 旧表达 | 新表达 |
|---|---|
| 这个问题不能简单回答 | 先看这几个条件 |
| 这部分需要进一步确认 | 还要确认：身份、日期、是否逾期、是否收到通知 |
| 建议咨询专家 | 出现以下情况时，找行政书士/社労士确认 |
| 结论 | 最紧的两件 |
| 今天先做 | 步骤 |
| 完整步骤 | 步骤 |
| 使用边界 | 来源与说明 |

## 文案 voice（保留）

工具感 voice：精确、冷静、不解释、不温情。

✓ 「已归档」「已识别」「期限冲突」
✗ 「我帮你看到了」「把这件事交给我」

---

## 中日混合（保留）

在留期間更新 / 在留資格 / 在留カード / 永住 / 経営・管理 /
住民税 / 国民年金 / 厚生年金 / 健康保険 / 確定申告 /
等保留日文。
