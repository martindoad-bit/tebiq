# TEBIQ 产品文案规范（PRODUCT_COPY_BIBLE）

**版本**：v1
**生成**：2026-04-30
**适用范围**：所有 TEBIQ 产品文案 — 首页 / answer / decision card / scenario card / document card / 产品功能 UI / 营销 / 客服话术 / 客户经理转发版

任何 TEBIQ 产品文案出场前先对照本规范。规范与个人审美冲突时，**规范胜出**。

---

## 一、TEBIQ 一句话

```
在日本遇到手续问题，TEBIQ 帮你整理下一步。
```

### 拆解

- **「在日本」** — 立位
- **「遇到手续问题」** — 用户场景（不是「焦虑」「困惑」「迷茫」等情感化词）
- **「TEBIQ 帮你」** — 主体（不是「智能判断」「AI 诊断」等技术黑话）
- **「整理」** — 动作（不是「解决」「搞定」「秒懂」等承诺）
- **「下一步」** — 输出（不是「答案」「攻略」「方案」等过度承诺）

### 不许变体

- ✗ 在日本遇到麻烦 / 困境 / 难题（"麻烦"过情绪化，"困境"过戏剧化）
- ✗ TEBIQ 智能判断你的情况
- ✗ TEBIQ 帮你解决日本手续
- ✗ 一键搞定日本生活
- ✗ TEBIQ 让在日生活变简单

### 衍生句式

- 简版：「整理下一步」
- 输入框句：「把情况说清楚」
- 工具入口句：「先整理 → 再办理」

## 二、不说什么（禁用清单）

### 2.1 AI / 智能 类（不说）

| ✗ 不说 | 为什么 |
|---|---|
| 问 AI | 用户已被 ChatGPT 教育，"问 AI" 没有差异化 |
| 智能判断 | 暗示「AI 替你做行政决定」，违反产品哲学 |
| 秒懂 | 时间承诺过强，且暗示用户智商不足 |
| 风险诊断 | "诊断" 是医疗 / 律师术语，越界 |
| AI 助理 / 智能助手 | 通用 AI 标签，无差异 |
| 让 AI 帮你 | 暗示替用户决策 |

### 2.2 营销 / 承诺 类（不说）

| ✗ 不说 | 为什么 |
|---|---|
| 拒签概率 | 法律不允许量化预测 + 加重用户焦虑 |
| 一定通过 | 法律意见禁止承诺 |
| 一键解决 | 行政手续不是一键能完成 |
| 专家马上回复 | 时间承诺无法保证 |
| 包过 / 包通过 | 违法（行政手续无法担保） |
| 100% / 成功率 X% | 量化承诺违法 |
| 赶紧委托 / 立即委托 | 营销施压 |
| 我们可以帮你办 | 越界（TEBIQ 是工具不是行政書士） |

### 2.3 情感化 / 戏剧化 类（不说）

| ✗ 不说 | 为什么 |
|---|---|
| 焦虑 / 困境 / 迷茫 / 痛苦 / 绝望 | 不替用户情绪化 |
| 让 X 变简单 / 让 X 不再难 | 承诺过强 |
| 我们陪你 / 我们一直在 | 温情话术，违反 voice |
| 你不孤单 | 自我感动 |
| 走出迷茫 / 重新开始 | 戏剧化叙事 |
| 干货满满 / 必看 / 收藏 | 小红书化 |

### 2.4 通用 AI 词（不说）

「让我们一起来看看」「值得我们深思」「不容忽视」「在当今 X 的背景下」「接下来我们将探讨」「综上所述」「由此可见」

## 三、要说什么（推荐词典）

### 3.1 动作词（推荐）

| ✓ 要说 | 用法 |
|---|---|
| **整理下一步** | 核心动作动词 |
| **准备材料** | 替代「搞定材料」 |
| **确认期限** | 替代「赶紧 / 立即」 |
| **办理窗口** | 替代「跑腿 / 折腾」 |
| **核对** | 替代「检查」 |
| **归档** | 档案中心化语境 |

### 3.2 边界词（推荐）

| ✓ 要说 | 用法 |
|---|---|
| **通常** | 取代「一般」时更克制 |
| **多数情况下** | 取代「都」「绝对」 |
| **需要先确认** | 不知道时不要瞎答 |
| **需要先整理** | 鼓励用户准备 |
| **如果涉及个别事实** | 边界声明 |
| **建议咨询专业人士** | 不替专家越界 |
| **需要专家确认的情况** | 引向专家而非替代 |

### 3.3 概率 / 时间词（推荐）

| ✓ 要说 | 替代 |
|---|---|
| **一般会看 [X 年]** | 「至少 X 年」/ 「必须 X 年」 |
| **目安 [金额]** | 「最低 X 円」/ 「门槛 X 円」 |
| **14 日内 / 30 日内** | 客观数字 |
| **续签前 3-6 个月** | 客观时间窗口 |

## 四、答案固定结构（所有 answer / decision card / scenario card 必须遵守）

### 4.1 8 字段结构（必填）

```yaml
1. 一句话结论    (conclusion)         一句话明确答案 / 高风险题写「这个问题不能简单回答」
2. 适用条件      (applies_when)       谁该看 / 不适用谁
3. 今天先做什么  (do_today)            1-3 件具体可执行的事
4. 去哪办理      (where_to_go)         机关名 + 具体窗口
5. 要带什么      (documents)           证件 / 资料清单
6. 期限          (timing)              具体天数 / 时间窗口
7. 不做后果      (consequences)        客观陈述时间梯度
8. 需要谁确认    (expert_trigger)      什么情况找哪种专家
```

### 4.2 高风险题特殊结构

不能简单回答的题（永住带父母 / 不法滞在 / 雇错签证 / 等）：

- `conclusion` 必须是「这个问题不能简单回答」+ 1 句主要原因
- `do_today` 必须是 1-3 件**用户现在能先确认**的事（不是泛泛建议）
- `where_to_go` 写「先不要直接申请。优先咨询：[具体专家]」
- `expert_trigger` 必须列 3 个具体触发条件，**不要只写「建议咨询专家」**

```
✗ 不要：建议咨询行政書士
✓ 要写：如果出现以下任一情况，建议找 [行政書士 / 弁護士]：
        1. 婚姻不满 3 年 + 没有日本籍子女
        2. 离婚后 6 ヶ月内未办理在留資格変更
        3. DV / 死别 等不可抗力情况
```

### 4.3 客户经理转发版（customer_message）

≤ 150 字，让客户经理可以直接复制粘贴发给客户。

风格：
- 微信对话语气
- 1 句结论 + 1 句先做什么 + 1 句必要时咨询专家
- 不写「我们」「我帮你」（避免承诺）
- 中日混合保留

例（永住直近 5 年纳税）：
```
永住申请要看你过去 5 年的纳税情况：住民税 / 国保 / 国民年金 都不能滞纳。
今天先去市役所拿直近 5 年课税 + 纳税证明，对照每年是否完整。
如果有滞纳记录，先补缴 + 取得完納証明書 再申请。情况复杂建议先找行政書士确认。
```

## 五、中日混合规则

### 5.1 必须保留日文原文

**签证类型**：技術・人文知識・国際業務 / 経営・管理 / 家族滞在 / 特定技能 / 定住者 / 永住者 / 高度専門職 / 留学 / 日本人配偶者等

**政府机构**：入国管理局（入管）/ 出入国在留管理庁 / 市役所 / 区役所 / 税務署 / 国税庁 / 年金事務所 / ハローワーク / 法務局 / 家庭裁判所

**政府文件**：在留カード / 在留資格認定証明書（COE）/ 住民票 / 確定申告書 / 源泉徴収票 / 納税証明書 / 課税証明書 / 戸籍謄本 / 健康保険証 / 雇用契約書

**专业概念**：住民税 / 所得税 / 健康保険 / 厚生年金 / 国民年金 / 国民健康保険 / 介護保険 / 雇用保険 / 源泉徴収 / 年末調整 / 標準報酬月額 / 高額療養費

**生活场景**：敷金 / 礼金 / 保証人 / 引っ越し / 確定申告 / 賃貸 / 転入届 / 転出届 / 再入国許可 / みなし再入国

**入管手续**：在留期間更新 / 在留資格変更 / 資格外活動許可 / 特例期間 / 不許可 / 補正 / 退去強制 / 出国命令 / 上陸拒否期間 / 不法滞在 / 不法就労

### 5.2 翻译成中文

普通动词（提交、准备、查看、办理、申请）/ UI 操作（取消、确认、返回）/ 应用层概念（会员、订阅、邀请）/ 解释性内容（风险等级、判断逻辑、行动建议）

## 六、文案 voice（铁律）

### 6.1 工具感 voice

像东京电车一样：精确、冷静、不解释、用户可以骂它、但离不开它。

✓ **正确**：
- 「已归档」
- 「已识别」
- 「期限冲突」
- 「第 N 件 X 类文书」

✗ **错误**：
- 「我帮你看到了」
- 「把这件事交给我」
- 「陪你度过这一年」

### 6.2 不替用户做行政决定

TEBIQ 给制度 + 一般动作 + 复杂时引向专家。
TEBIQ 不说「你应该选 X」「我建议你 Y」。

例：
- ✗ 「你应该立即申请永住」
- ✓ 「永住申请条件 + 今天先做的事 + 复杂时找行政書士」

### 6.3 不戏剧化时间和金额

- ✗ 「立即」「必须」「严重」「不可逆」「绝对」
- ✓ 客观数字：「14 日内」「3000 万円」「2 年内」

### 6.4 不撒娇 / 不温情

- ✗ 「你不孤单」「TEBIQ 永远在你身边」「我懂你」
- ✓ 「档案永久保留 ¥1,580/年」「在留期間更新 还有 60 日」

## 七、客户经理转发版规范（customer_message）

### 7.1 字数

≤ 150 字（中文计）

### 7.2 结构

```
[1 句结论] + [1 句先做什么] + [1 句必要时咨询专家]
```

### 7.3 语气

- 微信对话风格（对话式句子，不写营销 / 标题党）
- 客户经理是中介专家而非营销，转发的文字必须**让客户经理觉得**「这话我能签字」
- 不要写「我们」「我帮你」 — 客户经理不能替 TEBIQ 承诺

### 7.4 不写

- 「赶紧」「立即」「不能等」
- 「不做就来不及了」
- 「TEBIQ 现在限时 X」
- 营销 CTA（「现在就咨询」「立即委托」）

### 7.5 写

- 客观时间和金额
- 必要时一句「情况复杂建议先和行政書士聊一下」
- 中日混合保留

## 九、特别注意：高风险议题

### 9.1 不法滞在 / 不法就労

- 严格走制度路径，不评价道德
- 区分 出国命令 vs 退去強制 vs 在留特別許可（三独立路径）
- 必须 弁護士 + 行政書士 联合 + 出頭前同行

### 9.2 DV / 家暴

- 严格走制度路径（DV 防止法 + 入管特例措施）
- 不情感化（不写「勇敢」「保护自己」）
- 制度引导：DV センター（#8008）+ 警察（#9110）+ 弁護士 + 行政書士

### 9.3 雇主雇错签证

- 严格区分雇主刑责（不法就労助長罪）vs 雇员处分（資格外活動 → 退去強制）
- 雇员当事人立即 弁護士 + 行政書士
- 不引导自责

### 9.4 永住带父母 / 永住者親族

- misconception 类，必须先纠偏「家族滞在不含父母」
- 写真实 3 路径（短期滞在 90 日 / 特定活動 老親扶養 / 高度専門職 親族帯同）
- 不写温情承诺
