## 给 CCA 的接入建议

### 前端渲染层做字段映射

不需要改 yaml schema 字段名。前端渲染时映射：

```python
SECTION_MAPPING = {
    "conclusion": "最紧的两件",
    "do_now": "步骤",
    "how_to_do": "步骤",
    "documents_needed": "要带什么",
    "documents": "要带什么",
    "deadline_or_timing": "期限",
    "timing": "期限",
    "consequences": "不做会怎样",
    "expert_handoff": "要找专家的情况",
    "expert_trigger": "要找专家的情况",
    "source_hint": "来源与说明",
    "customer_message": None,  # 底部按钮「复制给客户」
}
```

### 高风险题处理

`answer_type: needs_expert / misconception` 题：
- 不再写「这个问题不能简单回答」（已全部替换为「先看这几个条件」）
- 「最紧的两件」section 第 1 项写「先看这几个条件：[列条件]」
- 「最紧的两件」section 第 2 项写「出现以下情况时，找 [专家] 确认：[列触发]」

### 复制给客户按钮

底部按钮，调 `customer_message` 字段。每条 ≤150 字微信语气。

### 首页 placeholder 轮播

UI 实现可考虑 5 个备选 placeholder 轮播（每 5 秒切换一次），增加首屏多样性。
