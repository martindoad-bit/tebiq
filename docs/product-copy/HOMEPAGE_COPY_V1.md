## 版本 A：情况入口（推荐默认）

### 方向

让用户**先把情况说清楚**，TEBIQ 整理下一步。

最克制，最贴在日中文用户的实际心智（用户不会主动说「我想问 AI」，而会说「我现在的情况是这样的...」）。

### 文案

```yaml
hero_title: 在日本遇到手续问题，先把情况说清楚。
subtitle: TEBIQ 整理下一步：办什么、去哪办、要带什么、什么时候办。

input_label: 描述你现在的情况
placeholder: 例：「日配婚姻 2 年离婚，有 1 岁小孩，签证还有 5 个月到期。」

popular_chips:
- 换工作了 14 天没办届出
- 永住申请前换了工作
- 经管签证收到改正通知
- 配偶签证刚离婚
- 收到一封日文催缴信
- 公司倒闭了我的签证怎么办

tool_entries:
- title: 自查续签条件
  desc: 按签证 + 维度逐项核对
- title: 拍照识别文书
  desc: 寄件机关 + 期限 + 处理路径
- title: 找专家咨询
  desc: 行政書士 / 弁護士 / 税理士 / 社労士

footer_safety: |
  TEBIQ 不替代行政書士 / 弁護士 / 税理士 / 社労士。
  具体个案请咨询专业人士。本页内容根据公式信源整理，可能因法令改正变更。
```

### 配套微文案（建议）

- 用户输入后等待状态：「整理中。先看你需要确认什么。」
- 用户问题已识别：「这个情况通常涉及 3 件事。」
- 复杂情况引导：「这个情况建议先和行政書士聊一下。」
- 收藏 / 归档按钮：「归档到我的时间线」

## 版本 B：下一步导向

### 方向

强调 TEBIQ 是「行动整理工具」— 不知道下一步做什么时来。

适合**已经知道自己要办手续，但不知道顺序 / 期限 / 材料**的用户。

### 文案

```yaml
hero_title: 不知道下一步做什么时，先来 TEBIQ 整理。
subtitle: 把日文文书 + 行政手续 + 期限 整理成可执行清单。

input_label: 现在卡在哪一步？
placeholder: 例：「换了工作 14 天到了，不知道届出该怎么交。」

popular_chips:
- 收到 督促状 不知道怎么办
- 在留期間更新 申请被退回了
- 改正通知 让我补材料
- 公司搬迁 要办什么手续
- 离职后 国保社保 怎么切换
- 第一次 確定申告 不知道从哪开始

tool_entries:
- title: 续签自查
  desc: 60+ 个维度按签证类型对照
- title: 文书识别
  desc: 50+ 类常见文书 + 处理路径
- title: 专家咨询
  desc: 复杂个案 → 行政書士 / 弁護士

footer_safety: |
  TEBIQ 不替代专业人士。本页内容仅供整理参考。
  具体个案 + 在留資格 个別审查 = 必须咨询行政書士 / 弁護士。
  内容根据公式信源整理，可能因法令改正变更。
```

### 配套微文案（建议）

- 步骤导引提示：「按顺序做：1 / 2 / 3」
- 期限提示：「14 日内」 / 「3 ヶ月内」 / 「续签前 6 ヶ月」
- 材料未齐：「还差 X 份材料」
- 已完成步骤：「已归档」

## 版本 C：文书 + 手续结合

### 方向

强调 TEBIQ 的两大入口（拍照识别 + 手续整理）作为统一价值主张。

适合**已经收到日文文书 + 不确定如何处理**的用户。

### 文案

```yaml
hero_title: 日文文书和日本手续，先整理成下一步。
subtitle: 拍照识别 + 续签自查 + 行政手续 — 一起整理到你的时间线。

input_label: 拍照 / 描述都可以
placeholder: 例：上传一张催缴信照片，或描述你现在的情况。

popular_chips:
- 拍照识别 一封催缴信
- 收到 補正通知 怎么应对
- 工作签 续签材料 自查
- 永住申请 直近 5 年纳税
- 公司倒了 签证怎么办
- 离婚后 6 ヶ月内 该做什么

tool_entries:
- title: 拍照识别文书
  desc: 50 类典型文书 + 寄件机关 + 期限
- title: 自查续签 / 永住条件
  desc: 按签证 + 维度逐项核对
- title: 行政書士咨询
  desc: ¥9,800 起，复杂个案专家把关

footer_safety: |
  TEBIQ 是日本生活手续整理工具，不是行政書士 / 弁護士。
  涉及个別审查的 case → 必须咨询专业人士。
  内容根据公式信源整理，可能因法令改正变更。
```

### 配套微文案（建议）

- 拍照后识别成功：「已识别为 [文书类型]」
- 拍照后未识别：「无法识别。可以描述这封信吗？」
- 时间线归档：「已归档第 [N] 件 [类型] 文书」
- 期限提醒：「[文书] 还有 [X] 天到期」

## 推荐 CCA / codexUI 使用方式

### 默认上线：版本 A

理由：
- 最克制（无营销 CTA）
- 最贴用户心智（用户不会主动说「我要拍照」，会说「我的情况是...」）
- 输入框最自然（描述情况 = 用户日常说话方式）
- 工具入口在 hero 之下，不抢主位

### A/B 测试候选

如果创始人想测，建议：
- 版本 A vs 版本 C — 测试「情况入口」vs「拍照入口」哪个 funnel 更优
- 版本 A vs 版本 B — 测试「描述情况」vs「描述下一步」哪个用户更顺手输入

### 可交叉组合

- hero 标题用 A，工具入口用 C 的 3 个（拍照在前）
- subtitle 用 B（强调下一步整理）
- 底部安全边界 3 版统一用 A 版

## 不再讨论的方案

明确**不上**这些方向：

| ✗ 不上 | 为什么 |
|---|---|
| 「问 AI 你的日本生活问题」 | 通用 AI 标签无差异 |
| 「TEBIQ 智能判断你的情况」 | 越界 + 通用 AI 词 |
| 「在日华人专属 AI 助理」 | 营销话术 + 「助理」承诺过强 |
| 「拒签率自动评估」 | 违法 |
| 「秒懂日本生活」 | 时间承诺 + 暗示用户智商 |
| 「让 AI 帮你解决日本手续」 | 替用户决策违规 |

## 安全 / 法律边界（所有版本通用）

底部必须含：
1. 「TEBIQ 不替代行政書士 / 弁護士 / 税理士 / 社労士」
2. 「具体个案请咨询专业人士」
3. 「内容根据公式信源整理，可能因法令改正变更」

CTA 不许写：
- 「我们帮你办」
- 「赶紧委托」
- 「100% / 一定 / 保证」
