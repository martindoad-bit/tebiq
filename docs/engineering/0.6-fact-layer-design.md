## Goal

Wire the AI-first Fact Layer into production `/api/consultation/stream`:

```
user question
  → fact matcher (loads only state=human_approved cards)
  → matches → inject card.injection_format into system prompt
  → record fact_card_ids on the consultation row
  → Learning Console / /c/[id] / /me/consultations show injected cards
```

No card content authored by ENGINE. ENGINE is the runtime; content
lives in `docs/fact-cards/*.md` (filesystem) and `fact_cards` table
(runtime).

---

## State machine (gating) — AI-first (revised 2026-05-07 per PL)

Two layers of gating: **state** AND **risk_level**. Both must pass.

### State layer

| state | can match? | can inject? |
|---|---|---|
| `draft` | no | no |
| `ai_extracted` | yes (dry-run only) | no |
| `ai_verified` | yes | yes (subject to risk gate) |
| `human_reviewed` | yes | yes |
| `needs_review` | yes (only to fire conservative hint) | no facts |
| `conflict` | no | no |
| `disabled` | no | no |

### Risk layer (only applies to `ai_verified`; `human_reviewed` always injects)

| risk_level | Alpha frontend? | production frontend? |
|---|---|---|
| `low` | inject | inject |
| `medium` | inject | inject |
| `high` | inject (use card.cautious_copy if present) + enqueue for human audit | inject + enqueue |
| `critical` | inject **only if** `controlled_alpha_eligible: true` AND PL changelog signoff; otherwise NOT injected (return as conservative hint via state=needs_review fallback) | NOT injected unless `human_reviewed` OR explicit PL changelog override |

### Per-field gating within an `ai_verified` card

Cards may be `ai_verified` overall but mark specific facts as
`needs_review_flags`. The matcher splits `injection_format` into:

- `certain_block` → injected
- `needs_review_addendum` → NOT injected as facts; matcher emits a
  generic "本回答涉及若干尚未完全确认的细节，建议向行政書士確認最新規則"
  marker that the frontend may surface (Workstream G integration).

### Internal dry-run endpoint

`/api/internal/fact-layer/dry-run` (gated by `EVAL_LAB_ENABLED`):

- Accepts `{question, state_filter, risk_filter, include_dry_run_states}`
- Can load `ai_extracted` for QA preview before promotion
- Returns `{matches, certain_block, needs_review_addendum, would_inject_in_prod, gate_decisions}`
- Never reachable from the user-facing app.

## Data model

### Filesystem (canonical source-of-truth)

`docs/fact-cards/<slug>.md` with YAML frontmatter (see existing
`keiei-kanri-2025-10.md` as the canonical shape).

Editing the file in repo + PR review = the human approval gate.
Merging the PR = approval. Loader reads filesystem at boot or via
hot-reload signal.

### Runtime table (Drizzle migration `0025_fact_cards.sql`)

```sql
CREATE TABLE fact_cards (
  fact_id                     text PRIMARY KEY,
  title                       text NOT NULL,
  state                       text NOT NULL,    -- draft|ai_extracted|ai_verified|human_reviewed|needs_review|conflict|disabled
  risk_level                  text NOT NULL,    -- low|medium|high|critical
  confidence                  text NOT NULL,    -- low|medium|high  (also stored as numeric 0..1 in card body)
  source_quality              text NOT NULL,    -- official|quasi_official|secondary
  controlled_alpha_eligible   boolean NOT NULL DEFAULT false,
  applies_to                  jsonb NOT NULL,   -- string[]
  trigger_keywords            jsonb NOT NULL,   -- string[]
  injection_certain_block     text NOT NULL,    -- always-injected (when card injects at all). {{TODAY_ISO}} substituted at request time
  injection_needs_review_addendum text,         -- NOT injected as fact; surfaces hint only
  needs_review_flags          jsonb NOT NULL DEFAULT '[]'::jsonb,  -- string[] of field names withheld
  source_urls                 jsonb NOT NULL,   -- string[]
  reviewer                    text,
  last_verified_at            timestamptz NOT NULL,
  approved_at                 timestamptz,
  approved_by                 text,
  filesystem_path             text NOT NULL,    -- docs/fact-cards/<slug>.md
  content_hash                text NOT NULL,    -- sha256 for drift detection
  created_at                  timestamptz NOT NULL DEFAULT now(),
  updated_at                  timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX fact_cards_state_idx ON fact_cards(state);
CREATE INDEX fact_cards_risk_idx ON fact_cards(risk_level);
CREATE INDEX fact_cards_state_risk_idx ON fact_cards(state, risk_level);
```

### Consultation extension (Drizzle migration `0026_consultation_fact_cards.sql`)

```sql
ALTER TABLE ai_consultations
```

## Sync pipeline (filesystem ↔ DB)

`scripts/fact-layer-sync.ts`:

1. Glob `docs/fact-cards/*.md` (excl. `README.md`).
2. Parse YAML frontmatter + body.
3. Compute `content_hash`.
4. For each card, upsert into `fact_cards` table by `fact_id`.
5. Hard-fail if a card has `state: human_approved` but is missing
   any required field.
6. Hard-fail if a card declares an `official_sources[].url` outside
   the source whitelist (`README.md` declares the whitelist).
7. Run on Vercel deploy (post-build hook) and via manual CLI.

Drift detection: at boot, the production loader compares each row's
`content_hash` to the on-disk file. Mismatch → log warning + emit
metric `fact_layer.drift_detected`. (No auto-resync in production —
mismatch is a deploy bug.)

## Matcher (Workstream C0+C6 core)

`lib/answer/fact-layer/matcher.ts`:

```typescript
type Match = {
  fact_id: string;
  trigger: string;       // which keyword/phrase fired
  score: number;         // 0..1
};

export function matchFactCards(question: string): Match[];
```

**v0.6 strategy** (intentionally simple, per PL "不追求完美"):

1. Normalize question: lowercase, strip punctuation, keep CJK.
2. Candidate set: cards with state in
   `(ai_verified, human_reviewed, needs_review)`. (`needs_review` is
   matched only to fire conservative hint, no facts injected.)
3. For each candidate, scan `trigger_keywords` for substring presence.
4. Score = matched_keyword_count / total_keywords_in_card (capped at 1).
5. Filter: score >= 0.15 OR a `high|critical` card matched any keyword
   (liberal on high-risk — false positive injection is far cheaper
   than missed P0 misinformation).
6. Apply gate (state × risk_level table above) to decide
   `inject_facts | hint_only | drop`.
7. Return top 3 matches by `(risk_level rank, score)`.

Future (post-0.6, NOT in scope now):

- Embedding similarity for `common_user_phrases`
- LLM-based intent → fact_id classifier
- Multi-card composition rules

**Cap**: maximum 2 cards injected per question (prompt budget). If
matcher returns >2, take top 2 by `(risk_level, score)`.

## Injection point

In `app/api/consultation/stream/route.ts`, **after** the system prompt
is assembled but **before** the user message is added:

```typescript
const matches = matchFactCards(question);
const todayIso = new Date().toISOString().slice(0,10);
const audit: FactCardAuditRow[] = [];
const factBlocks: string[] = [];
const hintMarkers: string[] = [];

for (const m of matches) {
  const card = loadCard(m.fact_id);
  const decision = gateDecision(card);  // 'inject' | 'hint_only' | 'drop'
  if (decision === 'drop') continue;

  if (decision === 'inject') {
    factBlocks.push(card.injection_certain_block.replaceAll('{{TODAY_ISO}}', todayIso));
  } else {
    // hint_only: emit a generic conservative marker, do NOT push facts
    hintMarkers.push(card.title);
  }
  audit.push({
    fact_id: card.fact_id,
    fact_card_state: card.state,
    risk_level: card.risk_level,
    confidence: card.confidence,
    source_quality: card.source_quality,
    official_sources: card.official_sources.map(s => s.url),
    injected_fields: decision === 'inject' ? card.injected_field_names : [],
    needs_review_flags: card.needs_review_flags,
    decision,
  });
}

if (factBlocks.length) {
  messages.push({
    role: 'system',
    content: factBlocks.join('\n\n---\n\n')
  });
}

// record audit (Learning Console)
factCardIdsToPersist = audit.map(a => a.fact_id);
factCardAuditToPersist = audit;
```

`completeAiConsultation` and `failAiConsultation` write
`fact_card_ids` AND `fact_card_audit` to the consultation row.

**SSE event**: emit `fact_cards_injected` event after `received` and
before `first_token`, with the full audit payload:

```json
{
  "event": "fact_cards_injected",
  "items": [{
    "fact_id": "keiei-kanri-2025-10",
    "fact_card_state": "ai_verified",

    "source_quality": "official",
    "decision": "inject",
    "needs_review_flags": ["jigyou_keikakusho_expert_check", "jlpt_alternatives", "jimusho_menseki"]
  }]
}
```

UI can use this for the personalized risk hint (Workstream G) and

## Deploy order

1. Migration 0025 (fact_cards table) — backwards compatible
2. Migration 0026 (ai_consultations.fact_card_ids + fact_card_audit) — backwards compatible (default `[]`)
3. Sync script + matcher + loader + gate decision code
4. Internal dry-run endpoint (gated by `EVAL_LAB_ENABLED`)
5. Stream route inject point — gated behind `FACT_LAYER_ENABLED=true` env (default false on first deploy)
6. Sync first card (`keiei-kanri-2025-10`) at state=`ai_verified`, risk=`critical`
7. QA: run dry-run regression suite to confirm matcher hits + correct gate decisions
8. Flip `FACT_LAYER_ENABLED=true` — `ai_verified` low/medium/high cards begin Alpha injection;
   critical cards inject only if `controlled_alpha_eligible: true` is set per-card
9. Schedule weekly human audit sample (5 random `ai_verified` high/critical cards)

Rollback (any time): flip `FACT_LAYER_ENABLED=false` (zero schema rollback needed).
Rollback per-card: edit card's `state: disabled` → sync script picks it up next deploy.

## Incident review path (when a user / 書士 reports wrong fact)

1. Pull consultation row, read `fact_card_audit` array
2. Identify suspect `fact_id`s
3. Open the `docs/fact-cards/<slug>.md` file
4. Either:
   - Edit content + bump changelog + state stays/moves to `human_reviewed`
   - Or set `state: needs_review` (immediately stops injection)
   - Or set `state: disabled` (permanent retirement)
5. Merge → next deploy applies via sync script
6. If urgent: deploy hotfix that flips `FACT_LAYER_ENABLED=false`
