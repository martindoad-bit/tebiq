# TEBIQ Current Fact Layer — `docs/fact-cards/`

**Owner**: AI-first pipeline (collect → extract → map → QA-gen) + human publish gate
**Sprint**: 0.6 — Workstream C
**Authority**: PL 0.6 sprint kickoff, decision (c) AI-first Fact Layer (2026-05-07)

---

## What this directory is

Source-of-truth for **Current Fact Cards** that get injected into the
production consultation prompt (`/api/consultation/stream`) when a user
question matches the card's scenarios.

Each card is a single markdown file with YAML frontmatter holding the
machine-readable fields, and a markdown body with full source quotes,
scenarios, and QA cases.

## What this directory is NOT

- Not a knowledge base.
- Not a replacement for the model's general knowledge.
- Not a place for opinion / interpretation / 文风 guidance — that
  belongs in `docs/voice/` and `lib/answer/prompt/`.
- Not a list of every visa fact — only **high-impact, time-sensitive,
  policy-change-driven** facts that the model would otherwise answer
  from stale memory.

## State machine — AI-first (revised 2026-05-07 per PL)

> **Principle**: AI-generated by default; source-backed; confidence-gated;
> risk-tiered; human-audited *after* / *on high-risk*. Human is **not**
> a per-card prerequisite to ship.

Every card declares one `state`:

| state | meaning | Alpha frontend? | production? |
|---|---|---|---|
| `draft` | scaffold only, no source-backed content | NO | NO |
| `ai_extracted` | AI pulled fields from source(s); structure / scenarios / QA cases not yet complete | NO | NO |
| `ai_verified` | AI completed source check, scenario mapping, must_say/must_not_say, QA cases, confidence assessment, needs_review flags | **YES** (with provenance recording) | YES, gated by `risk_level` (see below) |
| `human_reviewed` | DOMAIN/PL/書士 audited and approved | YES | YES |
| `needs_review` | AI flagged uncertain / conflicting / low-confidence / high-risk fields for human review | NO (specific facts); may trigger conservative *"need to confirm latest rule"* hint only | NO |
| `conflict` | two official sources disagree | NO | NO |
| `disabled` | retired or broken | NO | NO |

**Per-card risk gating on top of state**:

| risk_level | `ai_verified` injects in Alpha? | `ai_verified` injects in prod? |
|---|---|---|
| `low` | YES | YES |
| `medium` | YES | YES |
| `high` | YES (more conservative copy required) | YES (enters human audit queue) |
| `critical` | only if `controlled_alpha_eligible: true` is set on the card AND PL has signed off in the card's changelog; otherwise requires `human_reviewed` | requires `human_reviewed` unless explicit PL override in changelog |

Sync additionally hard-fails `ai_verified + critical + controlled_alpha_eligible=true`
unless `confidence: high`. Medium-confidence critical cards must stay hint-only
until the flagged source gaps are resolved.

**Per-field gating within an `ai_verified` card**:

A card may be `ai_verified` overall but mark specific fields with
`needs_review_flags: [...]`. Those specific fields **must not be
injected** as confirmed facts. The matcher splits the card's
`injection_format` into a `certain_block` (injected) and a
`needs_review_addendum` (NOT injected; may produce only a conservative
"need to confirm specifics" hint).

**Required record per injection** (Learning Console):

- `fact_card_ids`
- `fact_card_state`
- `risk_level`
- `confidence`
- `source_quality` (`official` | `quasi_official` | `secondary`)
- `official_sources`
- `injected_fields` (which fields actually went into the prompt)
- `needs_review_flags` (which fields were withheld)

This is the audit trail for incident review when a user / 書士 reports
the model gave a wrong fact.

## Source whitelist

Only these are acceptable as `official_sources`:

- 出入国在留管理庁 (`isa.go.jp`, `moj.go.jp/isa`)
- 法務省 (`moj.go.jp`)
- e-Gov 法令検索 (`elaws.e-gov.go.jp`)
- 厚生労働省 (`mhlw.go.jp`) — for 年金・健康保険・労務
- 国税庁 (`nta.go.jp`) — for 税金関連
- 総務省 (`soumu.go.jp`) — for 住民税・地方税制度
- デジタル庁 (`digital.go.jp`) — for マイナンバー / マイナ保険証制度
- 全国健康保険協会 (`kyoukaikenpo.or.jp`) — for 協会けんぽ・任意継続
- 各市町村区役所 official pages — for 住民税・国民健康保険 procedural
- 自治体公式ドメイン (`*.lg.jp`) — for local tax / NHI / address procedures
- 政府公式 PDF / 告示 / 省令

**NOT acceptable as source**: 行政書士ブログ, 中文中介公众号, 小红书,
Twitter/X, Yahoo知恵袋, 中文自媒体. These may be used as **scenario
seeds** (to know what users ask) but never as `official_sources`.

## File naming

`<topic-slug>.md` — kebab-case, English ASCII for filesystem safety.
Examples: `keiei-kanri-2025-10.md`, `eijuu-nenkin-risk.md`,
`gijinkoku-job-mismatch.md`.

## Required frontmatter fields

See any existing card for the canonical shape. AI-generated cards must
mark each field as either `source: <url>` (direct quote) or
`source: ai_inference` (AI synthesis). Human reviewers gate on
inference fields first.

## Workflow (AI-first)

```
1. AI source collector  (C1)  → raw_excerpt + source_id
2. AI fact extractor    (C2)  → state=ai_extracted, with direct_fact_fields vs ai_inferred_fields split
3. AI scenario mapper   (C3)  → common_user_phrases + scenario triggers
4. AI QA generator      (C4)  → must_have / must_not_have / regression Qs
5. AI self-verification       → if all fields have source or labeled ai_inference,
                                and confidence + source_quality assessable → state=ai_verified
                                if uncertain/conflicting/high-risk fields exist → those fields
                                go into needs_review_flags (card may still be ai_verified overall)
6. ENGINE matcher (C0,C6)     → loads ai_verified + human_reviewed; per-card risk gate; per-field needs_review_flags filter
7. Human audit (C5)           → DOMAIN/PL/書士 sample-checks high/critical cards; flips state to human_reviewed
                                or marks needs_review / conflict
8. QA regression (C7)         → automated test suite (must_have/must_not_have)
9. Incident review            → user/書士 reports wrong fact → trace via fact_card_ids
                                → re-evaluate card → state may flip to needs_review/disabled
```

## When AI may self-promote `ai_extracted` → `ai_verified`

The AI extractor MUST satisfy ALL of the following before flipping
state to `ai_verified`. Otherwise the card stays at `ai_extracted`.

- [ ] At least 1 `official_sources` entry with `last_checked_at <= 30 days`
- [ ] Every line of `current_effective_fact` is annotated either
      `source: <id>` (with quote in body) OR `source: ai_inference`
- [ ] Any line marked `ai_inference` is also listed in
      `needs_review_flags` (and excluded from the `certain_block` of
      `injection_format`)
- [ ] `current_date_logic` exists and is decidable from the system clock alone
- [ ] `applies_to` enumerates which application types this covers
- [ ] `exceptions_or_transition` exists OR card explicitly states "no exceptions"
- [ ] `common_user_phrases` has ≥ 5 colloquial Chinese phrasings
- [ ] `must_say` and `must_not_say` populated
- [ ] `qa_cases` has ≥ 3 entries, each with `must_have` AND `must_not_have`
- [ ] `confidence: low|medium|high` set
- [ ] `risk_level: low|medium|high|critical` set
- [ ] `source_quality: official|quasi_official|secondary` set
- [ ] `injection_format` is split into `certain_block` and (optional) `needs_review_addendum`
- [ ] No `conflict` markers in body
- [ ] `changelog` entry recording the AI run

## Human audit — what humans DO and do NOT do

**DO**:

- Sample-check `high` and `critical` `ai_verified` cards
- Resolve `needs_review` and `conflict` states
- Approve `critical` cards for production (`human_reviewed`)
- Run `incident_review` when a user / 書士 reports a wrong fact
- Periodic random audit (e.g. weekly N=5 sample) for quality drift

**DO NOT**:

- Author cards from scratch
- Hand-write `current_effective_fact`, `must_say`, `must_not_say`
- Block ai_verified low/medium/high cards from Alpha shipping
- Be the gate for every card

When a card is audited, append a `changelog` row and set `reviewer`,
`approved_at`. If approved, set `state: human_reviewed`.

## Frontend voice (for any injected card)

Even when a card is `human_reviewed`, the Alpha frontend MUST hedge:

- ✅ "根据当前已确认的官方信息，普通新申请/变更场景下，不能再按旧的 500万标准简单准备。"
- ❌ "绝对必须 / 一定不可以 / 最终结论是 ..."

Fact layer supplies *current facts*. The model interprets. TEBIQ remains
in Alpha.
