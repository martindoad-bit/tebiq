# 真实问题分类体系 v1（QUESTION_TAXONOMY_V1）

**生成**：2026-04-30
**分支**：`content/real-question-data-v1`
**用途**：为 backlog / TOP20 / decision-seed-cards 提供统一标注口径

本表用中文解释每个字段，不写技术黑话。**任何标注 / 卡片 / Decision Card 生成都必须先对照此表**。

---

## 1. visa_type — 签证 / 身份

用户当前持有的在留資格。决定问题适用范围与法条入口。

| 取值 | 说明 |
|---|---|
| `定住者` | 离婚定住 / 死别定住 / 抚养日本人子女 等 |
| `日本人配偶者等` | 日本人配偶 / 子（実子 / 特別養子） |
| `永住者の配偶者等` | 永住者配偶 / 子 |
| `技人国` | 技術・人文知識・国際業務（最常见的工作签） |
| `経営・管理` | 公司経営者 / 役員（含新规 + 续签 + 改正） |
| `特定技能` | 1 号 / 2 号（14-16 業種） |
| `家族滞在` | 工作签 / 留学 持有人的配偶 / 子 |
| `留学` | 在学中外国人 |
| `技能実習` | 1 号 / 2 号 / 3 号 |
| `永住者` | 已取得永住 |
| `通用` | 跨签证类型适用（如年金 / 国保 / 文书识别） |

多个用 `;` 分隔。例：`技人国;経営・管理`。

## 2. life_event — 场景 / 触发事件

用户问问题时所处的现实事件。决定 Decision Card 的触发入口。

| 取值 | 说明 |
|---|---|
| `离婚` | 协议 / 调停 / 裁判 离婚后 6 ヶ月内 |
| `死别` | 配偶死别后在留資格选项 |
| `结婚` | 婚姻届 + 在留資格変更 検討 |
| `转职` | 跳槽 / 雇主变更（14 日届出窗口） |
| `离职` | 退职后空白期 / 失業給付 |
| `公司倒闭` | 雇主 倒産 / 民事再生 |
| `永住申请` | 申请节点（直近 1 年公司在职 / 補正 / 不许可后） |
| `永住准备` | 申请前 1 年准备（10 年路径 / 高度人材 1-3 年路径） |
| `経営・管理 新规` | 新设公司 + 取签 |
| `経営・管理 续签` | 续签前 3-6 ヶ月 |
| `経営・管理 改正` | 2025/10/16 改正过渡期对应 |
| `公司设立` | 法人設立 / 登記 / 法人口座開設 |
| `年金缴纳` | 国民年金 / 厚生年金 滞纳清算 |
| `国保社保切换` | 入社 / 退职 切换 + 加入空白防止 |
| `税务` | 確定申告 / 法人税 / 住民税 / 课税证明 |
| `家庭团聚` | 家族滞在 招聘 / 配偶 招聘 |
| `带父母来日` | 短期滞在 招聘理由書 |
| `子女` | 子女 在留資格取得 / 出生届 / 入学 / 转学 |
| `搬家` | 転入届 + 国保 / 年金 / 银行 / 入管 联动 |
| `分居` | 分居期间婚姻実態証明 |
| `不法滞在` | 期限超过 / 補正不能 / 退去強制 |
| `刑事/罚金` | 罰金 / 略式命令 / 出頭通知 |
| `归化` | 申请前 6 ヶ月准备 |
| `回国清算` | 回国前 永住放棄 / 再入国 / 税务清算 |
| `怀孕生子` | 妊娠届 / 出産育児一時金 |
| `文书识别` | 收到日文信件无法判断时 |
| `通用` | 跨场景（如概念性提问） |

多个用 `;` 分隔。

## 3. procedure_type — 手続き类型

问题指向的实际行政 / 法律 / 手续动作。

| 取值 | 说明 |
|---|---|
| `在留資格変更` | 现签证类型 → 其他类型 |
| `在留期間更新` | 同类型续签 |
| `永住申請` | 永住者取得 |
| `帰化申請` | 国籍取得（法務局而非入管） |
| `届出` | 入管 14 日届出（契約機関 / 配偶者）/ 市役所 転入転出 等 |
| `申告` | 確定申告 / 法人税申告 / 消費税申告 |
| `登記` | 法人設立 / 役員変更 / 商業登記 |
| `申請` | 一般行政申请（児童手当 / 高額療養費 / 雇用保険） |
| `異議申立` | 不许可后 行政不服 / 取消訴訟 |
| `判断/认知` | 不涉及具体动作，仅认知纠偏 |
| `决策` | 多路径选择（如 永住放棄 vs 再入国） |
| `通用` | 不限定 |

## 4. urgency — 紧急程度

| 取值 | 判定标准 |
|---|---|
| `high` | 14 日届出窗口 / 不法滞在已发生 / 在留期限 ≤ 30 日 / 退去強制风险 / 收到 補正通知 / 出頭通知 / 罰金通知 |
| `medium` | 续签前 3-6 月 / 申请准备期 / 政策施行过渡窗口 / 收到决定通知書类 / 出国前预防 |
| `low` | 长期规划 / 概念性提问 / 多年期决策（永住 / 帰化 / 不动产） |

## 5. answer_level — 答案等级

决定能不能用 1 张卡 / 1 个流程图 解决。

| 等级 | 说明 | 适合卡片 |
|---|---|---|
| `L1` | 1 句话事实（政府明文 / 制度定义） | info |
| `L2` | 多步骤流程（标准 workflow） | workflow |
| `L3` | 多路径决策（条件分歧多） | decision_card |
| `L4` | 复杂个案 / 需专家判断（变量多 / 案例依赖） | expert_handoff |

---

## 6. card_type — 卡片类型

TEBIQ 内容产品形态。决定渲染层和 schema 字段。

| 类型 | 说明 | 示例 |
|---|---|---|
| `decision_card` | 多路径分歧 → 给条件 → 推荐方向 + 解释为什么不选其他 | 永住放棄 vs 再入国 |
| `workflow` | 单一线性流程 → 步骤 1/2/3 + 必备材料 | 転入届 + 国保 + 银行 |
| `risk_chain` | 用户当前状态 + 不处理后果链 | 14 日届出未交 → 续签影响 |
| `misconception` | 用户常见错误认知 + 真实情况 | 永住带父母来日 |
| `info` | 事实说明 / 概念定义（不涉及决策） | 定住者是什么 |

## 7. needs_expert — 是否需要专家

| 取值 | 含义 |
|---|---|
| `yes` | 必须 行政書士 / 弁護士 / 税理士 / 社労士 介入 |

| `maybe` | 简单 case 自己 / 复杂 case 找专家（条件分歧） |

## 10. suggested_priority — 建议优先级

```
P0 = 现在做 seed card / 现在补内容
P1 = 下一个 batch（按高频 + 高 risk）
P2 = 季度内做
P3 = 半年内做（长尾 / 个案）
```

判定参考（参考 TOP20 优先级判定）：
- 高频 + 紧急 + 影响签证 → P0
- 高频 + 容易做错 + 适合 Decision Card → P0/P1
- 中频 + 已有内容覆盖 + 仅需 stitch → P1
- 低频 / 报价 / 商务 / 个案 → P2/P3
