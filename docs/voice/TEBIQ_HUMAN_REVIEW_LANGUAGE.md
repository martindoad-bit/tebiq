# TEBIQ Human Review Language

> 本文件定义需要人工复核的高风险场景下的文案规范。
> TEBIQ 在 `handoff_trigger` 命中时必须明确升级到专业确认路径。
> AI 不做最终法律判断；此文件定义如何将这条边界清晰传达给用户。
>
> 所有 `user_copy` 均为 B 层（待验证），上线前须 DOMAIN 复核 + Project Lead 裁决。

---

## 专业人士称呼参数化

所有文案中的 `{{specialist}}` 为参数化占位符：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `specialist` | `行政書士` | VOICE 默认值 |
| 可选值 | 入管専門弁護士 / 税理士 / 社労士 / 司法書士 / 相关专业人士 | 由 DOMAIN 在对应 DR 中指定 |

**规则**：VOICE 不自行决定专业人士类型。ENGINE 渲染时使用 DOMAIN 指定值；无 DOMAIN 指定时使用默认值 `行政書士`。

---

## 总体原则

- 高风险场景必须**明确升级**，不能模糊带过
- 文案不得最小化风险，但也不得制造恐慌而不给行动路径
- 每条 HR 文案都必须包含：**是什么情况 → 为什么要找专业确认 → 具体下一步是什么**
- 不允许 TEBIQ 暗示自己已经解决了问题
- 不允许没有下一步的风险提示（"风险提示"后必须跟具体动作）

## HR-01：経管 + 清算 / 离境连锁风险

| 字段 | 内容 |
|------|------|
| `hr_code` | HR-01 |
| `trigger_condition` | 経営管理（经营管理）在留资格持有者，同时触发清算手续信号或主动提出离境意向 |
| `urgency_level` | high |

### user_copy（中文）

> 这类情况涉及多个在留手续的连锁影响，建议与{{specialist}}确认具体步骤后再行动。
>
> 公司清算、在留卡返纳、出境时间点这几件事的顺序如果错了，会影响之后的日本入境记录和签证审查。

### action_direction

- 建议用户在行动前联系{{specialist}}确认步骤顺序
- 具体关注点：公司清算时间线 / 入管届出 / 纳税管理人指定 / 在留卡返纳时机 / 出境方式

### what_tebiq_must_not_claim

- 不得说"按照以下步骤就可以了"（TEBIQ 不代理、不保证步骤正确性）
- 不得给出具体的清算时间表（法律流程，需专业判断）
- 不得暗示"风险不大"或"通常没问题"

## HR-02：在留期限紧急（7日以内到期）

| 字段 | 内容 |
|------|------|
| `hr_code` | HR-02 |
| `trigger_condition` | 在留卡到期日与当前日期差距 ≤ 7 天，且更新申请尚未提出 |
| `urgency_level` | critical |

### user_copy（中文）

> 在留卡到期日不足 7 天，更新申请必须尽快处理。
>
> 最近的入国管理局或出入国在留管理局窗口是现在最快的路径——带上在留卡和护照，直接说明到期紧迫情况。{{specialist}}也可以协助紧急申请，但今天就要开始联系。

### action_direction

- 立即前往最近的入管局窗口（非预约制，当日受理）
- 同步联系{{specialist}}处理紧急申请
- 带齐材料：在留卡、护照、在职证明（如适用）

### what_tebiq_must_not_claim

- 不得说"提交申请后就可以放心了"（受理 ≠ 许可）
- 不得给出具体期限的法律承诺（"在 X 日前申请一定没问题"）
- 不得省略行动路径只说"请注意期限"

## HR-03：不法就労リスク 检出

| 字段 | 内容 |
|------|------|
| `hr_code` | HR-03 |
| `trigger_condition` | 用户描述的工作内容、就职状态、雇用形式与其在留资格许可范围存在不符信号 |
| `urgency_level` | critical |

### user_copy（中文）

> 你描述的工作情况和现在的在留资格许可范围之间有一些需要核对的地方。
>
> 这类情况如果没有及时处理，可能影响后续的在留更新和永住申请——现在最重要的是找{{specialist}}确认实际情况，不要先继续工作再问。

### action_direction

- 立即联系{{specialist}}，说明具体工作内容和在留资格
- 不要在确认之前继续从事争议性工作
- 不要向雇主或入管单方面说明，先由专业人士评估实际风险

### what_tebiq_must_not_claim

- 不得最小化风险（"可能没什么大问题"类表述）
- 不得做"是/否不法就労"的定性判断（需专业法律评估）
- 不得建议用户自行向入管解释

## HR-04：不許可通知受領後

| 字段 | 内容 |
|------|------|
| `hr_code` | HR-04 |
| `trigger_condition` | 用户明确告知已收到在留更新/变更不许可通知 |
| `urgency_level` | critical |

### user_copy（中文）

> 收到不许可通知后，时间非常紧——通常只有短暂的出国准备期，期间可以异议或重新申请的窗口有限。
>
> 现在最重要的一步：今天就联系{{specialist}}，带上通知书原件，让他们看清楚不许可的具体理由。理由不同，后续路径完全不同。

### action_direction

- 不许可通知书原件需由专业人士解读（理由记载是关键）
- 时间窗口极短，不得拖延咨询
- {{specialist}}可评估：再申请可行性 / 特定活动签证临时过渡 / 异议程序

### what_tebiq_must_not_claim

- 不得说"不许可通知还有机会"（需专业评估，不能保证）
- 不得说"不许可就只能离境"（路径多样，需具体分析）
- 不得给出再申请材料清单（材料取决于不许可理由，TEBIQ 没有读取通知书的能力）

## HR-05：配偶離婚後 6ヶ月切迫

| 字段 | 内容 |
|------|------|
| `hr_code` | HR-05 |
| `trigger_condition` | 配偶者（日本人の配偶者等 / 永住者の配偶者等）在留资格持有者，离婚后距 6 个月届出期限不足 30 天 |
| `urgency_level` | high |

### user_copy（中文）

> 离婚后 6 个月内需要向入管提交届出，或者申请其他在留资格——这个期限过了，在留状态会直接受影响。
>
> 如果还不确定下一步申请哪种在留资格，现在就需要跟{{specialist}}谈，因为资格变更的申请材料准备需要时间。

### action_direction

- 确认离婚日期，计算 6 个月截止日
- 联系{{specialist}}评估：就労資格への変更 / 定住者申請 / その他在留資格
- 届出本身不延续在留资格，仅是义务；资格变更申请是实质动作

### what_tebiq_must_not_claim

- 不得告知具体可申请的资格类型（需评估个人情况）
- 不得保证资格变更一定批准
- 不得说"只要提交届出就没问题了"（届出 ≠ 资格续期）

## HR-06：複数在留問題重複

| 字段 | 内容 |
|------|------|
| `hr_code` | HR-06 |
| `trigger_condition` | 同一用户在单次对话或同一问题中触发 2 个或以上的 `handoff_trigger`（如期限紧急 + 就労可疑 / 离婚 + 更新期限同时临近） |
| `urgency_level` | high |

### user_copy（中文）

> 你的情况同时涉及几个不同的在留手续，各自的时间节点和处理顺序会互相影响。
>
> 这种叠加情况 TEBIQ 没有办法给出单一的处理顺序建议，因为优先级取决于你的具体状态。建议找一位有経管签证经验的{{specialist}}，把几件事一起理清楚，再分头处理。

### action_direction

- 不要分别处理每个问题，找一人统一评估
- 强调{{specialist}}"有経管签证经验"（领域匹配很重要）
- 用户应带齐所有相关材料去一次会谈

### what_tebiq_must_not_claim

- 不得给出多问题的综合处理顺序（超出 TEBIQ 能力边界）
- 不得说"先处理 X 再处理 Y"（顺序可能错误且后果严重）
- 不得暗示问题可以分开独立解决

## 通用禁止模式（Human Review 文案适用）

以下模式在任何 HR 状态的用户文案中均禁止出现：

| 禁止模式 | 说明 |
|----------|------|
| 最小化风险 | "通常没事"、"问题不大"、"可能还好"——此类表述在高风险场景是 P0 |
| 暗示 TEBIQ 已解决 | "TEBIQ 已帮你分析完毕"、"以下是解决方案"——TEBIQ 不做最终判断 |
| 恐吓无出路 | "情况很严重，你的在留可能会被取消"——恐吓 + 不给行动路径是禁止组合 |
| 没有下一步 | 任何风险描述后面必须跟具体的下一步动作（谁、做什么、怎么开始） |
| 法律承诺 | "做了这几步就一定没问题"——绝对禁止 |
| 导流腔 | "立即预约专家"、"一键联系"——营销感 + 无具体指引 |
| 揭露内部判断 | "触发了 HR-03 规则"、"系统检测到不法就労风险"——内部字段外泄 |
| 确认 AI 身份 | "AI 无法处理此类问题，请找人工"——禁止暴露底层判断逻辑 |

## 附：urgency_level 对照

| urgency_level | 含义 | 对应特征 |
|---------------|------|----------|
| `critical` | 时间窗口极短 / 后果不可逆 | HR-02、HR-03、HR-04 |
| `high` | 需要尽快行动，但有操作窗口 | HR-01、HR-05、HR-06 |

`critical` 状态下，用户文案必须在第一句就体现时间紧迫性，不得先铺垫再说紧急。
