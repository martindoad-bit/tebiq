# Risk Level Vocabulary v0.1

本文件定义 TEBIQ 在用户端使用的四级风险语言规范。  
风险等级由 DOMAIN 判断。VOICE 负责定义每个等级的表达方式和禁忌词。

---

## 核心原则

- 风险等级语言的核心功能：让用户感受到正确的优先级，推动正确的行动
- 不是通过恐吓传达紧迫感，是通过方向感和边界感传达
- 等级越高，语言越清楚、越具体，不是越戏剧化

---

## 四级风险词汇表

### L0 — 低风险 / 手续型

**定义：** 问题明确，路径清楚，无特殊条件，按正常流程处理即可。

**用户端语言特征：**
- 直接说出手续和步骤
- 不需要引入风险框架
- 可以有"下一步是……"的清楚指向

**示例方向：**
> 这类情况按正常更新流程处理。需要准备的材料包括……

**禁止：**
- 把手续型问题包装成"需要特别注意""建议谨慎处理"
- 无必要地插入专业参照（低风险不需要专业人士确认）
- 为了显得严谨而升级风险语气

---

### L1 — 中风险 / 条件确认型

**定义：** 问题本身有解，但答案取决于用户具体条件。条件不同，结论可能不同。

**用户端语言特征：**
- 说明"这件事因具体条件不同，处理方式会有差异"
- 触发 clarification 流程，确认关键条件
- 给出"如果……则……"的方向性说明（在 DOMAIN 确认范围内）

**示例方向：**
> 这个问题的处理方式，因你目前的在留资格类别不同会差很多。先确认一个情况：你现在的在留资格是什么？

**禁止：**
- 假设用户条件然后直接给结论（未经 clarification）
- 把条件不确定写成"情况复杂，建议专业人士"（这是 L2/L3 的表达）
- 用"通常"作为独立结论（"通常没问题"）

---

### L2 — 高风险 / 不建议搁置

**定义：** 存在明确的在留风险或时间紧迫性，搁置可能导致状态恶化。DOMAIN 确认为高风险场景。

**用户端语言特征：**
- 清楚传达"不建议拖延"的信号
- 说明风险方向（不是具体后果预测）
- 给出下一步行动方向，不停在"这很重要"
- 如有具体时限信息，须经 DOMAIN 确认后才能写出

**示例方向：**
> 这件事有时间方面的考量，不建议拖延处理。建议先确认你当前的在留期限，再判断下一步。

**示例方向（强表达，P0 场景 DOMAIN 确认后可用）：**
> 未在规定期限内处理，在留资格可能被取消，且影响日后申请记录。

**禁止：**
- 在无 DOMAIN 确认的情况下写具体时限或数字
- 使用空泛恐吓词（"情况很严重""重大后果""危险"）
- 把高风险处理成轻描淡写（降低用户对优先级的感知）

---

### L3 — 必须人工确认

**定义：** 超出 TEBIQ 自动判断范围，或错误判断会导致用户做出错误决策。必须由专业人士介入。

**用户端语言特征：**
- 清楚说明为什么需要人工确认（不是推销，是说明必要性）
- 给出专业参照方向（类型由 DOMAIN 指定，默认行政書士）
- 如果 TEBIQ 可以提供方向性说明，在触发 L3 之前先给
- L3 不等于"TEBIQ 什么都不说"

**示例方向（Advisory / L1-style）：**
> 这类情况 TEBIQ 可以提供方向，但具体适用性建议由行政書士确认。

**示例方向（Mandatory / 强触发）：**
> 这类情况超出 TEBIQ 自动判断范围，不建议仅凭自动回答做决定。建议直接由行政書士或专业人士处理。

**禁止：**
- 把 L3 触发写成销售转化（"立即预约""专家帮你搞定"）
- 在可以给方向的情况下直接触发 L3（浪费了 L1/L2 的空间）
- 滥用 L3（降低整体可信度）

---

## 风险等级与语言映射总表

| 等级 | 名称 | 核心语言信号 | 行动方向 |
|------|------|------------|---------|
| L0 | 低风险 / 手续型 | 直接说步骤 | 按流程处理 |
| L1 | 中风险 / 条件确认型 | "因条件不同，处理方式会差很多" | 触发 clarification |
| L2 | 高风险 / 不建议搁置 | "不建议拖延处理" / "有时间方面的考量" | 给出方向 + 推动行动 |
| L3 | 必须人工确认 | "超出自动判断范围" / "建议专业人士确认" | 给方向性说明 + 引导人工 |
