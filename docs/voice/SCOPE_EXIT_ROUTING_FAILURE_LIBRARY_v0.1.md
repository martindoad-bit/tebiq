# Scope Exit & Routing Failure Library v0.1

本文件定义 TEBIQ 在以下四种情形下的语言处理规范：

1. **真正不支持**：超出 TEBIQ 服务范围的问题
2. **信息不足**：无法路由，需要 clarification（非正常 clarification 流程中的极端情形）
3. **可桥接到在留风险**：表面上 OOS，但存在隐含在留风险维度
4. **疑似 routing failure**：系统本身的路由问题，不是用户问题

---

## 核心原则

- 不把 routing failure 的责任推给用户
- 不用"请换一种方式描述你的问题"
- OOS 也要给出方向，不能停在"TEBIQ 无法回答"
- 能桥接的要桥接，不能桥接的说清楚为什么

---

## 类型 1 — 真正不支持（True OOS）

**定义：** 问题超出 TEBIQ 当前支持的在留风险管理范围，且无法从在留风险角度建立关联。

**典型场景：**
- 纯日语学习问题
- 非在留相关的生活咨询（租房纠纷、家庭关系、职场骚扰等）
- 需要律师判断的法律诉讼问题（非在留相关）
- 纯财税规划（无在留相关维度）

**用户端语言模板：**
> 这类问题目前超出 TEBIQ 的判断范围。TEBIQ 专注于在留风险相关的情况。
>
> （如有方向性说明，加上：如果你的情况涉及在留资格变化或在日居住状态，可以从那个角度告诉 TEBIQ。）

**禁止：**
- "TEBIQ 暂时无法回答这个问题"（太消极，没有方向）
- "请换一种方式描述"（推责任给用户）
- 不给任何方向就结束回答

---

## 类型 2 — 信息不足（Extreme Information Gap）

**定义：** 用户描述极度模糊，无法判断问题属于哪个在留风险维度，也无法触发正常 clarification 流程。

**典型场景：**
- 极短输入（"我该怎么办""我快不行了""签证快到了"）
- 混合语言 / 语义不连贯
- 描述了情绪状态而非事实情况

**用户端语言模板：**
> TEBIQ 需要先了解你的基本情况，才能判断下一步。
>
> 可以先告诉我：你目前在日本，持有什么类型的在留资格？

**禁止：**
- 在用户描述模糊时假设场景然后作答
- "你的问题不够清楚"（评判用户表达）
- 同时问超过 2 个问题（见 Clarification 规则）

---

## 类型 3 — 可桥接到在留风险（Bridgeable OOS）

**定义：** 问题表面上不属于在留风险，但存在隐含的在留相关维度需要主动识别。

**典型场景：**
- 用户问工资发放问题 → 可能涉及就劳资格合规
- 用户问公司注册 → 可能涉及在留资格和经营管理要求
- 用户问扶养家属 → 可能涉及家族滞在在留资格
- 用户问租约问题 → 可能涉及住所变更报告义务
- 用户问税务申报 → 可能涉及居住者状态和在留影响

**用户端语言模板：**
> 你问的这件事 TEBIQ 不直接处理，但其中有一个部分可能和你的在留情况有关：（具体说明）。
>
> 如果你想从在留风险的角度进一步确认，可以告诉 TEBIQ 你目前的在留资格情况。

**禁止：**
- 把桥接场景当成真正 OOS 直接关闭（错失风险识别机会）
- 把所有周边问题都强行拉进在留风险框架（过度桥接）
- 在不确定有桥接维度时假装有（不自行补充专业结论）

---

## 类型 4 — 疑似 Routing Failure（系统问题）

**定义：** 用户描述看起来属于 TEBIQ 支持范围，但系统无法完成路由。不是用户的问题，是 TEBIQ 的问题。

**典型场景：**
- 路由逻辑无法匹配当前输入组合
- 输入包含 TEBIQ 当前不能处理的信息结构
- 系统出现技术性路由错误

**用户端语言模板：**
> TEBIQ 需要先确认你的当前情况，再判断下一步。可以告诉我你目前的在留资格类别吗？

**禁止：**
- "请重新描述你的问题"（把失败原因推给用户）
- "你的问题比较特殊，无法处理"（不准确，也有推责嫌疑）
- 向用户暴露内部标签（routing_failure、FULL_COMPARABLE 等）

**Console 端处理（中台语言）：**
> 路由失败。DOMAIN 可标注语义原因：ambiguous_input / missing_context / unsupported_scenario。

## 场景分类表

| 场景描述 | 分类 | 处理方向 |
|---------|------|---------|
| 用户问日语课程推荐 | True OOS | 说明范围 + 给出在留角度入口 |
| 用户输入"我不知道怎么办" | Extreme Information Gap | 问最关键的一个条件 |
| 用户问能不能开公司 | Bridgeable OOS | 桥接到经营管理在留资格维度 |
| 用户问租约纠纷 | Bridgeable OOS（弱）| 检查住所变更义务角度；如无关则 True OOS |
| 用户问工资拖欠 | Bridgeable OOS | 桥接到就劳资格合规维度 |
| 系统无法路由，用户描述正常 | Routing Failure | 用 TEBIQ 视角要求补充条件 |
| 用户问非在留法律诉讼 | True OOS | 直接说明范围，不桥接 |
| 用户问签证申请（非日本）| True OOS | 说明 TEBIQ 仅覆盖在日在留，给方向 |
