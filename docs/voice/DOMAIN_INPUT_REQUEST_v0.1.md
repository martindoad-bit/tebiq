# DOMAIN Input Request v0.1

本文件列出 VOICE 需要 DOMAIN 输入的待确认项目。  
共 10 条：P0 × 5（阻塞性）+ P1 × 5（建议性）。

DOMAIN 输入后，VOICE 将依据确认结论更新对应文案资产。

---

## P0 — 阻塞性（VOICE 无法在确认前定稿）

### D-P0-01 — 具体时限的适用范围确认

**问题：** VOICE 当前在所有时间敏感场景中使用"有时间方面的考量，不建议拖延处理"的泛化表达，刻意不写具体时限（如"14 天内""3 个月前"）。

**需要 DOMAIN 确认：**
1. 哪些场景允许写出具体时限？（例如：更新申请窗口期、再入国许可有效期等）
2. 对于可以写时限的场景，具体时限是多少？适用条件是什么？
3. 是否有"超过某个时点则路径改变"的节点需要在文案中体现？

**当前处理：** 所有时限表达使用泛化语言，等待 DOMAIN 确认后补充具体内容。

---

### D-P0-02 — P0 场景清单与强提示确认

**问题：** VOICE 原则允许在 DOMAIN 确认的 P0 场景中使用明确但克制的强提示（包括高情绪词和不可逆后果表达）。但 VOICE 不自行判断哪些场景是 P0。

**需要 DOMAIN 确认：**
1. 当前 TEBIQ 100Q 中哪些问题被确认为 P0 场景？
2. 每个 P0 场景允许使用的"不可逆后果"具体是什么？（需要写明具体后果，不能空泛）
3. 是否有"场景看起来 P1，但特定条件下升级为 P0"的触发条件？

**当前处理：** 高情绪词和不可逆后果表达暂时不进任何文案，等待 DOMAIN P0 场景清单。

---

### D-P0-03 — OOS 桥接场景的技术验证

**问题：** VOICE 识别了 5 类可能桥接到在留风险的"表面 OOS"场景（工资→就劳合规、公司注册→经营管理、扶养→家族滞在等）。但桥接是否技术上正确，需要 DOMAIN 验证。

**需要 DOMAIN 确认：**
1. SCOPE_EXIT_ROUTING_FAILURE_LIBRARY 中的桥接场景表，哪些桥接方向是技术上正确的？
2. 哪些情形下桥接不成立，应直接处理为 True OOS？
3. 是否有 VOICE 未识别但应该桥接的场景？

**当前处理：** 桥接场景文案标注为 draft，等待 DOMAIN 验证后才能定稿。

---

### D-P0-04 — Human Review L2 触发条件的技术边界

**问题：** VOICE 定义了 5 个 L2 Mandatory 触发场景。需要 DOMAIN 确认触发条件的技术边界，避免过度触发或触发不足。

**需要 DOMAIN 确认：**
1. HR-L2-01 到 HR-L2-05 的触发条件，哪些是准确的？哪些需要修正？
2. 是否有其他必须列为 L2 的场景 VOICE 未覆盖？
3. HR-L2-02（强制退去场景）和 HR-L2-05（材料真实性）的具体触发语言（关键词或语义信号）？

**当前处理：** L2 场景定义为 draft，等待 DOMAIN 技术边界确认。

---

### D-P0-05 — 100Q 种子题的风险等级与文案方向对应

**问题：** VOICE 需要为 100Q 中的题目提供风险等级语言对应，但需要 DOMAIN 先完成风险等级标注。

**需要 DOMAIN 确认：**
1. 100Q 中每道题的 VOICE 风险等级（L0/L1/L2/L3）是什么？
2. 对于 P0×20 的高优先级题目，每道题的：
   - 关键 clarification 缺口是什么？
   - 是否触发 human review，触发等级（L1/L2）？
   - 用户端核心语言信号是什么（一句话方向）？
3. 标注完成后，VOICE 根据标注结果生成对应题目的文案草稿。

**当前处理：** 等待 DOMAIN 完成 100Q 风险等级标注后激活。
