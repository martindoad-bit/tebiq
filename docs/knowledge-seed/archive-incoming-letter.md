# 收到信件：档案如何快速比对历史

## 这事跟我有关吗？
你刚从信箱里取出一封 市役所、税务署、年金事務所、入管 或 健康保険組合 寄来的信件，封面写着 課税決定通知書、督促状、催告書、納税通知書、年金保険料納入告知書 之类的标题。在拆封读懂的同时，你需要知道这封信和过去同 issuer 寄来的信件相比有什么变化——是常规年度通知，还是异常事件。档案的"拍照即懂 + 历史比对"就是为这个场景设计的。

## 这是什么
档案对收到信件的处理流程：拍照识别 → 提取 issuer / doc_type / amount / deadline → 自动匹配档案中同 issuer 同 doc_type 的历史事件 → 显示对比视图。

档案能做的客观比对：

- **同 issuer 同 doc_type 上一次出现是何时**：例如这次的 督促状 是 市役所 寄来的住民税督促，档案显示前一次督促是 2025 年 11 月，间隔 5 个月——属于重复事件。
- **金额变化**：例如 課税決定通知書 显示本年度住民税 245,000 円，档案显示前年度同种通知是 198,000 円——年度变化 +24%，可在备注里写下原因（涨薪 / 兼业新增 等）。
- **是否已结清**：例如这次督促对应的 元税 / 保険料 在档案中是否有缴纳事件记录，缴纳日期与督促日期的关系。
- **是否触发后续文书**：例如档案显示前次 督促状 之后 30 天又收到 催告書、之后 60 天又收到 差押予告——可以推算本次督促如果不处理可能的后续节奏。

档案不做的事：

- **不替你判断"严重不严重"**：信件的处理紧迫度由你自己结合金额、deadline、自身资金状况判断。
- **不自动联系 issuer**：所有联系（电话咨询、申请分割缴纳）需你本人或委托专家操作。

## 你应该做什么
1. 在 TEBIQ 拍照识别这封信件，确认 doc_type、issuer、amount、deadline 识别正确。
2. 打开"历史比对"视图，看同 issuer / 同 doc_type 的过去事件。
3. 如果是常规年度通知（課税決定通知書 / 納税通知書 / 年金保険料納入告知書）：核对金额、缴纳期限，记入档案备注，按期缴纳。
4. 如果是异常通知（督促状 / 催告書 / 差押予告 / 不許可通知）：在档案中标记紧急，立即处理对应原税 / 保険料 / 申請 的状态。
5. 缴纳完成后，把 領収書 / 振込控え 拍照入档，关闭事件。

## 不做的后果
不拍照入档、只读完信件就放进抽屉，会失去跨年度的对比基线——下次收到同类信件时，仍需翻找抽屉确认上次是何时。督促 / 催告 类文书的时间间隔在审查时是有意义的（永住申请、续签时审查官会关注公的義務 履行轨迹），档案是这个轨迹的唯一来源。

## 什么时候你应该找专家
收到的信件涉及 退去強制令書（入管）、差押通知（税务署 / 市役所）、改善命令（経営・管理 相关）、不許可通知（在留期間更新 / 永住）等高风险文书，立即咨询行政書士 或弁護士。涉及金额较大的滞納补缴、分割缴纳交涉，专业指导能显著降低后续风险。TEBIQ 提供专业咨询服务（[预约](/consultation)）。
