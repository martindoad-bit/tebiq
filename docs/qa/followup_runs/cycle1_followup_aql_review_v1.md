#### sim-followup-v1-006

整体可用。答案能持续追踪“酒店前台是否属于技人国”“搬行李/打扫是否变成体力劳动”“先アルバイト入社是否安全”“留学期间 28 小时超时是否影响变更”。

#### sim-followup-v1-010

追问链条稳定。Round 2 对“超过一年机场说一下”明确纠偏，质量较高。Round 3 能把“住民票/健康保险”和“保留日本工作”联系起来。

## 归因

### 工程 bug

- `spouse-divorce-separation` 被注入到技人国待机、离职接单等场景。
- `kazoku-taizai-yoken` 被注入到留学转技人国酒店前台、不许可再申请等场景。
- `weak_latest_followup_overlap` 对多轮追问误报较多，无法直接代表答案没有接住追问。

### 事实卡 / DOMAIN 问题

- 留学转酒店前台与 CEFR B2 / 国际业务职种可行性。
- 半年海外照护时住民票、公司健康保险、休职无薪的处理边界。
- 不许可后期限临近、再申请、特例期间与资格外活动是否可继续。
- 离职后接海外客户自由设计单是否构成资格外活动、自营活动或税务风险。

### 启发式误报

- 多数 `weak_latest_followup_overlap` 是误报。答案没有逐字重复最新追问关键词，但语义上已回应。
- 未发现 question-list-only。答案都有实质判断和下一步。
- 未发现 language drift。中文输出稳定，日文术语使用合理。

## 下一轮 Top 3

1. **修 fact card routing 噪音**
   不相关事实卡虽然目前未污染答案，但长期会导致误导。优先修 `spouse-divorce-separation`、`kazoku-taizai-yoken` 在技人国/留学/不许可场景的误注入。

2. **高风险追问加“状态前置确认”**
   对“不许可后能不能打工”“离职后接单”“海外半年住民票/保险”等问题，答案必须先确认当前在留状态、资格外活动许可、学校/雇佣关系是否仍有效，再给行动建议。

3. **降低过度确定表达**
   对 DOMAIN 边界问题，不要用“都不用”“唯一”“必须停止”这类绝对句收束。改成“在 X 前提下通常可以/不建议；若 Y 则必须另行确认”，既保留行动感，也降低误导风险。

## 结论

追问 after-fix 通过 AQL 质量复核的“可继续内测”门槛。它已经不是上下文断裂问题，主要剩余风险是事实卡 routing 和高责任法律边界表述。下一轮最值得修的是 routing 噪音和高风险追问的状态确认，而不是继续扩大样本或单纯增加答案长度。
