## V2-01 ★ 経営・管理 → 技人国

```yaml
query: 我是经营管理签证，公司经营不下去想去新公司上班，能不能转技人国？

intent: visa-change-management-to-engineer

template: visa-change

answer:
  最紧的两件:
    - 1. 找接收公司 + 签雇用契約书：业务内容必须明确属于「自然科学 / 人文科学 / 国際業務」三领域
    - 2. 自身学历或职歴 vs 新岗位整合性：大学卒以上 + 専攻関連性，或 10 年以上同领域実務経験
  步骤:
    - 1. 接收公司 — 找已签 内定 / 雇用契約 的公司，业务内容必须符合技人国 三领域
    - 2. 岗位是否属于技人国 — 自然科学 / 人文科学 / 国際業務 三领域以内（販売 / 現場作業 / 単纯接待 不算）
    - 3. 学历 / 経歴 与岗位匹配 — 大学卒（含 専門士 部分）或 10 年以上同领域実務経験
    - 4. 雇用契約 — 必须書面 + 月給 + 業務内容 + 期間明示
    - 5. 原公司 / 代表关系处理 — 経営・管理 当代表時 → 退任登記 + 商業登記 + 法人解散 / 譲渡 / 役員変更（与新签証不可同时主导经营）
    - 6. 在留資格変更許可申請 — 新公司在留申請 + 立证書類齐备 → 居住地管轄 出入国在留管理局
  期限:
    - 在留期限到期前 6 个月开始准备
    - 取得内定 → 立即申「在留資格変更許可申請」
    - 役員退任登記 → 在留資格変更前完成（避免「同时双重身份」立证不清）
  不做会怎样:
    - 不办 役員退任登記 + 直接申技人国 → 入管会比对商業登記簿 → 「経営・管理 + 技人国」双重身份矛盾 → 不许可
    - 业务内容不符三领域硬递（如販売 / サービス員）→ 不许可 + 留下记录
    - 学历不达标 + 実務経験不足 → 通常无法直接转技人国，需评估 経営・管理 縮小 / 高度専門職 / 帰国 等其他路径

复制给客户: |
  経営・管理 转技人国 不是「公司换一下就行」— 必须满足 4 件事：① 接收公司业务符合自然科学 / 人文科学 / 国際業務 三领域；② 自身学历或 10 年実務 与岗位匹配；③ 雇用契約 書面 + 業務内容 + 月給；④ 原公司 役員退任登記 + 商業登記 完成。今天先确认前 3 件 + 取得内定。如果是中小型公司或学历边界，可以先和行政書士聊一下。

why_this_template: |
  visa-change 模板：当前签证 + 目标签证 + 接收公司 + 学历经历整合性 + 役員关系处理 = 5 个核心轴，必须全部立证。模板把 5 轴拆成步骤清单 + 4 期限节点 + 4 失败后果，覆盖经管转人文最高频实务陷阱（役員退任 + 三领域整合）。

must_not_answer:
  - 不要答成「経営・管理 续签」内容（visa-renewal-management）
  - 不要答成「経営・管理 → 永住」（visa-change-management-to-permanent）
  - 不要答成「公司休眠 → 経営・管理 影响」（dormant-company-management-impact）
  - 不要答成「資本金不够」（management-capital-insufficient）
  - 不要省略役員退任登記步骤（高频遗漏点）
```

## V2-02 技人国 → 経営・管理（人文转经管）

```yaml
query: 我是技人国上班族 想自己开公司 怎么转经管签证？

intent: visa-change-engineer-to-management

template: visa-change-with-incorporation

answer:
  最紧的两件:
    - 1. 設立公司 + 達成新规要件（資本金 3000 万円 / 常勤職員 1 名以上 / 経営学修士 等）
    - 2. 在留資格変更許可申請：从技人国 转 経営・管理（涉及辞职 + 新公司起步）
  步骤:
    - 1. 公司設立準備 — 株式会社 / 合同会社 选择 + 商号 + 事業目的 + 本店所在地
    - 2. 設立资本金 — 自己出资 / 亲属 / 投资人 / 跨境时 ODI 备案
    - 3. 法務局 商業登記 — 設立登記（2 週間）
    - 4. 税務署 / 都道府県税事務所 / 市町村役場 — 法人設立届出
    - 5. 招聘常勤職員 1 名以上（日本人 / 永住者 / 定住者 等）+ 加入厚生年金 + 健康保険
    - 6. 事務所 — 賃貸契約（法人名义 + 事業使用可）+ 看板 + 平面图
    - 7. 在留資格変更許可申請 — 立证书類齐备 → 居住地管轄入管
    - 8. 原公司 — 在变更前办理退职 / 雇用契約解除
  期限:
    - 公司設立 + 招聘常勤 → 通常 6-12 个月
    - 在留期限到期前 6 个月开始 + 在留資格変更 审查 1-3 个月
    - 経過措置（截至 2028/10/16）— 改正前已设立公司可适用旧规
  不做会怎样:
    - 公司未达新规 + 无経過措置 → 不许可
    - 未招常勤職員 → 経営実態 立证不足
    - 仅事務所沒有実態（共享办公 / 自宅兼用边界）→ 立证不足
    - 同时持技人国 + 経営・管理 双重身份 → 入管視為矛盾

复制给客户: |
  技人国 转 経営・管理 不是「注册个公司就行」— 新规要 資本金 3000 万円 + 常勤職員 1 名以上 + 経営学修士（経過措置至 2028/10/16）。今天先确认 3 件事：① 資本金 + 来源；② 拟招常勤職員 候选；③ 事務所 法人名义 賃貸契約。整个准备期 6-12 个月。如果跨境出資或经验边界，可以先和行政書士 + 司法書士 + 税理士 聊一下。

why_this_template: |
  visa-change-with-incorporation 模板：技人国 转 経営・管理 不仅是签证变更，更是公司設立 + 経営実態 + 経過措置 三轴整合。模板覆盖公司設立 7 步 + 在留資格変更 + 经過措置 时间表。

must_not_answer:
  - 不要答成「技人国 续签」（visa-renewal-engineer）
  - 不要答成「技人国 → 永住」（visa-change-engineer-to-permanent）
  - 不要省略「経過措置」时间窗口
```

## V2-03 特定技能 1 号 → 技人国

```yaml
query: 我特定技能1号还有2年 想转技人国 能行吗

intent: visa-change-skilled-to-engineer

template: visa-change-with-education-check

answer:
  最紧的两件:
    - 1. 学历 / 実務経験 必须满足技人国 — 大学卒以上 或 10 年同领域実務（特定技能 期間可计入）
    - 2. 接收公司 + 业务内容 必须符合三领域（自然科学 / 人文科学 / 国際業務）
  步骤:
    - 1. 自查学历是否「大学卒以上」+ 専攻 与 拟岗位 相关
    - 2. 自查特定技能 期间 + 之前経歴 是否累计 10 年以上同领域実務（替代学历路径）
    - 3. 找接收公司 + 雇用契約（业务内容必须三领域以内）
    - 4. 准备「在留資格変更許可申請」+ 立证書類
    - 5. 学历 / 実務都不达 → 评估 経営・管理 起業 / 高度専門職 / 帰国
  期限:
    - 通算 5 年到期前 6 个月开始转换准备
    - 在留資格変更 审查通常 1-3 个月
    - 通算 5 年到期日 = 法令明文，不得超过
  不做会怎样:
    - 学历不达 + 実務不足 → 不许可
    - 特定技能 通算 5 年到期不申请 → 在留期限失效 → 不法滞在风险
    - 業務内容仍是現場作業 → 不许可

复制给客户: |
  特定技能 1 号 转技人国 不是「自动」 — 必须有内定 + 业务内容符合三领域 + 大学卒以上（或 10 年実務）。今天先做 3 件事：① 比对学历 + 専攻 vs 拟岗位；② 整理特定技能 期间 + 之前経歴 是否凑够 10 年実務；③ 找接收公司签内定。如果学历或实务边界，可以先和行政書士聊一下其他路径（経管起業 / 高度専門職）。

why_this_template: |
  visa-change-with-education-check 模板：特定技能 → 技人国 最大障碍是学历 + 業務内容 二选一不可缺。模板优先做学历 / 実務 自查，再确认接收公司业务内容。

must_not_answer:
  - 不要答成「特定技能 → 永住」（特定技能与永住路径独立）
  - 不要答成「特定技能 1 号 → 2 号 移行」（同业种内升级，不是转换）
  - 不要省略「業務内容三领域」
```

## V2-04 留学 → 技人国

```yaml
query: 我是留学生明年毕业 想直接进公司 怎么转技人国签证

intent: visa-change-student-to-engineer

template: visa-change-fresh-graduate

answer:
  最紧的两件:
    - 1. 拿到内定 + 業務内容 符合「自然科学 / 人文科学 / 国際業務」三领域
    - 2. 在留期限到期前 / 毕业前 6 个月递「在留資格変更許可申請」（应届 4 月入职 → 3 月初递最稳）
  步骤:
    - 1. 找内定（业务内容必须三领域以内）
    - 2. 確認自己学历 — 大学卒（含修士 / 博士）或 専門学校卒「専門士」（部分情形可，须専攻関連性强）
    - 3. 取得 卒業証明書 + 成績証明書（含専攻内容）
    - 4. 拟雇主提供：雇用契約書 + 業務内容説明書 + 公司決算書 + 商業登記簿謄本
    - 5. 在留資格変更許可申請（4,000 円）→ 居住地管轄入管
    - 6. 4 月入职最稳：3 月初递申 → 3 月底 / 4 月初下签 → 4 月入职
  期限:
    - 应届 4 月入职：建议毕业 / 在留期限到期前 6 个月开始
    - 3 月初递「在留資格変更」最稳
    - 在留期限失效后申请 → 不法滞在（不可）
  不做会怎样:
    - 内定后不及时变更 + 在留期限失效 → 不法滞在 + 退去强制
    - 业务内容不符三领域硬递（販売 / サービス員）→ 不许可
    - 専門学校卒 + 業務内容跨専攻 → 不许可率高

复制给客户: |
  留学转技人国 不是「自动」— 必须有内定 + 业务内容符合三领域 + 大学卒以上（或専門学校卒「専門士」+ 専攻強関連）。今天先做 3 件事：① 比对学历 / 専攻 与岗位的関連性；② 拿雇用契約 + 業務内容説明書；③ 在留期限到期前 6 个月开始申「在留資格変更」（4 月入职 3 月初递最稳）。如果是専門卒或业务内容边界，可以先和行政書士聊一下。

why_this_template: |
  visa-change-fresh-graduate 模板：应届毕业生有明确时序（4 月入职 / 3 月初递）+ 専攻関連性是最大障碍。模板按时间倒推 + 学历分类（大学卒 vs 専門卒）+ 業務内容三领域核对。

must_not_answer:
  - 不要答成「留学 → 経営・管理 起業」（visa-change-student-to-management）
  - 不要答成「留学 → 特定技能」（不同路径）
  - 不要答成「資格外活動 28 時間 / 週」（这是在学中打工，与变更无关）
```

## V2-05 ★ 公司休眠 → 国民年金

```yaml
query: 我们公司休眠了 我个人还要不要交国民年金

intent: dormant-company-personal-pension

template: pension-switch-on-status-change

answer:
  最紧的两件:
    - 1. 公司休眠不等于个人年金义务消失。先确认你是否仍有厚生年金覆盖
    - 2. 资格丧失日 起 14 日内必须切换 国民年金 第 1 号 + 国民健康保険
  步骤:
    - 1. 个人是否仍有厚生年金覆盖 — 查 ねんきんネット 或 公司给的「健康保険資格喪失証明書」
    - 2. 资格丧失日 — 喪失証明書 上的日期就是 14 日窗口起点（公司应在喪失日起 5 日内交付证明）
    - 3. 国民年金第 1 号 — 携 喪失証明書 + 在留カード + マイナンバー → 居住地 区役所 / 市役所 国保年金課 加入
    - 4. 国民健康保険 — 同窗口同时办（一站式）
    - 5. 区役所 / 年金事务所 — 区役所国保年金課 = 一站办；年金事務所 单独办国民年金时去
  期限:
    - 资格丧失日起 14 日内（国民年金 + 国民健康保険 加入义务）
    - 喪失証明書 取得：公司应在 5 日内发给本人
    - 国保保険料 軽減 / 国民年金 免除：同窗口办或后续补申
  不做会怎样:
    - 14 日内未办：保険料 遡及徴収（最大 2 年分）+ 督促状
    - 病院就诊 100% 自費（後日还付申请繁琐）
    - 永住申請 直近 2 年「公的義務履行」评价负分
    - 国民年金 长期未缴：将来年金额减少 + 老龄给付资格不足

复制给客户: |
  公司休眠不等于个人年金 / 健保义务消失。资格丧失日确认后 14 日内必须切到 国民年金 第 1 号 + 国民健康保険，否则保险料会被 遡及徴収 最大 2 年分 + 影响永住审查。今天先去公司要「健康保険資格喪失証明書」，然后带 在留カード + マイナンバー 去居住地 区役所 国保年金課 一站办理。如果同时收入低或申请倒産特例軽減，可以先和社労士聊一下。

why_this_template: |
  pension-switch-on-status-change 模板：核心是「公司事件 → 个人公的义务履行」的因果链。用户常误以为公司休眠 = 个人不用交（错误）。模板先纠偏 + 立即给 14 日窗口 + 区役所一站办路径。经営・管理 在留资格影响是另一条线，不抢主答。

must_not_answer:
  - 不要答成「公司休眠 経営・管理 在留资格影响」（dormant-company-management-impact）
  - 不要答成「年金免除申请」（pension-exemption）— 这是后续可选，不是主答
  - 不要答成「任意継続健康保険」— 仅当原健康保険加入 2 ヶ月以上 + 退职后 20 日内申且短期回归的边界路径
  - 不要省略「14 日窗口」— 这是法令明文核心
```

## V2-06 公司休眠 + 経営・管理 在留资格

```yaml
query: 我经管签证 公司决议休眠 我的签证会怎么样

intent: dormant-company-management-impact

template: visa-status-change-on-business-event

answer:
  最紧的两件:
    - 1. 公司休眠 ≠ 在留资格立即取消，但経営活動消失 → 续签时経営実態无法立证 → 不许可可能
    - 2. 评估窗口：在留期限剩余 + 休眠决议日 + 経営活動再开 / 在留資格変更 / 帰国 三选一
  步骤:
    - 1. 确认在留期限剩余期 + 休眠决议日 + 商業登記
    - 2. 路径 A — 重启経営活動（决议复活 + 取引先回归 + 役員報酬支払 + 立证整合）
    - 3. 路径 B — 在留資格変更（→ 技人国 / 家族滞在 / 高度専門職 / 等）
    - 4. 路径 C — 放棄経営・管理 + 离日（短期内不打算回归）
    - 5. 个人厚生年金 / 健康保険 → 14 日内切 国民年金 + 国保（看 V2-05）
  期限:
    - 在留期限剩余 < 6 ヶ月 + 休眠 → 立即决策
    - 休眠决议日起：商業登記是必须 + 入管届出 14 日内
    - 経営活動復活：建议在续签前 1-2 ヶ月有実態回归
  不做会怎样:
    - 休眠 + 拖到在留期限失效 + 不変更 → 不法滞在 + 退去强制
    - 续签时経営実態立证不足 → 不许可
    - 同时不交個人 国民年金 / 国保 → 公的義務不履行 → 永住封路

复制给客户: |
  経管签证 公司休眠 ≠ 签证立即取消，但経営活動消失会让续签时経営実態无法立证。今天先确认 3 件事：① 在留期限剩余多久；② 休眠是「臨時」还是「長期」；③ 是否打算重启経営 / 转技人国 / 帰国。其中如果在留剩 < 6 ヶ月 必须立即決策。这个情况建议先和行政書士 + 税理士 聊一下三个路径的可行性。

why_this_template: |
  visa-status-change-on-business-event 模板：公司事件触发签证连锁影响 — 必须给出多路径决策（重启 / 变更 / 放棄）+ 时间窗口 + 不行动的不可逆后果。

must_not_answer:
  - 不要答成「公司休眠 国民年金切换」（dormant-company-personal-pension） — 这是个人年金线
  - 不要答成「経営・管理 续签」（visa-renewal-management） — 这是正常状态续签
  - 不要省略路径 B（在留資格変更）— 用户最容易遗漏的选项
```
