# TEBIQ 项目记忆

> 每次开始工作前先读这个文件。包含产品认知、技术决策、关键边界。

## 产品定位

**TEBIQ = 你的在日生活好帮手 / 在日生活のお守り**

不是续签工具、不是信息站、不是社区。是面向在日外国人家庭的 AI 生活管家。

## 核心用户

第一版明确目标：在日中文母语者（中国大陆 + 港澳台 + 部分东南亚华裔）。
战略上未来覆盖所有外国人，但第一版不做多语言。

特定技能签证人群中越南/菲律宾/印尼大头不是中文用户，第一版不是目标。

## 双核心功能

1. 续签自查（免费引流，自媒体主入口）
2. 拍照即懂（订阅核心，任何日文文件拍照即理解）

## 收费策略（已锁定 v2）

| 层级 | 价格 | 包含 |
|---|---|---|
| 免费 | ¥0 | 自查 + 拍照 3次/月 + 政策阅读 |
| 基础会员 | ¥280/月 或 ¥2,800/年 | 拍照 30次 + 档案 + 提醒 + 材料包9折 |
| 高级会员（暂不上线，schema 预留） | ¥980/月 | 无限 + 全家 + 周报 + 咨询 |
| 续签材料包 | ¥980 一次性 | 个性化材料 + 申请书指南 |
| 专家咨询 | ¥9,800/30分钟 | 复杂 case 兜底 |

首月策略：基础会员首月 ¥1（不是免费，建立"已付费"心理）。

分享机制：分享链接 → 朋友注册 → 双方各得 1 个月基础会员免费。

## 不做的事

- 抽成（任何形式）
- 电商
- 社区
- 申请书代写（行政书士法风险）
- 书士会员费
- 价格 A/B 测试（创始人决定，等"哇时刻"再说）
- 第一版不做家庭 UI（schema 预留 family + members 结构）
- 第一版不做多语言

## 技术决策（已锁定）

- Next.js 14 App Router + TypeScript strict + Tailwind
- 数据库：Supabase Postgres（Tokyo region），数据必须在日本境内
- ORM：Drizzle ORM
- 认证：手机号 + SMS OTP（保留并升级现有系统）
- 支付：Stripe（卡 / PayPay / 便利店）
- AI：AWS Bedrock Claude（Tokyo region）
- 通知：阶段 0-1 邮件优先，App 上线后 App Push 主、邮件兜底，后期加 LINE
- 仓库：GitHub martindoad-bit/tebiq

## Schema 原则

- 订阅以 family 为单位，不是 member
- 第一版每 family 只有 1 个 member（is_owner=true），UI 不暴露家庭概念
- 将来加家庭功能不需要数据迁移
- 所有静态数据（政策、概念、监控列表、quiz 题库、材料库）保留 code-as-data，但通过新类型系统访问
