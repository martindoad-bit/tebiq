## 产品定位

**TEBIQ = 你的在日生活好帮手 / 在日生活のお守り**

不是续签工具、不是信息站、不是社区、不是档案中心。是面向在日中文母语者的「日常麻烦事工具集」。

## 核心用户（已收窄）

**第一版明确：在日中文母语者**（中国大陆 + 港澳台 + 部分东南亚华裔）。

不是「全外国人」。特定技能签证人群中的越南/菲律宾/印尼大头不是中文用户，第一版不是目标。

战略上未来覆盖所有外国人，但 1.0 不做多语言（详见多语言策略）。

## 1.0 产品形态（已锁定）

**主页 = 工具入口 + 历史记录**，不是 dashboard。

理由：1.0 阶段档案数据稀薄（用户只填了几个字段、做了几次自查），dashboard 形态会暴露产品的空。等 2.0 数据沉淀后再升级为 dashboard。

**两个核心功能**：
1. 续签自查（免费引流，自媒体主入口）
2. 拍照即懂（订阅核心，任何日文文件拍照识别）

**主导航 4 个 tab**：首页 / 我的档案 / 知识 / 我的

档案信息藏在「我的档案」tab，不是首页。

## 增长机制

**邀请朋友送会员**：分享给 3 个朋友，双方各得 7 天会员体验。

触发时机：用户做完续签自查看到结果时 + 拍照配额用完时。不是付费后才送。

## 多语言策略

1.0 不做多语言。所有精力投入中文版。

第二语言不是英文，而是日文 B 端版（日本企业 HR 管理外国员工签证）。

越南语/印尼语/菲律宾版不自己做，将来用合伙人模式输出技术底座。

理由：多语言瓶颈不是技术，是「懂日本签证 + 该语言母语」的稀缺人才。每加一种语言需要长期审核员（不是一次性翻译），每年 ¥100-300 万人力成本。1.0 阶段不能承担。

## 文案规范

- 中性、克制、信息密度高
- 不写戏剧化问句（"看不懂，怕出事？" 这种是网页营销，不是 App 文案）
- 不写 AI 戏剧腔（"我帮你盯着 3 件事" 这种）
- 按钮直接叫功能名（"拍照识别"、"续签自查"），不要 emoji + 戏剧动词

## 中日混合规则（重要）

TEBIQ 用中文为主，但所有日本生活的专有名词保留日文原文。这不是翻译策略，是产品定位 — 用户身份是「在日华人」，他们用中文聊天但用日文办手续。

**保留日文原文的类别**：
- 签证类型：技術・人文知識・国際業務、経営・管理、家族滞在、特定技能、定住者、永住者
- 政府机构：入国管理局（入管）、市役所、税務署、年金事務所、ハローワーク
- 政府文件：在留カード、住民票、住民票記載事項証明書、確定申告書、源泉徴収票、納税証明書、課税証明書
- 专业概念：住民税、所得税、住民基本台帳、マイナンバー、年金、健康保険
- 生活场景：敷金、礼金、保証人、引っ越し、確定申告、年末調整、契約書

**翻译成中文的类别**：
- 普通动词：提交、准备、更新、保存、查看
- UI 操作：取消、确认、返回、分享
- 解释性内容：风险等级、判断逻辑、行动建议
- 应用层概念：会员、订阅、邀请

**判断标准**：用户去日本现实场景（市役所窗口、政府文件、官网）会看到这个词的日文原文吗？会 → 保留日文。不会（是产品自己的概念）→ 翻译。

**例子**：

错误：「你的居留期限延长还有 60 天」
正确：「你的**在留期間更新**还有 60 天」

错误：「市政府发的居民税通知」
正确：「**市役所**发的**住民税**通知」

错误：「提醒你做年度报税」
正确：「提醒你做**確定申告**」

## 视觉规范

Visual Polish 2.1 后，TEBIQ 的视觉从「冷灰绿文具感」回到更有生命力的日本生活工具：成熟、清楚、有人味，但不儿童化。旧的「政府蓝 + 明橙 + 奶油米」和上一版过冷的墨绿/赤金都不再作为主框架。

- 调性：温暖陪伴 + 工具感 + 日式克制。页面要像可长期使用的生活文具：安静、清楚、可信，不喊口号。
- 主色：深海军蓝 #18324A。用于标题、正文重点、主图标和深色控件；比政府蓝暖一点，比墨绿更有精神。
- 强调色：珊瑚赤 #E56F4F。用于 CTA、进度、重要行动；替代幼稚亮橙和沉闷赤金，主按钮上使用白字。
- 背景：温暖纸白 #FAF7F1；卡片：#FFFFFF；浅强调底：#FFF0E8；边线：rgba(24, 50, 74, 0.12)。
- 辅助色：浅蓝灰 #EAF3F7、成功 #2E7D65、警示 #C64F45、正文灰 #405161、次级灰 #6E7A84、浅灰 #B7C0C7、chip #F1EAE1。
- 字体：-apple-system, BlinkMacSystemFont, "PingFang SC", "Hiragino Sans", "Yu Gothic", "YuGothic", "Noto Sans SC", "Noto Sans CJK SC", sans-serif。日文术语优先让系统日文字体参与渲染。
- 字重：正文 400，按钮 / label / 小标题 500；只有长正文 SEO 标题、材料包封面等需要层级的场景可局部使用 600。避免整站 600/700 造成幼稚和粗糙。
- 字距 / 行高：中文 letter-spacing 0，日文术语容器可用 0.015em；中文正文 line-height 1.62，日文或中日混排长段落 1.72，英文/数字 1.5。
- 数字：日期、金额、计数使用 numeric 字体栈（SF Mono / SFMono-Regular / Noto Sans Mono CJK SC / ui-monospace）和 tabular-nums，提升工具感。
- 圆角：卡片 16px，按钮 14px，标签 / chip 10px。核心行动区可以更亲和，SEO 长文和表单密集区保持收敛。
- 阴影：中性蓝灰阴影，不用大面积彩色发光。卡片使用 `0 1px 0 rgba(24,50,74,.035), 0 10px 28px rgba(24,50,74,.06)`；raised 用更宽但低透明度的蓝灰阴影；CTA 只保留轻微珊瑚赤投影。
- 图标：优先 lucide 线性图标，小尺寸 stroke-width 1.5-1.7；不要混用硬填色插画图标和线性图标；旧 #1E3A5F / #F6B133 不再硬编码。
- 插画：主插画用 Image 2.0 生成的成熟静物 bitmap，不用人物、吉祥物、礼物盒、文件角色和泛 AI 清单图。画面应像日本生活桌面上的真实物件：文件、日历、手机、信封、印章、留白和自然光。SVG 只允许作为小型 UI 空状态或图标，不承担主视觉。

## 5 维度产品决策标尺（每次大方向变化用这个 review）

每次重要产品决策变化时，对新方案打分（每项 2 分，共 10 分）：

1. **用户视角真实性** — 描述了真实场景还是空想
2. **AI 时代独特性** — 在 AI 时代有不可替代位置吗
3. **1.0 可执行性** — 2-4 周能上线吗
4. **商业模型健康度** — 单位经济成立吗
5. **创始人能力匹配** — 和创始人的具体能力 match 吗

低于 7 分的方案不要进入 production。
